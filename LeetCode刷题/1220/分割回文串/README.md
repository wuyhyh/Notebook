LeetCode第131题是《分割回文串》（Palindrome Partitioning），题目要求给定一个字符串 `s`，将其分割成一些子串，使得每个子串都是回文串。返回所有可能的回文分割方案。

### 题目描述

给定字符串 `s`，将其分割成一些子串，使得每个子串都是回文串。返回所有可能的回文分割方案。

**回文串**：一个字符串是回文的，意味着它从前往后读和从后往前读是一样的。

### 示例

**示例 1：**

```plaintext
输入: s = "aab"
输出:
[["aa", "b"], ["a", "a", "b"]]
```

**示例 2：**

```plaintext
输入: s = "a"
输出:
[["a"]]
```

### 解题思路

这个问题可以通过 **回溯算法** 来解决。回溯算法非常适合解决组合问题，在这里我们需要将字符串分割成回文子串。核心思路是从每个位置开始，依次检查是否能从当前位置分割出一个回文子串，递归进行分割。

#### 具体步骤：

1. **回溯函数设计**：
    - 从字符串的当前位置开始，尝试所有可能的子串。
    - 对每个子串，检查它是否是回文串。
    - 如果是回文串，将其加入当前的分割路径中，然后递归地处理剩余的部分。
    - 如果遇到末尾的情况，说明找到了一种有效的分割方案。

2. **回文检查**：在每次尝试分割时，我们需要检查当前子串是否是回文。可以使用双指针从两头向中间扫描。

3. **递归终止条件**：当当前字符串为空时，表示已经找到了一组有效的回文分割，将其加入结果。

### 代码实现

```python
def partition(s):
    result = []

    # 判断一个字符串是否为回文串
    def is_palindrome(sub):
        return sub == sub[::-1]

    # 回溯函数，cur_list 存储当前的分割方案
    def backtrack(start, cur_list):
        # 如果起始位置已经达到字符串的末尾，说明一组分割完成
        if start == len(s):
            result.append(cur_list[:])  # 将当前分割方案加入结果
            return
        
        # 从 start 开始遍历
        for end in range(start + 1, len(s) + 1):
            # 提取当前子串
            substring = s[start:end]
            if is_palindrome(substring):
                cur_list.append(substring)  # 将回文子串加入当前分割方案
                backtrack(end, cur_list)    # 递归处理剩余部分
                cur_list.pop()  # 回溯，移除当前子串
    
    # 从位置 0 开始回溯
    backtrack(0, [])
    return result
```

### 代码解析

1. **`is_palindrome(sub)`**：
    - 检查一个子串是否是回文串。通过 `sub[::-1]` 来获取字符串的逆序，如果原字符串和逆序相等，说明是回文串。

2. **`backtrack(start, cur_list)`**：
    - `start`：表示当前开始搜索的字符位置。
    - `cur_list`：当前分割路径，用来存储已经找到的回文子串。
    - **递归终止条件**：当 `start` 达到字符串的末尾时，说明当前路径是一个有效的回文分割方案，将其加入 `result`。
    - **子串遍历**：从当前位置 `start` 开始，遍历所有可能的子串，并检查它们是否为回文串。如果是回文串，递归处理剩余部分。

3. **`backtrack(0, [])`**：
    - 从字符串的起始位置开始进行回溯，空列表 `[]` 用来存储当前的回文分割方案。

4. **回溯过程**：
    - 每次尝试一个新的回文子串时，将其加入当前路径 `cur_list` 中，递归进入下一层。
    - 当返回上一层时，通过 `cur_list.pop()` 来移除最后一个加入的回文子串，回溯到上一状态，尝试其他可能的子串。

### 时间复杂度

- **时间复杂度**：`O(2^n * n)`，其中 `n` 是字符串的长度。每个字符有两种选择：要么被包含在回文子串中，要么不包含。最坏情况下每个递归分支都要遍历字符串的一部分进行回文检查，检查一个子串是否是回文的时间复杂度为 `O(n)`。因此，总时间复杂度是 `O(2^n * n)`。

- **空间复杂度**：`O(n)`，递归栈的最大深度为 `n`（即字符串的长度），因此空间复杂度为 `O(n)`。

### 示例解析

以 `s = "aab"` 为例，回溯的过程如下：

1. **第一次递归**：
    - 从 `s[0]` 开始，首先尝试子串 `"a"`，它是回文串，将其加入当前分割路径 `["a"]`。
    - 递归处理剩余部分 `"ab"`。

2. **第二次递归**：
    - 继续分割 `"ab"`，尝试子串 `"a"`，它是回文串，将其加入当前路径 `["a", "a"]`。
    - 递归处理剩余部分 `"b"`。

3. **第三次递归**：
    - 处理剩余的 `"b"`，尝试子串 `"b"`，它是回文串，将其加入当前路径 `["a", "a", "b"]`。
    - 递归到达末尾，记录这一分割方案 `["a", "a", "b"]`。

4. **回溯**：
    - 回溯到第二层，尝试其他分割方法，最终得到另一种分割方案 `["aa", "b"]`。

最终返回结果：

```plaintext
[["aa", "b"], ["a", "a", "b"]]
```

### 总结

- **回溯算法**：用于解决所有可能的组合或分割问题，尤其适合生成所有满足条件的方案。
- **回文检查**：每次分割后都需要检查子串是否为回文，确保每个子串都符合要求。
- **剪枝**：通过在递归中添加回文检查条件，避免了无效分割的路径。

这个问题展示了如何使用回溯算法解决字符串分割问题，特别是当我们需要满足某种条件（如回文）时，如何利用递归和回溯生成所有有效解。
