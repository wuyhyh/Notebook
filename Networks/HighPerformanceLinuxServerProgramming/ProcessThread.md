这是个很好的问题，尤其是你打算深入理解 Linux 的多进程与多线程编程时，掌握核心的 **数据结构（编程模型中）** 和 **系统函数（API）
** 是非常关键的。

下面我会按层次结构地总结你应该掌握的内容，包括：**关键概念、常用系统调用/API、相关数据结构、以及推荐补充的知识点。**

---

## 🧠 一、核心概念总览

| 类别      | 关键点说明                        |
|---------|------------------------------|
| 多进程模型   | 每个进程有独立地址空间，进程间通信复杂但安全性高。    |
| 多线程模型   | 同一进程内多个线程共享地址空间，通信高效，但需注意同步。 |
| 进程/线程状态 | 就绪、运行、等待、僵尸等。                |
| 同步与互斥   | 避免竞态条件、死锁。                   |
| IPC     | 管道、共享内存、消息队列、信号、socket 等。    |

---

## 📚 二、进程（Process）编程要掌握的函数和结构

### ✅ 1. **进程创建与终止**

| 函数/系统调用                  | 说明       |
|--------------------------|----------|
| `fork()`                 | 创建子进程    |
| `execve()`/`execlp()`    | 子进程加载新程序 |
| `wait()` / `waitpid()`   | 等待子进程结束  |
| `exit()` / `_exit()`     | 退出当前进程   |
| `getpid()` / `getppid()` | 获取进程ID   |
| `kill()` / `raise()`     | 向进程发送信号  |

---

### ✅ 2. **进程间通信（IPC）**

| 通信方式       | 相关函数或接口                                               |
|------------|-------------------------------------------------------|
| 管道（pipe）   | `pipe()`, `read()`, `write()`                         |
| 命名管道（FIFO） | `mkfifo()`                                            |
| 共享内存       | `shmget()`, `shmat()`, `shmdt()`                      |
| 消息队列       | `msgget()`, `msgsnd()`, `msgrcv()`                    |
| 信号量        | `semget()`, `semop()`, `semctl()`                     |
| 套接字 socket | `socket()`, `bind()`, `connect()`, `send()`, `recv()` |

---

### ✅ 3. **相关数据结构**

| 数据结构               | 来源/作用                                    |
|--------------------|------------------------------------------|
| `pid_t`            | 表示进程ID                                   |
| `struct sigaction` | 信号处理注册                                   |
| `struct stat`      | 文件状态信息（用于 `fstat()`）                     |
| IPC专用结构体           | 如 `struct shmid_ds`, `struct msqid_ds` 等 |

---

## 🧵 三、线程（Thread）编程要掌握的函数和结构（基于 POSIX pthread）

### ✅ 1. **线程创建与管理**

| 函数                 | 说明       |
|--------------------|----------|
| `pthread_create()` | 创建线程     |
| `pthread_exit()`   | 线程退出     |
| `pthread_join()`   | 等待线程结束   |
| `pthread_self()`   | 获取自身线程ID |
| `pthread_detach()` | 分离线程资源   |

---

### ✅ 2. **线程同步**

| 机制          | 相关函数                                                                     |
|-------------|--------------------------------------------------------------------------|
| 互斥锁（mutex）  | `pthread_mutex_init()`, `pthread_mutex_lock()`, `pthread_mutex_unlock()` |
| 条件变量        | `pthread_cond_wait()`, `pthread_cond_signal()`                           |
| 读写锁         | `pthread_rwlock_*` 系列                                                    |
| 屏障（barrier） | `pthread_barrier_*` 系列                                                   |
| 原子操作        | `__sync_*` 或 `atomic_*`（GCC 提供）                                          |

---

### ✅ 3. **线程相关数据结构**

| 数据结构                                | 说明           |
|-------------------------------------|--------------|
| `pthread_t`                         | 线程标识         |
| `pthread_attr_t`                    | 线程属性结构体      |
| `pthread_mutex_t`, `pthread_cond_t` | 线程同步原语       |
| `pthread_key_t`                     | 线程私有数据键（TLS） |

---

## 💡 四、建议补充学习内容

| 主题       | 内容                                      |
|----------|-----------------------------------------|
| 死锁与饥饿    | 如何识别和避免                                 |
| 内存可见性    | volatile、原子变量、内存屏障                      |
| 多核并发模型   | NUMA、Cache一致性                           |
| 信号安全函数   | 哪些函数可以在信号处理函数里调用？（如 `write()` 是安全的）     |
| I/O 多路复用 | `select()`, `poll()`, `epoll()`（进程异步并发） |

---

## 📘 推荐学习路线（顺序）：

1. 熟练掌握 `fork()/exec()` 等进程控制函数
2. 理解线程模型，学会用 `pthread_create()` 和 mutex
3. 学习 IPC 各种机制（从 pipe 到 socket）
4. 学会信号处理和多线程同步
5. 掌握 I/O 多路复用和高并发场景设计
