### **LeetCode 994 题 - 腐烂的橘子（Rotting Oranges）解决方案**

#### **问题描述**
给定一个二维网格，表示一个橘子园。每个单元格的值有三种可能：
- `0`：空格
- `1`：新鲜橘子
- `2`：腐烂橘子

腐烂的橘子每分钟会腐烂其上下左右相邻的新鲜橘子。每个腐烂橘子可以影响其他相邻的新鲜橘子。我们需要返回将所有新鲜橘子腐烂所需要的最短时间。如果无法腐烂所有新鲜橘子，则返回 `-1`。

#### **思路分析**
这是一个典型的 **广度优先搜索（BFS）** 问题。因为腐烂的橘子会像波纹一样从已经腐烂的橘子扩散到周围的新鲜橘子，而 BFS 能够模拟这一过程。

- **BFS**：从所有腐烂的橘子开始，逐层扩展，计算新鲜橘子变腐烂的时间。
- **队列初始化**：首先将所有腐烂橘子的坐标加入队列，并统计新鲜橘子的数量。
- **BFS 扩展**：每次 BFS 逐步腐烂四个方向的新鲜橘子，记录腐烂的时间。
- **边界条件**：如果有新鲜橘子永远无法腐烂，则返回 `-1`。

#### **解题步骤**
1. **队列初始化**：
    - 将所有腐烂橘子的坐标加入队列。
    - 统计网格中新鲜橘子的数量。

2. **BFS 过程**：
    - 使用队列逐层扩展腐烂的橘子，检查每个腐烂橘子的四个相邻位置。
    - 如果相邻位置是新鲜橘子，将其腐烂，并加入队列。
    - 每处理一层的腐烂，增加时间（分钟）。

3. **判断是否所有橘子都腐烂**：
    - 如果 BFS 完成后，还有新鲜橘子没有腐烂，则返回 `-1`。
    - 否则返回所需的最短时间。

#### **代码实现**

```python
from collections import deque

class Solution(object):
    def orangesRotting(self, grid):
        """
        :type grid: List[List[int]]
        :rtype: int
        """
        # 如果网格为空，返回0
        if not grid:
            return 0
        
        # 初始化队列，加入所有腐烂橘子的坐标
        queue = deque()
        fresh_count = 0
        rows, cols = len(grid), len(grid[0])
        
        # 将所有腐烂橘子加入队列，统计新鲜橘子的数量
        for r in range(rows):
            for c in range(cols):
                if grid[r][c] == 2:  # 腐烂橘子
                    queue.append((r, c))
                elif grid[r][c] == 1:  # 新鲜橘子
                    fresh_count += 1
        
        # 如果没有新鲜橘子，直接返回0
        if fresh_count == 0:
            return 0
        
        # BFS过程，模拟腐烂的扩散
        directions = [(-1, 0), (1, 0), (0, -1), (0, 1)]
        minutes = 0
        
        # 开始BFS
        while queue:
            size = len(queue)
            # 记录当前层次（每分钟）腐烂橘子的扩展
            for _ in range(size):
                r, c = queue.popleft()
                # 尝试四个方向扩散
                for dr, dc in directions:
                    nr, nc = r + dr, c + dc
                    # 判断新的位置是否合法且为新鲜橘子
                    if 0 <= nr < rows and 0 <= nc < cols and grid[nr][nc] == 1:
                        grid[nr][nc] = 2  # 将新鲜橘子变为腐烂
                        fresh_count -= 1
                        queue.append((nr, nc))
            
            # 只有当当前层级（即一轮腐烂扩展）结束后才增加分钟数
            if queue:  # 如果队列中还有腐烂橘子
                minutes += 1
        
        # 如果还有新鲜橘子未腐烂，返回-1
        return minutes if fresh_count == 0 else -1
```

#### **代码解析**
- **队列初始化**：将所有腐烂橘子的坐标加入队列，并统计新鲜橘子的数量。
- **BFS 扩展**：通过 BFS 模拟腐烂的橘子扩展过程，每扩展一层就增加一分钟。
- **时间计算**：每次扩展一个腐烂橘子，直到没有更多新鲜橘子可以腐烂。
- **返回值**：如果 BFS 完成后新鲜橘子数量为 0，则返回所需时间；如果剩下新鲜橘子，返回 `-1`。

#### **时间复杂度与空间复杂度**
- **时间复杂度**：遍历整个网格一遍来初始化腐烂橘子队列和新鲜橘子计数，且 BFS 每个节点最多处理一次。时间复杂度为 \(O(m \times n)\)，其中 \(m\) 是网格的行数，\(n\) 是列数。
- **空间复杂度**：BFS 需要使用队列存储腐烂橘子的坐标，队列大小最多为 \(O(m \times n)\)，因此空间复杂度为 \(O(m \times n)\)。

#### **示例**

##### 示例 1：

输入：
```python
grid = [
  [2,1,1],
  [1,1,0],
  [0,1,1]
]
```
输出：
```python
4
```
解释：4分钟后，所有新鲜橘子都会腐烂。

##### 示例 2：

输入：
```python
grid = [
  [2,1,1],
  [0,1,1],
  [1,0,1]
]
```
输出：
```python
-1
```
解释：有新鲜橘子永远不会腐烂，返回 `-1`。

#### **总结**
- 通过 BFS 模拟腐烂的橘子扩散过程，可以高效地计算出腐烂所有新鲜橘子的最短时间。
- BFS 通过逐层处理腐烂的橘子，确保了每一层的腐烂时间准确计算。
- 本解法时间复杂度和空间复杂度均为 \(O(m \times n)\)，适用于大规模网格的处理。
