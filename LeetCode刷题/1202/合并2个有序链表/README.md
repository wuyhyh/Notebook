第21题是“合并两个有序链表”（Merge Two Sorted Lists），要求将两个已排序的链表合并为一个新的排序链表，并返回该链表。

### 问题描述

给定两个单链表 `l1` 和 `l2`，它们是 **已排序** 的。将这两个链表合并为一个新的 **有序** 链表，返回新的链表。

### 示例

**示例 1:**

输入：
```plaintext
l1 = [1, 2, 4], l2 = [1, 3, 4]
```

输出：
```plaintext
[1, 1, 2, 3, 4, 4]
```

**示例 2:**

输入：
```plaintext
l1 = [], l2 = [0]
```

输出：
```plaintext
[0]
```

**示例 3:**

输入：
```plaintext
l1 = [2], l2 = []
```

输出：
```plaintext
[2]
```

### 解题思路

我们需要将两个 **已排序** 的链表合并为一个新的 **已排序** 链表。可以通过以下步骤来实现：

1. **创建一个虚拟头节点**：
    - 创建一个虚拟头节点 `dummy`，这样可以简化链表操作，避免在链表合并过程中需要对头节点的特殊处理。

2. **遍历两个链表**：
    - 使用两个指针 `l1` 和 `l2` 分别指向两个链表的当前节点。
    - 每次比较两个节点的值，较小的节点添加到结果链表中。
    - 将较小的节点的指针移动到下一个节点。

3. **处理剩余节点**：
    - 当其中一个链表遍历完后，将另一个链表剩余的部分直接连接到结果链表的末尾。

4. **返回结果链表**：
    - 返回虚拟头节点的下一个节点（即合并后的链表的头节点）。

### 代码实现

```python
# Definition for singly-linked list.
class ListNode:
    def __init__(self, val=0, next=None):
        self.val = val
        self.next = next

def mergeTwoLists(l1: ListNode, l2: ListNode) -> ListNode:
    # 创建一个虚拟头节点
    dummy = ListNode()
    current = dummy
    
    # 合并两个链表
    while l1 and l2:
        if l1.val <= l2.val:
            current.next = l1
            l1 = l1.next
        else:
            current.next = l2
            l2 = l2.next
        current = current.next
    
    # 连接剩余的部分
    if l1:
        current.next = l1
    if l2:
        current.next = l2
    
    # 返回合并后的链表
    return dummy.next
```

### 代码解释

1. **虚拟头节点**：
    - 创建一个虚拟节点 `dummy`，并设置 `current` 指向该虚拟节点，`current` 用来跟踪结果链表的当前节点。

2. **遍历两个链表**：
    - 使用 `while l1 and l2` 循环，比较两个链表当前节点的值，将较小的节点添加到合并链表中。
    - 根据哪个链表当前节点值较小，更新 `current.next`，并移动相应链表的指针（`l1` 或 `l2`）。

3. **连接剩余部分**：
    - 一旦一个链表遍历完，直接将另一个链表的剩余部分接到结果链表的末尾。

4. **返回结果链表**：
    - 返回 `dummy.next`，因为 `dummy` 是虚拟的头节点，实际的合并链表从 `dummy.next` 开始。

### 时间复杂度和空间复杂度

- **时间复杂度**：`O(n + m)`，其中 `n` 和 `m` 分别是两个链表 `l1` 和 `l2` 的长度。我们需要遍历每个链表一次。
- **空间复杂度**：`O(1)`，我们只使用了常数空间来存储指针，结果链表在原地构建。

### 总结

- 使用两个指针分别遍历两个已排序的链表，依次将较小的节点插入到结果链表中。
- 创建一个虚拟头节点来简化链表操作。
- 时间复杂度 `O(n + m)`，空间复杂度 `O(1)`，非常高效。
