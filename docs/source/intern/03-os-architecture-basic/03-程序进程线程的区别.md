# 03-程序进程线程的区别

## 1 程序是什么

程序是存放在磁盘上的可执行文件或脚本文件，本质上是一组指令和数据。

例如：

```bash
/bin/ls
/usr/bin/python3
./hello
test.sh
```

程序本身是静态的。它没有真正运行，也不会占用 CPU 执行时间。只有当程序被加载到内存并开始执行后，才会产生进程。

可以简单理解为：

```text
程序 = 磁盘上的文件
```

## 2 进程是什么

进程是程序运行起来之后的实例。

例如执行：

```bash
./hello
```

系统会把 `hello` 这个程序加载到内存中，并为它创建一个进程。

进程拥有自己独立的资源，例如：

```text
虚拟地址空间
代码段
数据段
堆
栈
打开的文件
环境变量
进程号 PID
```

可以简单理解为：

```text
进程 = 正在运行的程序
```

同一个程序可以启动多个进程。例如同时打开多个终端，或者同时运行多个 `vim`，它们使用的是同一个程序文件，但对应的是不同的进程。

## 3 线程是什么

线程是进程内部的执行流。

一个进程至少有一个线程，这个线程通常叫主线程。一个进程也可以创建多个线程，让多个执行流在同一个进程内部并发运行。

线程之间共享同一个进程的大部分资源，例如：

```text
代码段
数据段
堆
打开的文件
全局变量
```

但是每个线程也有自己独立的资源，例如：

```text
线程 ID
寄存器状态
栈
调度状态
```

可以简单理解为：

```text
线程 = 进程内部的执行单位
```

## 4 程序、进程、线程的关系

三者关系可以这样理解：

```text
程序存放在磁盘上
程序运行起来后变成进程
进程内部可以包含一个或多个线程
线程是真正被 CPU 调度执行的单位
```

示意图：

```text
程序文件
  |
  | 运行
  v
进程
  |
  +-- 线程 1
  +-- 线程 2
  +-- 线程 3
```

举例：

```bash
/usr/bin/firefox
```

这是一个程序。

当你打开 Firefox 浏览器后，系统中会出现 Firefox 相关进程。

每个 Firefox 进程内部又可能有多个线程，比如：

```text
界面线程
网络线程
渲染线程
磁盘缓存线程
```

## 5 进程和线程的主要区别

| 对比项 | 进程 | 线程 |
|---|---|---|
| 所属关系 | 系统资源分配单位 | 进程内部的执行单位 |
| 地址空间 | 进程之间通常相互独立 | 同一进程内线程共享地址空间 |
| 创建开销 | 较大 | 较小 |
| 通信方式 | 需要 IPC 机制 | 可以直接共享全局变量、堆内存 |
| 稳定性 | 一个进程崩溃通常不影响其他进程 | 一个线程崩溃可能导致整个进程崩溃 |
| 调度 | 可以被系统调度 | 线程是 CPU 调度的重要单位 |

## 6 为什么需要多进程

多进程适合任务之间相对独立、需要隔离的场景。

例如：

```text
一个 Web 服务器创建多个进程处理不同客户端连接
浏览器为不同页面创建不同进程
系统中的不同服务使用不同进程运行
```

多进程的优点：

```text
隔离性好
稳定性好
一个进程崩溃不容易影响其他进程
适合利用多核 CPU
```

缺点：

```text
创建和切换开销较大
进程之间通信相对麻烦
共享数据不如线程方便
```

## 7 为什么需要多线程

多线程适合在同一个进程内部同时处理多个任务。

例如：

```text
一个线程负责界面显示
一个线程负责网络收发
一个线程负责磁盘读写
一个线程负责后台计算
```

多线程的优点：

```text
创建开销比进程小
线程之间共享数据方便
适合处理并发任务
适合提高程序响应速度
```

缺点：

```text
线程之间共享内存，容易出现数据竞争
需要使用锁、信号量等同步机制
一个线程出错可能影响整个进程
调试难度比单线程程序高
```

## 8 Linux 中如何查看进程

使用 `ps` 查看进程：

```bash
ps aux
```

查看指定进程：

```bash
ps aux | grep ssh
```

查看进程树：

```bash
pstree
```

查看实时进程状态：

```bash
top
```

或者：

```bash
htop
```

查看当前 shell 的进程号：

```bash
echo $$
```

## 9 Linux 中如何查看线程

在 Linux 中，线程也可以被内核调度。可以使用下面命令查看某个进程中的线程。

先找到进程 PID：

```bash
ps aux | grep 程序名
```

查看该进程的线程：

```bash
ps -T -p <PID>
```

例如：

```bash
ps -T -p 1234
```

也可以查看 `/proc` 目录：

```bash
ls /proc/<PID>/task
```

其中每一个目录通常对应一个线程 ID。

## 10 常见理解误区

### 10.1 程序不等于进程

程序只是文件，进程是运行中的实例。

例如：

```text
/bin/ls 是程序
执行 ls 命令时产生的运行实例是进程
```

### 10.2 一个程序可以对应多个进程

同一个程序可以运行多次，每次运行都可以创建一个新的进程。

例如同时打开三个终端执行：

```bash
sleep 100
```

系统中会有多个 `sleep` 进程。

### 10.3 一个进程可以有多个线程

进程不是 CPU 执行的最小单位。进程内部可以有多个线程，线程才是真正参与调度执行的重要单位。

### 10.4 多线程不一定更快

多线程能不能提高性能，要看任务类型。

如果任务主要等待网络、磁盘、用户输入，多线程可以提高响应速度。

如果任务主要消耗 CPU，多线程能不能变快取决于 CPU 核心数、锁竞争、调度开销等因素。

## 11 简单实验

### 11.1 观察进程

执行：

```bash
sleep 1000
```

打开另一个终端：

```bash
ps aux | grep sleep
```

可以看到 `sleep` 程序运行后产生了一个进程。

### 11.2 查看进程号

```bash
sleep 1000 &
echo $!
```

`$!` 表示最近一个后台进程的 PID。

### 11.3 查看线程

如果系统中有多线程程序，可以使用：

```bash
ps -eLf | head
```

其中 `LWP` 字段可以理解为线程 ID。

也可以查看某个进程：

```bash
ps -T -p <PID>
```

## 12 总结

程序、进程、线程的区别可以用一句话概括：

```text
程序是静态文件，进程是程序运行后的资源容器，线程是进程内部真正执行代码的单位。
```

更简单地记忆：

```text
程序：放在磁盘上的代码
进程：运行起来的程序
线程：进程里面干活的执行流
```

在 Linux 系统中，理解程序、进程和线程的区别，是学习进程管理、系统调用、服务管理、并发编程和驱动开发的基础。
