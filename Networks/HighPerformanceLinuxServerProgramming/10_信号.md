# 第10章 信号

信号是由用户、系统或者进程发送给目标进程的信息，以通知目标程序某个状态的改变或者系统异常。

Linux信号可以由如下条件产生：

- 对于前台进程，可以从键盘发送，比如Ctrl+C
- 系统异常，比如浮点异常或非法内存访问
- 系统状态变化，比如alarm定时器
- 运行kill命令或者kill系统调用

服务器程序必须处理信号（或者选择忽略），以免异常终止。

## 10.1 Linux信号概述

### 10.1.1 发送信号

一个进程给其他进程发送信号使用kill()

```c++
int kill(pid_t pid, int sig);
```

### 10.1.2 信号处理方式

目标进程收到信号时，需要定义一个接收函数来处理：

```c++
typedef void (*__sighandler_t) (int);
```

## 10.2 信号函数

### 10.2.1 signal系统调用

要为一个信号设置处理函数，使用下面的系统调用：

```c++
_sighandler_t signal (int sig, _sighandler_t _handler);
int sigaction(int sig, const struct sigaction *act, struct sigaction *oact);
```
