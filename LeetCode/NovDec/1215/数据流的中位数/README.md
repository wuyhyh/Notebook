### **LeetCode 第295题 - 数据流的中位数 (Find Median from Data Stream)**

#### **题目描述**

设计一个支持以下两种操作的数据结构：
1. **`addNum(num: int)`** - 添加一个整数 `num` 到数据流中。
2. **`findMedian()`** - 返回当前数据流的中位数。

中位数的定义：
- 如果数据集的大小是奇数，中位数是中间的数字。
- 如果数据集的大小是偶数，中位数是两个中间数的平均值。

### **解题思路**

我们可以利用 **最大堆** 和 **最小堆** 来设计一个高效的解决方案。

#### **基本思路：**
- 使用两个堆来保存数据：
    1. **最大堆**（left heap）: 用来存储较小的一半元素。
    2. **最小堆**（right heap）: 用来存储较大的一半元素。

  这两个堆的性质保证了：
    - 最大堆的堆顶元素是当前较小一半中的最大值。
    - 最小堆的堆顶元素是当前较大一半中的最小值。

- **堆的平衡**：
    - 堆的大小差不超过 1（即最多只有一个堆比另一个堆多一个元素）。
    - 如果总元素个数是奇数，`最大堆` 会比 `最小堆` 多一个元素。
    - 如果总元素个数是偶数，`最大堆` 和 `最小堆` 的大小相等。

- **查找中位数**：
    - 如果 `最大堆` 的大小大于 `最小堆`，则中位数是最大堆的堆顶元素。
    - 如果两个堆的大小相等，则中位数是两个堆顶元素的平均值。

### **操作细节：**
1. **`addNum(num)`**：
    - 将新的数字添加到 `最大堆` 或 `最小堆` 中。
    - 根据堆的大小情况，可能需要调整堆的平衡。

2. **`findMedian()`**：
    - 如果 `最大堆` 中元素多，返回 `最大堆` 的堆顶元素。
    - 如果两个堆大小相等，返回两个堆顶元素的平均值。

### **代码实现**

```python
import heapq

class MedianFinder:
    def __init__(self):
        # 最大堆：存储较小的一半元素 (使用负数模拟最大堆)
        self.left = []  # 最大堆
        # 最小堆：存储较大的一半元素
        self.right = []  # 最小堆

    def addNum(self, num: int) -> None:
        # 总是将新元素添加到最大堆（left）
        heapq.heappush(self.left, -num)
        
        # 将最大堆的最大元素（注意是负数）移到最小堆（right）
        heapq.heappush(self.right, -heapq.heappop(self.left))
        
        # 保证右边堆的大小要么等于左边堆，要么比左边堆多一个元素
        if len(self.left) < len(self.right):
            heapq.heappush(self.left, -heapq.heappop(self.right))

    def findMedian(self) -> float:
        # 如果左堆多一个元素，则中位数就是左堆的堆顶元素
        if len(self.left) > len(self.right):
            return -self.left[0]
        # 否则，中位数是左右堆的堆顶元素的平均值
        return (-self.left[0] + self.right[0]) / 2.0
```

### **代码解析**

1. **`__init__` 方法**：
    - 我们初始化了两个堆：`left` 用来存储较小的一半元素（最大堆），`right` 用来存储较大的一半元素（最小堆）。
    - 由于 Python 的 `heapq` 库实现的是最小堆，因此我们通过将元素取负值来模拟最大堆。

2. **`addNum(num)` 方法**：
    - 每次添加数字时，先将数字添加到 `最大堆`（`left`），并将 `最大堆` 的最大元素移到 `最小堆`（`right`）。
    - 如果 `右堆` 比 `左堆` 大，则将 `右堆` 的堆顶元素移回 `左堆`，确保两个堆的大小平衡。

3. **`findMedian()` 方法**：
    - 如果 `左堆` 大于 `右堆`，说明当前中位数是 `左堆` 的堆顶元素。
    - 如果两个堆的大小相等，中位数是两个堆顶元素的平均值。

### **时间复杂度**

- **`addNum(num)`**：每次添加一个数字的时间复杂度是 `O(log n)`，因为每次向堆中插入或移除元素的时间复杂度是 `O(log n)`。
- **`findMedian()`**：返回中位数的时间复杂度是 `O(1)`，只需要访问堆顶元素。

因此，整体时间复杂度：
- `addNum(num)` 的时间复杂度是 `O(log n)`。
- `findMedian()` 的时间复杂度是 `O(1)`。

### **空间复杂度**
- 使用两个堆来存储数据，因此空间复杂度是 `O(n)`，其中 `n` 是当前数据流中的元素个数。

### **示例**

#### 示例 1：

```python
mf = MedianFinder()
mf.addNum(1)
print(mf.findMedian())  # 输出 1.0
mf.addNum(2)
print(mf.findMedian())  # 输出 1.5
mf.addNum(3)
print(mf.findMedian())  # 输出 2.0
```

**解释**：
1. 添加 `1` 后，中位数是 `1`。
2. 添加 `2` 后，中位数是 `(1 + 2) / 2 = 1.5`。
3. 添加 `3` 后，中位数是 `2`。

#### 示例 2：

```python
mf = MedianFinder()
mf.addNum(3)
mf.addNum(1)
mf.addNum(4)
print(mf.findMedian())  # 输出 3.0
```

**解释**：
1. 添加 `3` 后，中位数是 `3`。
2. 添加 `1` 后，中位数是 `(3 + 1) / 2 = 2`。
3. 添加 `4` 后，中位数是 `3`。

### **总结**

- 利用最大堆和最小堆来维护数据流的前后部分，使得两部分的元素个数差不超过 1，从而能够高效地找到中位数。
- `addNum` 操作的时间复杂度是 `O(log n)`，`findMedian` 操作的时间复杂度是 `O(1)`，适合需要频繁计算中位数的场景。
