### 动态规划的核心思想

动态规划本质上是一种“分治策略”，其核心思想是将一个复杂的问题分解成若干个子问题，并且每个子问题只求解一次，存储这些结果（通常称为“记忆化”或“备忘录”），避免重复计算，从而提高效率。

### 动态规划的基本步骤

1. **定义状态**：
    - 状态是指问题的中间结果。我们需要将原问题转换成一个可以通过子问题的解得到的形式。

2. **状态转移方程**：
    - 状态转移方程表示如何从已知的子问题解推导出当前问题的解。我们需要写出一个递推公式，用来描述当前的解是如何依赖于之前子问题的解。

3. **边界条件**：
    - 这些是最基本的情况，即我们在没有任何计算时就可以确定的解。例如，在 **最大子数组和** 问题中，子数组只有一个元素时，最大和就是该元素本身。

4. **优化空间**：
    - 通过分析状态的依赖关系，我们可能会发现可以使用常数空间来保存所有需要的信息，从而优化空间复杂度。

### 动态规划解法：最大子数组和

让我们通过 **最大子数组和** 问题（LeetCode 第 53 题）来一步步理解动态规划是如何工作的。

#### 问题回顾

给定一个整数数组 `nums`，我们需要找出 **连续** 子数组，使得其元素和最大，并返回该最大和。

#### 解题思路

1. **定义状态**：
    - 我们定义 `dp[i]` 为以 `nums[i]` 结尾的子数组的最大和。
    - 通过这个定义，我们的目标是计算 `dp[i]`，从而找出所有可能的 `dp[i]` 中的最大值。

2. **状态转移方程**：
    - `dp[i]` 可以通过两个选择来计算：
        1. 继续加上 `nums[i]`，即 `dp[i-1] + nums[i]`。
        2. 从 `nums[i]` 开始一个新的子数组，即 `nums[i]`。
    - 因此，`dp[i] = max(dp[i-1] + nums[i], nums[i])`。

   这个状态转移方程的意思是：如果前面的子数组和已经为负数，那么加上 `nums[i]` 会让和变得更小，此时应该从 `nums[i]` 开始一个新的子数组；否则，继续扩展当前子数组。

3. **边界条件**：
    - 初始时，`dp[0] = nums[0]`，因为只有一个元素时，最大子数组和就是这个元素。

4. **最终结果**：
    - 我们最终需要找到所有 `dp[i]` 中的最大值，这个值就是最大子数组和。

---

### 代码实现

```python
def maxSubArray(nums):
    # 初始化最大和为第一个元素
    max_sum = nums[0]
    # 初始化当前子数组和为第一个元素
    current_sum = nums[0]
    
    # 从第二个元素开始遍历
    for i in range(1, len(nums)):
        # 判断是扩展当前子数组还是从当前元素重新开始子数组
        current_sum = max(current_sum + nums[i], nums[i])
        
        # 更新最大子数组和
        max_sum = max(max_sum, current_sum)
    
    return max_sum
```

#### 代码解释：

1. **初始化**：
    - `max_sum`：用于记录当前的最大子数组和，初始化为第一个元素 `nums[0]`。
    - `current_sum`：表示当前子数组的和，初始化为第一个元素 `nums[0]`。

2. **遍历数组**：
    - 从第二个元素开始，更新 `current_sum`。通过比较 `current_sum + nums[i]`（即继续扩展当前子数组的和）和 `nums[i]`（即从当前元素开始一个新的子数组）来更新 `current_sum`。

3. **更新最大和**：
    - 每次更新 `current_sum` 后，比较并更新 `max_sum`，即保存当前遍历到的最大子数组和。

4. **返回结果**：
    - 最后返回 `max_sum`，它就是最大的子数组和。

### 关键概念解读

1. **为什么要用动态规划**：
    - 最大子数组和问题可以看作是一个典型的 **最优子结构问题**。也就是说，问题的最优解可以通过子问题的最优解来得到。
    - 对于每个 `i`，我们只需要知道 `i-1` 的解（即前一个子数组的和）就可以计算出 `i` 的解。因此，可以通过动态规划的方式逐步解决。

2. **状态转移方程**：
    - 你可以将问题的解拆解成子问题的解：当前子数组的和依赖于前一个子数组的和。
    - 如果前一个子数组的和是负数，那么加上当前元素只会让和变小，这时就从当前元素开始一个新子数组；否则，继续扩展当前子数组。

3. **空间优化**：
    - 原始动态规划方法中，我们通常使用一个数组 `dp[i]` 来保存每个位置的子数组和。这里我们可以用 `current_sum` 来代替 `dp[i]` 数组，从而节省空间。

### 例子分析

以 `nums = [-2, 1, -3, 4, -1, 2, 1, -5, 4]` 为例，逐步分析动态规划的过程：

- 初始时：`max_sum = -2`, `current_sum = -2`
- 第一个元素 `1`：
    - `current_sum = max(-2 + 1, 1) = 1`
    - `max_sum = max(-2, 1) = 1`
- 第二个元素 `-3`：
    - `current_sum = max(1 + (-3), -3) = -2`
    - `max_sum = max(1, -2) = 1`
- 第三个元素 `4`：
    - `current_sum = max(-2 + 4, 4) = 4`
    - `max_sum = max(1, 4) = 4`
- 依此类推，最终得到 `max_sum = 6`。

---

### 总结

动态规划的关键在于 **将问题分解为子问题**，并通过递推关系来逐步求解每个子问题，最终得到整体问题的解。在最大子数组和问题中，我们通过定义一个状态 `current_sum` 来记录当前子数组的和，通过状态转移方程决定是继续扩展当前子数组，还是从当前元素开始新的子数组。

通过这种方式，动态规划将问题的求解变得高效，避免了暴力求解时的重复计算。
