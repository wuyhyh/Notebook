# 02-C语言结构体与指针基础

## 1 文档目标

本文用于帮助实习生掌握 C 语言中结构体和指针的基础用法。

学习完本文后，应能够：

1. 看懂基本的结构体定义和使用。
2. 理解指针保存的是地址。
3. 区分 `.` 和 `->` 的用法。
4. 理解结构体指针在驱动和嵌入式代码中的作用。
5. 能够阅读简单的链表、设备对象、寄存器封装代码。

## 2 结构体基础

## 2.1 什么是结构体

结构体用于把多个相关的数据组合成一个整体。

示例：

```c++
struct uart_port {
    unsigned long base;
    unsigned int irq;
    unsigned int baudrate;
};
```

这个结构体描述了一个 UART 端口，其中包含：

1. `base`：寄存器基地址。
2. `irq`：中断号。
3. `baudrate`：波特率。

结构体适合表示一个“对象”或者一个“设备”。

## 2.2 定义结构体变量

定义结构体变量：

```c++
struct uart_port port;
```

给成员赋值：

```c++
port.base = 0x28001000;
port.irq = 32;
port.baudrate = 115200;
```

访问结构体普通变量的成员时，使用 `.`。

## 2.3 初始化结构体

可以在定义时初始化：

```c++
struct uart_port port = {
    .base = 0x28001000,
    .irq = 32,
    .baudrate = 115200,
};
```

这种写法更清晰，不容易搞错成员顺序。

不推荐只依赖顺序初始化：

```c++
struct uart_port port = {0x28001000, 32, 115200};
```

因为结构体成员较多时，可读性较差。

## 2.4 结构体成员类型

结构体成员可以是普通变量：

```c++
struct point {
    int x;
    int y;
};
```

也可以是数组：

```c++
struct device_name {
    char name[32];
};
```

也可以是指针：

```c++
struct buffer {
    char *data;
    unsigned int len;
};
```

也可以嵌套另一个结构体：

```c++
struct device {
    struct uart_port port;
    char name[32];
};
```

## 3 指针基础

## 3.1 什么是指针

指针是保存地址的变量。

示例：

```c++
int value = 10;
int *p = &value;
```

含义：

1. `value` 是一个普通整数变量。
2. `&value` 表示 `value` 的地址。
3. `p` 是一个指针变量，保存 `value` 的地址。
4. `*p` 表示通过指针访问这个地址上的内容。

## 3.2 取地址和解引用

取地址：

```c++
int *p = &value;
```

解引用：

```c++
int x = *p;
```

修改指针指向的内容：

```c++
*p = 20;
```

此时 `value` 的值也会变成 20。

## 3.3 指针变量本身也有地址

指针变量也是变量，所以它自己也有地址。

```c++
int value = 10;
int *p = &value;
int **pp = &p;
```

含义：

1. `p` 保存 `value` 的地址。
2. `pp` 保存 `p` 的地址。
3. `*pp` 得到 `p`。
4. `**pp` 得到 `value`。

初学阶段只需要先掌握一级指针，二级指针后面再深入理解。

## 3.4 空指针

空指针表示当前没有指向有效对象：

```c++
int *p = NULL;
```

使用指针前应确认它不是 `NULL`：

```c++
if (p != NULL) {
    *p = 10;
}
```

在 Linux 内核和嵌入式代码中，访问空指针通常会导致严重错误。

## 4 结构体指针

## 4.1 结构体指针的定义

结构体指针保存结构体变量的地址：

```c++
struct uart_port port;
struct uart_port *p = &port;
```

这里：

1. `port` 是结构体变量。
2. `&port` 是结构体变量的地址。
3. `p` 是结构体指针。

## 4.2 使用结构体指针访问成员

普通结构体变量访问成员：

```c++
port.baudrate = 115200;
```

结构体指针访问成员：

```c++
p->baudrate = 115200;
```

`p->baudrate` 等价于：

```c++
(*p).baudrate
```

因为 `p` 是指针，需要先解引用，再访问成员。

## 4.3 . 和 -> 的区别

`.` 用于结构体变量：

```c++
struct uart_port port;

port.irq = 32;
```

`->` 用于结构体指针：

```c++
struct uart_port *p = &port;

p->irq = 32;
```

记忆方法：

1. 变量用 `.`。
2. 指针用 `->`。

## 4.4 结构体指针作为函数参数

在 C 语言中，函数参数默认是值传递。

如果直接传结构体变量，会复制整个结构体：

```c++
void uart_init(struct uart_port port)
{
    port.baudrate = 115200;
}
```

这个函数修改的是副本，不会修改外部的 `port`。

更常见的写法是传结构体指针：

```c++
void uart_init(struct uart_port *port)
{
    port->baudrate = 115200;
}
```

调用：

```c++
struct uart_port port;

uart_init(&port);
```

这样函数内部可以修改外部结构体。

## 5 指针和数组

## 5.1 数组名和地址

数组定义：

```c++
int arr[4] = {1, 2, 3, 4};
```

数组名 `arr` 在很多表达式中会转换为首元素地址。

```c++
int *p = arr;
```

等价于：

```c++
int *p = &arr[0];
```

## 5.2 通过指针访问数组

可以通过数组下标访问：

```c++
int x = arr[2];
```

也可以通过指针访问：

```c++
int x = *(p + 2);
```

这两种写法访问的是同一个元素。

## 5.3 指针加法

指针加 1，不是地址数字简单加 1，而是移动到下一个元素。

```c++
int *p = arr;

p = p + 1;
```

如果 `int` 占 4 字节，那么地址实际增加 4 字节。

所以指针加法和指针类型有关。

## 6 指针和字符串

## 6.1 字符数组

字符串可以用字符数组保存：

```c++
char name[] = "uart0";
```

这个数组内容实际包括结尾的 `\0`：

```text
u a r t 0 \0
```

## 6.2 字符指针

也可以用字符指针指向字符串常量：

```c++
const char *name = "uart0";
```

推荐给字符串常量加上 `const`，表示不应该修改它。

不推荐：

```c++
char *name = "uart0";
name[0] = 'U';
```

这可能导致程序错误。

## 7 结构体和数组结合

## 7.1 结构体数组

可以定义多个设备：

```c++
struct uart_port ports[] = {
    {
        .base = 0x28001000,
        .irq = 32,
        .baudrate = 115200,
    },
    {
        .base = 0x28002000,
        .irq = 33,
        .baudrate = 115200,
    },
};
```

访问第一个设备：

```c++
ports[0].base = 0x28001000;
```

访问第二个设备：

```c++
ports[1].base = 0x28002000;
```

## 7.2 遍历结构体数组

```c++
int i;

for (i = 0; i < 2; i++) {
    uart_init(&ports[i]);
}
```

这里传入的是每个结构体元素的地址。

## 8 嵌入式代码中的典型用法

## 8.1 用结构体描述设备

在驱动代码中，经常用结构体描述设备状态：

```c++
struct led_device {
    unsigned long gpio_base;
    unsigned int gpio_num;
    int active_low;
};
```

这个结构体把一个 LED 设备需要的信息组织在一起。

## 8.2 用结构体指针传递上下文

函数中传入结构体指针：

```c++
int led_set(struct led_device *led, int value)
{
    if (led == NULL) {
        return -1;
    }

    if (led->active_low) {
        value = !value;
    }

    /* write gpio value */

    return 0;
}
```

这样函数不需要依赖全局变量，而是通过 `led` 参数知道要操作哪个设备。

## 8.3 用结构体封装寄存器信息

```c++
struct reg_field {
    unsigned int offset;
    unsigned int mask;
    unsigned int shift;
};
```

示例：

```c++
struct reg_field baud_field = {
    .offset = 0x08,
    .mask = 0xf << 4,
    .shift = 4,
};
```

这种写法可以让寄存器字段描述更加清晰。

## 9 常见错误

## 9.1 使用未初始化指针

错误：

```c++
int *p;

*p = 10;
```

`p` 没有指向有效地址，直接写入会导致错误。

正确：

```c++
int value;
int *p = &value;

*p = 10;
```

## 9.2 混淆 . 和 ->

错误：

```c++
struct uart_port *p;

p.baudrate = 115200;
```

正确：

```c++
p->baudrate = 115200;
```

前提是 `p` 已经指向有效结构体。

## 9.3 返回局部变量地址

错误：

```c++
int *get_value(void)
{
    int value = 10;

    return &value;
}
```

`value` 是局部变量，函数返回后就失效了。

不要返回局部变量地址。

## 9.4 忘记检查空指针

错误：

```c++
void uart_set_baudrate(struct uart_port *port)
{
    port->baudrate = 115200;
}
```

更稳妥：

```c++
int uart_set_baudrate(struct uart_port *port)
{
    if (port == NULL) {
        return -1;
    }

    port->baudrate = 115200;
    return 0;
}
```

## 9.5 结构体复制成本过高

不推荐频繁传递大结构体：

```c++
void handle_device(struct device dev)
{
    /* dev is copied */
}
```

推荐传指针：

```c++
void handle_device(struct device *dev)
{
    /* use dev->member */
}
```

## 10 建议掌握的习惯

## 10.1 结构体使用习惯

建议：

1. 用结构体组织相关数据。
2. 结构体名字要表达含义。
3. 初始化结构体时优先使用 `.member = value`。
4. 只在需要对外暴露时，把结构体定义放进头文件。
5. 内部使用的结构体可以只放在 `.c` 文件中。

## 10.2 指针使用习惯

建议：

1. 指针定义时尽量初始化。
2. 不确定时先赋值为 `NULL`。
3. 使用指针前检查是否为 `NULL`。
4. 不要返回局部变量地址。
5. 结构体较大时，函数参数优先传指针。
6. 能用 `const` 表示只读时，就使用 `const`。

示例：

```c++
int uart_print_info(const struct uart_port *port)
{
    if (port == NULL) {
        return -1;
    }

    /* only read port */

    return 0;
}
```

`const struct uart_port *port` 表示函数不会修改 `port` 指向的结构体内容。

## 11 简单练习

## 11.1 结构体练习

定义一个 LED 设备结构体：

```c++
struct led_device {
    unsigned long gpio_base;
    unsigned int gpio_num;
    int active_low;
};
```

创建一个变量并初始化：

```c++
struct led_device led0 = {
    .gpio_base = 0x30000000,
    .gpio_num = 5,
    .active_low = 1,
};
```

然后访问它的成员：

```c++
led0.gpio_num = 6;
```

## 11.2 结构体指针练习

写一个函数：

```c++
int led_init(struct led_device *led)
{
    if (led == NULL) {
        return -1;
    }

    /* init led */

    return 0;
}
```

调用：

```c++
led_init(&led0);
```

要求：

1. 函数参数使用结构体指针。
2. 函数内部检查 `NULL`。
3. 使用 `->` 访问结构体成员。

## 11.3 数组和结构体练习

定义两个 LED：

```c++
struct led_device leds[] = {
    {
        .gpio_base = 0x30000000,
        .gpio_num = 5,
        .active_low = 1,
    },
    {
        .gpio_base = 0x30000000,
        .gpio_num = 6,
        .active_low = 0,
    },
};
```

遍历初始化：

```c++
int i;

for (i = 0; i < 2; i++) {
    led_init(&leds[i]);
}
```

## 12 小结

结构体和指针是 C 语言中最重要的基础内容之一。

需要重点记住：

1. 结构体用于组织多个相关数据。
2. 指针保存的是地址。
3. `&` 用于取地址。
4. `*` 用于通过地址访问内容。
5. 普通结构体变量用 `.` 访问成员。
6. 结构体指针用 `->` 访问成员。
7. 结构体作为函数参数时，常用指针传递。
8. 使用指针前要确认它指向有效对象。
9. 不要返回局部变量地址。
10. 在驱动和嵌入式代码中，结构体通常用于描述设备、寄存器、状态和上下文。

掌握结构体和指针后，才能真正开始阅读 Linux 驱动、U-Boot、BSP 和嵌入式 C 项目代码。
