LeetCode 第 41 题：**缺失的第一个正数（First Missing Positive）** 是一个经典的数组问题。该题要求在 `O(n)` 的时间复杂度内找到数组中缺失的第一个正整数。

### 问题描述

给定一个未排序的整数数组 `nums`，找到其中没有出现的最小的正整数。

**示例 1:**
```plaintext
输入: [1, 2, 0]
输出: 3
```

**示例 2:**
```plaintext
输入: [3, 4, -1, 1]
输出: 2
```

**示例 3:**
```plaintext
输入: [7, 8, 9, 11, 12]
输出: 1
```

### 题目分析

我们需要在 `O(n)` 的时间复杂度内找到数组中缺失的第一个正整数，这意味着不能直接使用排序或暴力破解的方法。考虑到以下几点：

- **正整数**：我们只关心数组中的正整数部分（即大于 0 的整数）。
- **范围**：最小缺失正整数一定是在 `[1, n+1]` 范围内，其中 `n` 是数组的长度。因为如果数组中包含 1 到 n 的所有整数，那么缺失的第一个正整数一定是 `n+1`。
- **空间限制**：要求空间复杂度为 `O(1)`，即不能使用额外的数组或哈希表。

### 解题思路

1. **原地哈希**：
    - 我们可以通过将每个数放到它应该在的位置上来解决问题。我们会将每个正整数 `x` 放到索引 `x-1` 的位置。
    - 例如，如果数组中存在数字 `3`，则我们尝试将 `3` 放到 `nums[2]`（即 `x-1`）的位置。
    - 这样一来，我们只需要遍历数组两次：一次用于将元素移到正确的位置，另一次检查哪个位置的数字不符合条件。

2. **步骤**：
    1. 将所有不在 `[1, n]` 范围内的数字置为 `n+1`，因为这些数字无关紧要。
    2. 遍历数组，将每个数字 `x`（满足 `1 <= x <= n`）放到它应该在的位置，即 `nums[x-1]`。
    3. 再次遍历数组，找到第一个 `nums[i] != i+1` 的位置，返回 `i+1` 即为缺失的第一个正整数。如果所有位置都符合条件，返回 `n+1`。

### 代码实现

```python
def firstMissingPositive(nums):
    n = len(nums)
    
    # 第一步：将不在 [1, n] 范围内的数字置为 n+1
    for i in range(n):
        if nums[i] <= 0 or nums[i] > n:
            nums[i] = n + 1
    
    # 第二步：将每个数字放到它应该在的位置
    for i in range(n):
        val = abs(nums[i])
        if 1 <= val <= n:
            if nums[val - 1] > 0:  # 防止重复标记
                nums[val - 1] = -abs(nums[val - 1])  # 将该位置的数字标记为负数

    # 第三步：找到第一个位置 i，满足 nums[i] > 0，返回 i + 1
    for i in range(n):
        if nums[i] > 0:
            return i + 1
    
    # 如果所有位置都已经符合条件，返回 n + 1
    return n + 1
```

### 代码解释

1. **将不在 `[1, n]` 范围内的数字置为 `n+1`**：
    - 因为这些数字在寻找缺失的正整数时没有意义，所以我们可以将它们替换成一个大于 `n` 的数（`n+1`），确保它们不会干扰后续操作。

2. **将每个数字放到它应该在的位置**：
    - 对于数组中的每个数字 `val`，如果它在 `[1, n]` 范围内，我们将 `val` 放到索引 `val-1` 的位置。
    - 为了避免覆盖已处理的数字，我们使用绝对值（`abs(nums[i])`）来查找该数字，并将 `nums[val-1]` 标记为负数，表示数字 `val` 已经出现在了正确的位置。

3. **找到第一个 `nums[i] > 0` 的位置**：
    - 遍历数组，找到第一个 `nums[i] > 0` 的位置，返回 `i+1`，这就是缺失的第一个正整数。
    - 如果所有位置的数字都已正确放置，则缺失的正整数是 `n+1`。

### 示例分析

#### 示例 1：

输入：`nums = [1, 2, 0]`

1. **初始化**：`n = 3`。
2. **第一步**：所有非正数和大于 3 的数字被设置为 `4`：`[1, 2, 4]`。
3. **第二步**：数字 `1` 放到 `nums[0]`，数字 `2` 放到 `nums[1]`，数字 `4` 被跳过。结果为：`[1, 2, 4]`。
4. **第三步**：`nums[2]` 不是正数，因此缺失的第一个正整数是 `3`。

输出：`3`

#### 示例 2：

输入：`nums = [3, 4, -1, 1]`

1. **初始化**：`n = 4`。
2. **第一步**：所有非正数和大于 4 的数字被设置为 `5`：`[3, 4, 5, 1]`。
3. **第二步**：数字 `3` 放到 `nums[2]`，数字 `4` 放到 `nums[3]`，数字 `1` 放到 `nums[0]`。结果为：`[1, 4, 3, -1]`。
4. **第三步**：第一个正数是 `2`，因此缺失的第一个正整数是 `2`。

输出：`2`

#### 示例 3：

输入：`nums = [7, 8, 9, 11, 12]`

1. **初始化**：`n = 5`。
2. **第一步**：所有非正数和大于 5 的数字被设置为 `6`：`[6, 6, 6, 6, 6]`。
3. **第二步**：没有符合条件的数字，最终的结果是 `1`。

输出：`1`

### 时间复杂度与空间复杂度

- **时间复杂度**：`O(n)`，我们只进行了三次遍历。
- **空间复杂度**：`O(1)`，我们只用了原数组，并没有使用额外的空间。

---

### 总结

- 本题要求在 `O(n)` 的时间复杂度和 `O(1)` 的空间复杂度下找到缺失的第一个正整数。
- 使用 **原地哈希** 技术，我们通过将数字放到它们应该在的位置来实现这一目标。
- 这种方法的核心思想是使用数组索引来模拟哈希表，并通过标记位置来判断哪些数字已经出现。
