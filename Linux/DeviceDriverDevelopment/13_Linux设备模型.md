## 一、LDM 你必须知道的核心知识

### 1. 什么是 LDM？

> LDM（Linux Device Model）是 Linux 内核用于抽象和管理各种硬件资源的统一模型，它将总线、设备、驱动、类、属性等统一组织起来，提供结构化管理与自动化绑定机制。

它的核心思想是：**“设备 - 总线 - 驱动”三元模型 + 层次化的对象管理机制（kobject/kset）**。

---

### 2. 核心概念

| 概念               | 含义                                                        |
|------------------|-----------------------------------------------------------|
| **device**       | 表示一个硬件设备，用 `struct device` 表示。                            |
| **driver**       | 表示驱动程序，用 `struct device_driver` 表示。                       |
| **bus**          | 表示设备和驱动的匹配规则平台，用 `struct bus_type` 表示，如 platform、PCI、I2C。 |
| **class**        | 表示设备的功能类型，用于 `/sys/class/` 显示（如 input、net、tty）。           |
| **kobject/kset** | 内核对象模型的基础，用于支撑 sysfs 中的层次结构。                              |
| **udev**         | 用户空间守护进程，通过 LDM 暴露的信息创建 `/dev` 设备节点。                      |

---

### 3. 匹配过程（总线匹配驱动和设备）

- 每个总线定义自己的 `.match()` 函数；
- 当设备注册到内核时，内核会遍历该总线下的驱动列表，调用 match 比对；
- 匹配成功后，调用驱动的 `probe()` 函数。

---

### 4. sysfs 文件系统

LDM 将内核中的设备模型通过 sysfs 导出到用户空间：

- `/sys/bus/`：系统中的所有总线类型；
- `/sys/class/`：逻辑功能分类；
- `/sys/devices/`：实际的物理设备树结构；
- `/sys/block/`：块设备。

这使得用户空间可以通过 `udev` 动态创建设备节点。

## 二、面试常见问题清单（含参考答案要点）

| 面试问题                      | 你应该掌握的答题要点                                                              |
|---------------------------|-------------------------------------------------------------------------|
| 什么是 Linux Device Model？   | 内核统一的设备/驱动/总线管理机制，基于 kobject，支持自动匹配绑定。                                  |
| LDM 管理了哪几个核心结构体？          | `device`、`driver`、`bus_type`、`class`、`kobject`。                         |
| 总线在 LDM 中的作用是什么？          | 连接设备与驱动，定义匹配规则，负责调度 `probe()`。                                          |
| `struct device` 中有哪些重要成员？ | `struct device_driver *driver`、`struct bus_type *bus`、`kobject kobj` 等。 |
| sysfs 是如何与 LDM 结合的？       | 每个 `kobject` 都会对应 sysfs 中一个目录，属性文件由 `attribute` 系列导出。                   |
| class 和 device 的区别？       | `device` 是物理硬件，`class` 是逻辑功能分类，如 `tty`、`net`、`input`。                   |
| 驱动是如何和设备匹配的？              | 总线调用 `.match()` 比较设备和驱动中的名字/ID，匹配后执行 `.probe()`。                        |
| udev 是如何知道何时创建设备节点的？      | 监听 `uevent` 消息，内核中 device_register 会发送热插拔事件。                            |

## 三、建议掌握的函数 / 结构体

| 名称                                          | 说明                   |
|---------------------------------------------|----------------------|
| `device_register()` / `device_unregister()` | 注册 / 注销设备            |
| `driver_register()` / `driver_unregister()` | 注册 / 注销驱动            |
| `bus_register()`                            | 注册总线                 |
| `class_create()` / `device_create()`        | 用于自动在 `/dev` 下创建设备节点 |
| `kobject_init()` / `kobject_add()`          | 初始化内核对象              |
