# 02-C语言位运算与寄存器操作基础

## 1 学习目标

掌握 C 语言中的位运算基本用法，并理解它们为什么常用于寄存器配置、状态位判断和驱动开发。

学完后应能做到：

1. 看懂 `&`、`|`、`^`、`~`、`<<`、`>>` 的含义。
2. 能使用掩码设置、清除、翻转、读取某一位或某几位。
3. 能理解寄存器读改写操作的基本模式。
4. 能避免常见的位运算和寄存器操作错误。

## 2 位运算基础

## 2.1 位运算的对象

位运算直接操作整数的二进制位。嵌入式和驱动开发中，硬件寄存器通常由多个 bit 组成，每个 bit 或 bit field 表示一种控制或状态。

例如一个 8 bit 寄存器：

```text
bit:  7 6 5 4 3 2 1 0
val:  0 0 1 0 1 0 0 1
```

可以理解为：

```c
0b00101001
```

其中某一位可能表示：

```text
bit0: 使能位
bit3: 中断标志位
bit5: 工作模式位
```

## 2.2 常用位运算符

C 语言常用位运算符如下：

```c
&   按位与
|   按位或
^   按位异或
~   按位取反
<<  左移
>>  右移
```

为了避免 Markdown 表格和 `|` 字符冲突，本文不使用表格描述位运算符。

## 2.3 按位与 `&`

按位与常用于清零某些位，或者判断某些位是否为 1。

规则：两个 bit 都是 1，结果才是 1。

```c
0b1010 & 0b1100 = 0b1000
```

常见用途：判断 bit3 是否为 1。

```c
if (value & (1U << 3)) {
    /* bit3 is set */
}
```

## 2.4 按位或 `|`

按位或常用于设置某些位为 1。

规则：两个 bit 只要有一个是 1，结果就是 1。

```c
0b1010 | 0b0100 = 0b1110
```

常见用途：设置 bit3。

```c
value |= (1U << 3);
```

## 2.5 按位异或 `^`

按位异或常用于翻转某些位。

规则：两个 bit 不同则为 1，相同则为 0。

```c
0b1010 ^ 0b0100 = 0b1110
```

常见用途：翻转 bit3。

```c
value ^= (1U << 3);
```

## 2.6 按位取反 `~`

按位取反会把 0 变成 1，把 1 变成 0。

常见用途：配合 `&` 清除某些位。

```c
value &= ~(1U << 3);
```

这行代码表示：清除 bit3，其他位保持不变。

## 2.7 左移 `<<` 和右移 `>>`

左移常用于构造掩码。

```c
1U << 0    /* bit0 */
1U << 3    /* bit3 */
1U << 7    /* bit7 */
```

右移常用于把某些 bit field 移动到最低位，方便读取。

```c
mode = (value >> 4) & 0x3;
```

这表示从 `value` 中取出 bit5:bit4 两位。

## 3 掩码的概念

## 3.1 什么是掩码

掩码就是用来选择某些 bit 的二进制值。

例如：

```c
#define BIT0    (1U << 0)
#define BIT3    (1U << 3)
#define BIT7    (1U << 7)
```

如果要同时选择 bit3 和 bit7，可以写成：

```c
#define MASK    ((1U << 3) | (1U << 7))
```

## 3.2 设置某一位

```c
value |= (1U << 3);
```

含义：将 bit3 设置为 1，其他位保持不变。

## 3.3 清除某一位

```c
value &= ~(1U << 3);
```

含义：将 bit3 清零，其他位保持不变。

## 3.4 翻转某一位

```c
value ^= (1U << 3);
```

含义：如果 bit3 原来是 1，就变成 0；如果原来是 0，就变成 1。

## 3.5 判断某一位

```c
if (value & (1U << 3)) {
    /* bit3 is 1 */
} else {
    /* bit3 is 0 */
}
```

注意：判断结果不一定等于 1，只要非 0 就表示该位为 1。

## 4 bit field 操作

## 4.1 什么是 bit field

有些寄存器不是单独用一个 bit 表示功能，而是连续使用多个 bit 表示一个字段。

例如：

```text
bit5:bit4 表示模式选择：
00: mode0
01: mode1
10: mode2
11: mode3
```

## 4.2 读取 bit field

假设 bit5:bit4 是模式字段：

```c
#define MODE_SHIFT    4
#define MODE_MASK     (0x3U << MODE_SHIFT)

mode = (value & MODE_MASK) >> MODE_SHIFT;
```

含义：

1. 使用 `MODE_MASK` 选出 bit5:bit4。
2. 右移到最低位。
3. 得到字段的实际数值。

## 4.3 写入 bit field

写入 bit field 的标准步骤是：先清除旧字段，再写入新字段。

```c
value &= ~MODE_MASK;
value |= (mode << MODE_SHIFT) & MODE_MASK;
```

更完整的写法：

```c
value = (value & ~MODE_MASK) | ((mode << MODE_SHIFT) & MODE_MASK);
```

这样可以避免破坏寄存器中的其他位。

## 5 寄存器操作基础

## 5.1 寄存器是什么

在驱动开发中，寄存器通常是硬件模块提供的一组地址。CPU 通过读写这些地址来控制硬件。

例如：

```text
0x10000000: 控制寄存器
0x10000004: 状态寄存器
0x10000008: 数据寄存器
```

这些地址不是普通内存，而是硬件设备映射出来的 MMIO 地址。

## 5.2 为什么寄存器操作需要位运算

一个寄存器通常同时包含多个控制位或状态位。

例如一个控制寄存器：

```text
bit0: enable
bit1: reset
bit2: interrupt_enable
bit5: loopback_mode
```

如果只想打开 `enable`，不能直接把整个寄存器写成 `1`，因为这样可能会把其他位全部清零。

错误示例：

```c
reg = 1;
```

更安全的做法是只修改目标 bit：

```c
reg |= (1U << 0);
```

## 5.3 volatile 的作用

访问硬件寄存器时，通常需要使用 `volatile`。

```c
volatile unsigned int *reg = (volatile unsigned int *)0x10000000;
```

`volatile` 告诉编译器：这个地址的值可能被硬件改变，不要随意优化读写操作。

没有 `volatile` 时，编译器可能会缓存寄存器值，导致程序没有真正读取硬件当前状态。

## 5.4 基本读写寄存器

裸机或简单示例中可以这样理解：

```c
#define REG_BASE    0x10000000U
#define REG_CTRL    (*(volatile unsigned int *)(REG_BASE + 0x00))
#define REG_STAT    (*(volatile unsigned int *)(REG_BASE + 0x04))

REG_CTRL |= (1U << 0);

if (REG_STAT & (1U << 3)) {
    /* status bit3 is set */
}
```

Linux 内核驱动中一般不建议直接解引用物理地址，而是使用 `ioremap()`、`readl()`、`writel()` 等接口。

## 5.5 读改写操作

读改写是寄存器操作中最常见的模式。

```c
val = readl(base + REG_CTRL);
val |= ENABLE_BIT;
writel(val, base + REG_CTRL);
```

含义：

1. 先读取当前寄存器值。
2. 修改目标 bit。
3. 再写回寄存器。

这样可以尽量避免影响其他 bit。

## 6 常见寄存器操作模板

## 6.1 设置 enable 位

```c
#define CTRL_ENABLE    (1U << 0)

val = readl(base + REG_CTRL);
val |= CTRL_ENABLE;
writel(val, base + REG_CTRL);
```

## 6.2 清除 reset 位

```c
#define CTRL_RESET    (1U << 1)

val = readl(base + REG_CTRL);
val &= ~CTRL_RESET;
writel(val, base + REG_CTRL);
```

## 6.3 设置模式字段

```c
#define CTRL_MODE_SHIFT    4
#define CTRL_MODE_MASK     (0x3U << CTRL_MODE_SHIFT)

val = readl(base + REG_CTRL);
val &= ~CTRL_MODE_MASK;
val |= (mode << CTRL_MODE_SHIFT) & CTRL_MODE_MASK;
writel(val, base + REG_CTRL);
```

## 6.4 等待状态位变为 1

```c
#define STAT_READY    (1U << 0)

while (!(readl(base + REG_STAT) & STAT_READY)) {
    /* wait */
}
```

实际驱动中不建议无限等待，应该增加超时机制。

```c
int timeout = 100000;

while (!(readl(base + REG_STAT) & STAT_READY)) {
    if (--timeout == 0)
        return -1;
}
```

## 7 常见错误

## 7.1 直接覆盖整个寄存器

错误：

```c
writel(CTRL_ENABLE, base + REG_CTRL);
```

问题：可能会清除其他已经配置好的 bit。

更好的写法：

```c
val = readl(base + REG_CTRL);
val |= CTRL_ENABLE;
writel(val, base + REG_CTRL);
```

## 7.2 忘记使用无符号常量

不推荐：

```c
1 << 31
```

更推荐：

```c
1U << 31
```

原因：有符号整数左移到符号位可能带来未定义或不符合预期的行为。

## 7.3 掩码没有加括号

不推荐：

```c
#define BIT3 1U << 3
```

更推荐：

```c
#define BIT3 (1U << 3)
```

宏定义中要尽量加括号，避免运算优先级问题。

## 7.4 没有保护共享寄存器

如果一个寄存器可能被多个执行上下文访问，例如中断上下文和普通线程同时访问，简单读改写可能产生竞争。

例如：

```text
线程 A 读取寄存器
中断 B 修改寄存器
线程 A 写回旧值
```

这样可能会覆盖中断 B 的修改。

实际驱动中需要根据场景使用锁、原子操作，或者硬件提供的 set/clear 寄存器。

## 8 建议的编码习惯

## 8.1 使用宏定义 bit 和 mask

```c
#define CTRL_ENABLE        (1U << 0)
#define CTRL_RESET         (1U << 1)
#define CTRL_MODE_SHIFT    4
#define CTRL_MODE_MASK     (0x3U << CTRL_MODE_SHIFT)
```

不要在代码中到处写裸数字。

不推荐：

```c
val |= (1U << 13);
```

推荐：

```c
val |= CTRL_TX_ENABLE;
```

## 8.2 保持读改写结构清晰

推荐写法：

```c
val = readl(base + REG_CTRL);
val &= ~CTRL_MODE_MASK;
val |= (mode << CTRL_MODE_SHIFT) & CTRL_MODE_MASK;
writel(val, base + REG_CTRL);
```

不要为了少写几行，把表达式写得过于复杂。

## 8.3 注意寄存器手册说明

不同寄存器的行为可能不同。常见特殊行为包括：

1. 写 1 清除：W1C，write 1 to clear。
2. 只读状态位：软件不能写。
3. 写入后自动清零。
4. 保留位必须保持原值。
5. 某些 bit 必须按固定顺序配置。

所以寄存器操作不能只看 C 代码，还必须结合硬件手册。

## 9 练习

## 9.1 基础练习

给定：

```c
unsigned int value = 0x00000000;
```

完成以下操作：

1. 设置 bit0。
2. 设置 bit3。
3. 清除 bit0。
4. 判断 bit3 是否为 1。
5. 翻转 bit3。

参考代码：

```c
value |= (1U << 0);
value |= (1U << 3);
value &= ~(1U << 0);

if (value & (1U << 3)) {
    /* bit3 is set */
}

value ^= (1U << 3);
```

## 9.2 bit field 练习

假设 bit5:bit4 表示 mode，请将 mode 设置为 2。

参考代码：

```c
#define MODE_SHIFT    4
#define MODE_MASK     (0x3U << MODE_SHIFT)

unsigned int value = 0;
unsigned int mode = 2;

value = (value & ~MODE_MASK) | ((mode << MODE_SHIFT) & MODE_MASK);
```

## 10 小结

位运算是寄存器操作的基础。驱动开发中最常见的模式不是“把寄存器写成某个固定值”，而是：

```text
读取当前值 -> 修改目标位 -> 写回寄存器
```

需要重点掌握：

1. `|` 用于设置 bit。
2. `& ~` 用于清除 bit。
3. `&` 用于判断 bit。
4. `^` 用于翻转 bit。
5. `mask + shift` 用于操作 bit field。
6. 寄存器操作必须结合硬件手册，特别注意保留位、只读位和 W1C 位。
