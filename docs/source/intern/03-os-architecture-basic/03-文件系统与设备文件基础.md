# 03-文件系统与设备文件基础

## 1 为什么要学习文件系统和设备文件

在 Linux 中，很多东西都可以通过“文件”的方式访问。

例如：

```text
普通文本文件
目录
磁盘分区
串口设备
GPIO 设备
输入设备
块设备
字符设备
procfs/sysfs 虚拟文件
```

Linux 有一个重要思想：

```text
一切皆文件
```

这句话不是说所有东西真的都是磁盘上的普通文件，而是说 Linux 尽量用统一的文件接口来访问不同类型的对象。

例如用户程序可以用类似方式访问普通文件和设备文件：

```c
open()
read()
write()
ioctl()
close()
```

理解文件系统和设备文件，是学习 Linux 使用、系统编程和驱动开发的基础。

## 2 文件系统是什么

文件系统是操作系统用来组织、存储和管理文件的一套机制。

它负责解决这些问题：

```text
文件如何命名
目录如何组织
文件数据存放在哪里
文件权限如何管理
文件大小和时间信息如何记录
磁盘空间如何分配和回收
```

常见文件系统包括：

```text
ext4
xfs
btrfs
vfat
ntfs
tmpfs
procfs
sysfs
devtmpfs
```

其中，`ext4`、`xfs` 这类文件系统通常用于真实磁盘分区。

`procfs`、`sysfs`、`devtmpfs` 这类文件系统更多是内核提供的虚拟文件系统。

## 3 Linux 目录树

Linux 使用统一的目录树。

常见目录包括：

| 目录 | 作用 |
|---|---|
| / | 根目录 |
| /bin | 基本命令 |
| /sbin | 系统管理命令 |
| /etc | 配置文件 |
| /home | 普通用户目录 |
| /root | root 用户目录 |
| /lib、/lib64 | 库文件 |
| /usr | 用户态程序和资源 |
| /var | 可变数据，例如日志 |
| /tmp | 临时文件 |
| /dev | 设备文件 |
| /proc | 进程和内核信息 |
| /sys | 设备、驱动和内核对象信息 |
| /mnt | 临时挂载点 |
| /boot | 内核、initramfs、设备树等启动文件 |

简单理解：

```text
Linux 把所有文件系统都挂到同一棵目录树上
```

例如根文件系统挂载到 `/`，其他分区或虚拟文件系统再挂载到不同目录下面。

## 4 挂载是什么

挂载就是把一个文件系统接入到 Linux 目录树中的某个位置。

例如：

```bash
mount /dev/nvme0n1p1 /mnt
```

含义是：

```text
把 /dev/nvme0n1p1 这个分区上的文件系统挂载到 /mnt 目录
```

挂载后，访问 `/mnt`，就是在访问这个分区里的内容。

查看当前挂载情况：

```bash
mount
```

或者：

```bash
findmnt
```

查看磁盘和挂载点：

```bash
lsblk
```

卸载文件系统：

```bash
umount /mnt
```

注意命令是 `umount`，不是 `unmount`。

## 5 普通文件、目录和特殊文件

Linux 文件不只有普通文件和目录。

可以使用 `ls -l` 查看文件类型：

```bash
ls -l
```

输出第一列的第一个字符表示文件类型：

| 标记 | 类型 |
|---|---|
| - | 普通文件 |
| d | 目录 |
| l | 符号链接 |
| c | 字符设备 |
| b | 块设备 |
| p | 管道 |
| s | socket |

例如：

```text
-rw-r--r--  普通文件
drwxr-xr-x  目录
lrwxrwxrwx  符号链接
crw-rw----  字符设备
brw-rw----  块设备
```

设备文件通常位于 `/dev` 目录下。

## 6 什么是设备文件

设备文件是 Linux 为设备提供的一种文件接口。

用户程序可以像操作文件一样操作设备。

例如：

```text
/dev/ttyS0
/dev/ttyAMA0
/dev/input/event0
/dev/nvme0n1
/dev/gpiochip0
/dev/null
/dev/zero
```

设备文件本身通常不保存普通数据。

它更像是一个入口，用户程序通过这个入口进入内核，最终访问对应的设备驱动。

简单理解：

```text
设备文件 = 用户程序访问内核驱动的入口
```

例如：

```c
fd = open("/dev/ttyS0", O_RDWR);
read(fd, buf, len);
write(fd, buf, len);
close(fd);
```

这些操作最终会进入串口驱动。

## 7 字符设备和块设备

Linux 中常见设备文件主要有两类：

```text
字符设备
块设备
```

### 7.1 字符设备

字符设备通常按字节流方式访问。

典型例子：

```text
串口
终端
GPIO chip
input event
/dev/null
/dev/zero
```

字符设备的特点：

```text
通常顺序读写
不一定支持随机访问
常用于控制类、流式数据类设备
```

例如：

```bash
ls -l /dev/ttyAMA0
```

可能看到：

```text
crw-rw---- 1 root dialout ...
```

开头的 `c` 表示字符设备。

### 7.2 块设备

块设备通常以固定大小的数据块为单位访问。

典型例子：

```text
硬盘
SSD
NVMe
SD 卡
U 盘
磁盘分区
```

块设备的特点：

```text
支持随机访问
通常可以建立文件系统
常用于存储设备
```

例如：

```bash
ls -l /dev/nvme0n1
ls -l /dev/nvme0n1p1
```

可能看到开头是：

```text
b
```

表示块设备。

## 8 主设备号和次设备号

设备文件有主设备号和次设备号。

使用 `ls -l` 查看设备文件时，可以看到两个数字。

例如：

```bash
ls -l /dev/null
```

可能看到：

```text
crw-rw-rw- 1 root root 1, 3 /dev/null
```

其中：

```text
1：主设备号
3：次设备号
```

简单理解：

```text
主设备号：通常用于找到对应的驱动
次设备号：用于区分同一驱动管理的不同设备
```

例如一个串口驱动可能管理多个串口：

```text
/dev/ttyS0
/dev/ttyS1
/dev/ttyS2
```

它们可能主设备号相同，次设备号不同。

## 9 设备文件和驱动的关系

用户程序访问设备文件时，大致流程如下：

```text
用户程序
  |
  v
open/read/write/ioctl
  |
  v
VFS
  |
  v
字符设备或块设备框架
  |
  v
具体设备驱动
  |
  v
硬件设备
```

设备文件不是驱动本身。

设备文件只是用户态看到的入口。

真正处理请求的是内核里的驱动程序。

例如用户程序执行：

```c
write(fd, buf, len);
```

如果 `fd` 对应某个字符设备，那么内核最终会调用这个驱动注册的 `write` 回调函数。

## 10 VFS 是什么

VFS 是 Virtual File System，中文通常叫虚拟文件系统。

它是 Linux 内核中统一文件访问接口的一层抽象。

不同文件系统和设备类型差异很大，但用户程序可以使用统一接口：

```c
open()
read()
write()
ioctl()
close()
mmap()
```

VFS 负责把这些统一的系统调用转发给具体实现。

简单理解：

```text
VFS = Linux 文件访问的统一抽象层
```

例如：

```text
读 ext4 普通文件 -> 转到 ext4 文件系统
读 /proc/cpuinfo -> 转到 procfs
读 /dev/ttyS0 -> 转到串口驱动
读 /dev/nvme0n1 -> 转到块设备驱动
```

## 11 文件描述符是什么

用户程序打开文件或设备后，会得到一个文件描述符。

例如：

```c
int fd;

fd = open("/dev/ttyS0", O_RDWR);
```

`fd` 是一个整数。

用户程序后续通过这个整数操作对应对象：

```c
read(fd, buf, len);
write(fd, buf, len);
close(fd);
```

常见标准文件描述符：

| 文件描述符 | 含义 |
|---|---|
| 0 | 标准输入 stdin |
| 1 | 标准输出 stdout |
| 2 | 标准错误 stderr |

例如：

```c
write(1, "hello\n", 6);
```

表示向标准输出写入内容。

## 12 /dev 目录是什么

`/dev` 目录用于存放设备文件。

例如：

```bash
ls /dev
```

常见设备文件：

```text
/dev/null
/dev/zero
/dev/random
/dev/tty
/dev/ttyS0
/dev/ttyAMA0
/dev/input/event0
/dev/nvme0n1
/dev/nvme0n1p1
/dev/gpiochip0
```

很多现代 Linux 系统中，`/dev` 不是普通磁盘目录，而是由 `devtmpfs` 和 `udev` 动态管理。

也就是说，设备出现时，系统自动创建设备节点；设备移除时，设备节点可能自动消失。

## 13 devtmpfs 和 udev

`devtmpfs` 是内核提供的设备文件系统。

它可以在 `/dev` 下自动创建设备节点。

`udev` 是用户态设备管理机制。

它可以根据设备事件执行规则，例如：

```text
创建设备符号链接
修改设备权限
修改设备所属用户或组
执行设备初始化脚本
给设备起更友好的名字
```

简单理解：

```text
devtmpfs：内核自动创建设备节点
udev：用户态进一步管理设备节点和权限规则
```

查看 udev 设备信息可以使用：

```bash
udevadm info /dev/ttyAMA0
```

监听设备事件：

```bash
udevadm monitor
```

## 14 /proc 文件系统

`/proc` 是 procfs，主要用于暴露进程和内核运行信息。

例如：

```text
/proc/cpuinfo
/proc/meminfo
/proc/interrupts
/proc/iomem
/proc/cmdline
/proc/mounts
/proc/<PID>/
```

常用命令：

```bash
cat /proc/cpuinfo
cat /proc/meminfo
cat /proc/interrupts
cat /proc/iomem
cat /proc/cmdline
```

查看某个进程信息：

```bash
ls /proc/<PID>
cat /proc/<PID>/maps
cat /proc/<PID>/status
```

注意：

```text
/proc 中很多文件不是真实磁盘文件
它们是内核动态生成的信息接口
```

## 15 /sys 文件系统

`/sys` 是 sysfs，主要用于描述内核设备模型。

它里面可以看到设备、驱动、总线、类别等信息。

常见路径：

```text
/sys/bus
/sys/class
/sys/devices
/sys/module
/sys/kernel
```

例如查看平台设备：

```bash
ls /sys/bus/platform/devices/
```

查看 PCI 设备：

```bash
ls /sys/bus/pci/devices/
```

查看网络设备：

```bash
ls /sys/class/net/
```

查看块设备：

```bash
ls /sys/class/block/
```

简单理解：

```text
/proc 偏运行状态信息
/sys 偏设备模型和驱动信息
```

## 16 设备文件和 sysfs 的区别

`/dev` 和 `/sys` 都和设备有关，但作用不同。

| 路径 | 主要作用 |
|---|---|
| /dev | 给用户程序访问设备 |
| /sys | 展示设备、驱动、总线和属性信息 |

例如：

```text
/dev/ttyAMA0
```

用于用户程序打开串口。

而：

```text
/sys/class/tty/ttyAMA0
```

用于查看这个 tty 设备在内核设备模型中的信息。

简单理解：

```text
/dev 是操作入口
/sys 是信息和属性入口
```

## 17 设备文件是如何创建的

设备文件可以自动创建，也可以手动创建。

现代系统通常由 `devtmpfs` 和 `udev` 自动创建。

手动创建可以使用 `mknod`。

例如：

```bash
mknod /dev/mydev c 240 0
```

含义：

```text
创建字符设备 /dev/mydev
主设备号 240
次设备号 0
```

格式：

```bash
mknod <路径> <类型> <主设备号> <次设备号>
```

其中类型：

```text
c：字符设备
b：块设备
```

不过实际工作中，不建议随便手动创建设备节点。更常见的是让驱动、devtmpfs、udev 自动处理。

## 18 ioctl 和设备控制

普通文件操作主要是：

```text
read
write
```

但很多设备需要特殊控制命令。

例如：

```text
设置串口波特率
配置 GPIO 方向
读取设备状态
设置视频采集格式
向驱动下发私有控制命令
```

这些操作经常通过 `ioctl()` 完成。

简单理解：

```text
read/write：偏数据传输
ioctl：偏控制命令
```

用户程序示例：

```c
ioctl(fd, CMD, arg);
```

驱动中会实现对应的 ioctl 处理逻辑。

## 19 mmap 和设备文件

有些设备需要让用户程序直接访问一段内存。

这时可以通过 `mmap()` 把设备缓冲区或内核分配的内存映射到用户进程地址空间。

典型场景：

```text
图像采集缓冲区
显存
DMA buffer
高性能数据传输缓冲区
```

大致关系：

```text
用户程序
  |
  v
mmap(/dev/xxx)
  |
  v
驱动 mmap 回调
  |
  v
把某段内存映射到用户进程虚拟地址空间
```

`mmap()` 可以减少用户态和内核态之间的数据拷贝，但实现和使用都需要更小心。

## 20 块设备、分区和文件系统的关系

块设备、分区、文件系统经常容易混淆。

例如：

```text
/dev/nvme0n1      整块 NVMe 盘
/dev/nvme0n1p1    第 1 个分区
/dev/nvme0n1p2    第 2 个分区
```

分区只是磁盘上的一段范围。

文件系统是建立在分区或块设备上的数据组织格式。

例如：

```bash
mkfs.ext4 /dev/nvme0n1p1
mount /dev/nvme0n1p1 /mnt
```

含义：

```text
在 /dev/nvme0n1p1 上创建 ext4 文件系统
把这个文件系统挂载到 /mnt
```

简单关系：

```text
块设备 -> 分区 -> 文件系统 -> 挂载点 -> 文件和目录
```

## 21 常见命令和观察方法

查看文件类型和权限：

```bash
ls -l
```

查看 `/dev` 中设备文件：

```bash
ls -l /dev
ls -l /dev/tty*
ls -l /dev/nvme*
```

查看块设备：

```bash
lsblk
```

查看挂载情况：

```bash
mount
findmnt
```

查看文件系统类型和容量：

```bash
df -hT
```

查看设备主次设备号：

```bash
ls -l /dev/null
ls -l /dev/nvme0n1
```

查看系统 I/O 内存资源：

```bash
cat /proc/iomem
```

查看中断：

```bash
cat /proc/interrupts
```

查看设备和驱动：

```bash
ls /sys/bus/platform/devices/
ls /sys/bus/platform/drivers/
ls /sys/class/
```

查看某个设备文件被哪些进程使用：

```bash
lsof /dev/ttyAMA0
```

如果系统没有 `lsof`，也可以通过 `/proc` 间接排查。

## 22 简单实验

### 22.1 查看设备文件类型

执行：

```bash
ls -l /dev/null
ls -l /dev/zero
ls -l /dev/nvme0n1
```

观察第一列第一个字符：

```text
c：字符设备
b：块设备
```

同时观察主设备号和次设备号。

### 22.2 观察 /proc 信息

执行：

```bash
cat /proc/cpuinfo | head
cat /proc/meminfo | head
cat /proc/interrupts | head
```

这些文件看起来像普通文本文件，但内容是内核动态生成的。

### 22.3 观察 /sys 设备模型

执行：

```bash
ls /sys/class
ls /sys/class/net
ls /sys/class/block
```

可以看到网络设备、块设备等分类信息。

### 22.4 挂载和卸载测试

如果有一个空闲分区或测试镜像，可以尝试：

```bash
mkdir -p /mnt/test
mount /dev/xxx /mnt/test
findmnt /mnt/test
umount /mnt/test
```

注意不要在重要分区上随便执行格式化命令。

## 23 常见理解误区

### 23.1 设备文件不是普通磁盘文件

`/dev/ttyS0`、`/dev/null`、`/dev/nvme0n1` 这类文件不是普通数据文件。

它们是访问内核驱动和设备的入口。

### 23.2 /proc 和 /sys 里的文件很多不占真实磁盘空间

`/proc/cpuinfo`、`/proc/interrupts`、`/sys/class/net` 等内容通常是内核动态生成的。

它们不是普通意义上存放在磁盘里的文件。

### 23.3 块设备不等于文件系统

块设备只是存储设备或分区。

文件系统是建立在块设备之上的数据组织结构。

没有挂载文件系统之前，不能通过目录树访问其中的文件。

### 23.4 /dev 是操作设备，/sys 是观察和配置设备属性

`/dev` 主要用于打开设备并进行读写控制。

`/sys` 主要用于查看设备模型、驱动绑定关系和设备属性。

### 23.5 主设备号和次设备号不是随便看的编号

主设备号和次设备号是内核识别设备文件的重要信息。

它们决定设备文件最终关联到哪类驱动和哪个具体设备。

## 24 总结

文件系统和设备文件可以这样理解：

```text
文件系统负责组织文件和目录
VFS 提供统一的文件访问抽象
/dev 提供设备访问入口
/proc 和 /sys 提供内核状态与设备模型信息
```

设备文件的访问路径可以理解为：

```text
用户程序
  |
  v
open/read/write/ioctl/mmap
  |
  v
VFS
  |
  v
设备驱动
  |
  v
硬件设备
```

一句话总结：

```text
Linux 用统一的文件接口管理普通文件、虚拟文件和设备文件，设备文件是用户态程序访问内核驱动的重要入口。
```

驱动和系统调试中要特别记住：

```text
/dev 用来操作设备
/sys 用来查看设备和驱动关系
/proc 用来查看进程和内核运行状态
块设备通常承载文件系统
字符设备常用于流式数据和设备控制
```
