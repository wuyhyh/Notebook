# 02-C语言基础复习-变量函数数组指针

## 1 文档目标

本文用于帮助实习生复习 C 语言中最基础、最常用的几个概念：

1. 变量。
2. 函数。
3. 数组。
4. 指针。
5. 函数参数传递。
6. 数组和指针的关系。

学习完本文后，应能够看懂简单 C 程序，并为后续学习结构体、宏定义、寄存器操作、Linux 驱动代码打基础。

## 2 变量基础

## 2.1 什么是变量

变量是程序中用来保存数据的名字。

示例：

```c
int count = 10;
char ch = 'A';
unsigned int flag = 1;
```

这里：

1. `count` 是一个整数变量。
2. `ch` 是一个字符变量。
3. `flag` 是一个无符号整数变量。

变量的核心作用是保存程序运行过程中的数据。

## 2.2 常见基本类型

常见类型：

```c
char c;
int i;
unsigned int u;
long l;
unsigned long addr;
float f;
double d;
```

在嵌入式和驱动代码中，更常见的是使用固定宽度类型：

```c
#include <stdint.h>

uint8_t  a;
uint16_t b;
uint32_t c;
uint64_t d;
```

在 Linux 内核代码中，经常看到：

```c
u8  a;
u16 b;
u32 c;
u64 d;
```

这类类型能明确表示变量占多少 bit，更适合寄存器和协议字段操作。

## 2.3 有符号和无符号

有符号整数可以表示负数：

```c
int a = -1;
```

无符号整数不能表示负数：

```c
unsigned int b = 1;
```

需要注意：

```c
unsigned int x = 0;

x = x - 1;
```

此时 `x` 不会变成 `-1`，而是变成一个很大的无符号数。

在寄存器、地址、长度、bit mask 场景中，经常使用无符号类型。

## 2.4 变量初始化

定义变量时，尽量初始化：

```c
int ret = 0;
unsigned int value = 0;
char *name = NULL;
```

不推荐：

```c
int ret;
```

未初始化变量的值是不确定的，可能导致难以排查的问题。

## 2.5 局部变量和全局变量

局部变量定义在函数内部：

```c
int add(int a, int b)
{
    int sum = a + b;

    return sum;
}
```

`sum` 只在 `add()` 函数内部有效。

全局变量定义在函数外部：

```c
int debug_enable = 1;
```

全局变量可以被多个函数使用，但容易造成依赖混乱。

建议：

1. 能用局部变量就不要用全局变量。
2. 全局变量要有清晰命名。
3. 不要随意在头文件中定义全局变量。

## 3 函数基础

## 3.1 什么是函数

函数是完成某个功能的一段代码。

示例：

```c
int add(int a, int b)
{
    return a + b;
}
```

这个函数：

1. 函数名是 `add`。
2. 参数是 `a` 和 `b`。
3. 返回值类型是 `int`。
4. 功能是返回两个整数的和。

## 3.2 函数声明和函数定义

函数声明告诉编译器函数存在：

```c
int add(int a, int b);
```

函数定义提供具体实现：

```c
int add(int a, int b)
{
    return a + b;
}
```

通常：

1. 函数声明放在 `.h` 头文件中。
2. 函数定义放在 `.c` 源文件中。

## 3.3 void 返回值

如果函数不需要返回值，可以使用 `void`：

```c
void led_on(void)
{
    /* turn on led */
}
```

`void` 参数表示函数不接收参数。

推荐写法：

```c
void led_on(void);
```

不推荐省略参数：

```c
void led_on();
```

## 3.4 返回错误码

嵌入式和 Linux C 代码中，函数经常返回错误码：

```c
int device_init(void)
{
    if (/* failed */) {
        return -1;
    }

    return 0;
}
```

常见习惯：

1. 返回 `0` 表示成功。
2. 返回负数表示失败。
3. 返回正数通常表示长度、数量或状态值。

示例：

```c
int ret;

ret = device_init();
if (ret < 0) {
    return ret;
}
```

## 3.5 函数命名

函数名应该表达动作和对象。

推荐：

```c
uart_init();
uart_send();
gpio_set_value();
device_probe();
```

不推荐：

```c
doit();
func1();
test();
```

好的函数名可以减少注释依赖。

## 4 函数参数传递

## 4.1 C 语言默认是值传递

示例：

```c
void set_value(int x)
{
    x = 100;
}

int main(void)
{
    int a = 10;

    set_value(a);

    return 0;
}
```

调用 `set_value(a)` 时，函数内部的 `x` 是 `a` 的副本。

所以函数内部修改 `x`，不会修改外部的 `a`。

## 4.2 通过指针修改外部变量

如果希望函数修改外部变量，需要传入变量地址：

```c
void set_value(int *p)
{
    *p = 100;
}

int main(void)
{
    int a = 10;

    set_value(&a);

    return 0;
}
```

这里：

1. `&a` 表示 `a` 的地址。
2. `p` 保存 `a` 的地址。
3. `*p = 100` 修改的是 `a` 本身。

## 4.3 参数只读时使用 const

如果函数只读取指针指向的数据，不修改它，可以使用 `const`：

```c
int print_value(const int *p)
{
    if (p == NULL) {
        return -1;
    }

    /* only read *p */

    return 0;
}
```

`const int *p` 表示不能通过 `p` 修改它指向的整数。

这能让接口含义更清楚，也能减少误操作。

## 5 数组基础

## 5.1 什么是数组

数组用于保存一组相同类型的数据。

示例：

```c
int values[4] = {1, 2, 3, 4};
```

这个数组有 4 个元素。

访问元素：

```c
int x = values[0];
int y = values[3];
```

数组下标从 0 开始。

## 5.2 数组越界

数组长度是 4，有效下标是：

```text
0 1 2 3
```

错误访问：

```c
values[4] = 10;
```

这是数组越界，会写到不属于数组的内存区域。

数组越界是 C 语言中非常常见、也非常危险的问题。

## 5.3 字符数组

字符串可以保存在字符数组中：

```c
char name[] = "uart0";
```

实际内容包括字符串结尾的 `\0`：

```text
u a r t 0 \0
```

所以 `"uart0"` 这个字符串需要 6 个字符空间。

## 5.4 数组长度

数组长度可以这样计算：

```c
int values[4] = {1, 2, 3, 4};
int count = sizeof(values) / sizeof(values[0]);
```

这里：

1. `sizeof(values)` 是整个数组的大小。
2. `sizeof(values[0])` 是一个元素的大小。
3. 两者相除得到元素个数。

注意：数组作为函数参数传递后，这种方式通常不再有效。

## 6 指针基础

## 6.1 什么是指针

指针是保存地址的变量。

示例：

```c
int value = 10;
int *p = &value;
```

含义：

1. `value` 是普通变量。
2. `&value` 是 `value` 的地址。
3. `p` 是指针变量。
4. `p` 保存的是 `value` 的地址。
5. `*p` 表示访问这个地址上的内容。

## 6.2 取地址和解引用

取地址：

```c
int *p = &value;
```

解引用：

```c
int x = *p;
```

通过指针修改变量：

```c
*p = 20;
```

此时 `value` 的值也会变成 20。

## 6.3 空指针

如果指针暂时没有指向有效对象，应初始化为 `NULL`：

```c
int *p = NULL;
```

使用前检查：

```c
if (p != NULL) {
    *p = 10;
}
```

不要解引用空指针：

```c
*p = 10;
```

如果 `p` 是 `NULL`，这会导致程序错误。

## 6.4 指针类型

不同类型的指针表示不同类型数据的地址：

```c
int *pi;
char *pc;
unsigned long *pa;
```

指针类型会影响：

1. 解引用时读取多少字节。
2. 指针加法时移动多少字节。
3. 编译器如何检查类型。

## 7 数组和指针的关系

## 7.1 数组名常常表示首元素地址

示例：

```c
int values[4] = {1, 2, 3, 4};

int *p = values;
```

这里 `values` 在很多表达式中会转换为 `&values[0]`。

所以：

```c
int *p = values;
```

等价于：

```c
int *p = &values[0];
```

## 7.2 通过指针访问数组元素

数组下标访问：

```c
int x = values[2];
```

指针访问：

```c
int x = *(p + 2);
```

二者访问的是同一个元素。

## 7.3 指针加法

```c
int values[4] = {1, 2, 3, 4};
int *p = values;

p = p + 1;
```

如果 `int` 是 4 字节，`p + 1` 实际移动 4 字节。

指针加法不是简单的地址数字加 1，而是移动一个元素大小。

## 7.4 数组作为函数参数

函数参数中写数组：

```c
void print_values(int values[], int count)
{
    int i;

    for (i = 0; i < count; i++) {
        /* print values[i] */
    }
}
```

实际上和下面写法非常接近：

```c
void print_values(int *values, int count)
{
    int i;

    for (i = 0; i < count; i++) {
        /* print values[i] */
    }
}
```

所以数组传参时，通常还需要额外传入长度。

## 8 指针和字符串

## 8.1 字符串常量

```c
const char *name = "uart0";
```

推荐加 `const`，因为字符串常量通常不应该被修改。

不推荐：

```c
char *name = "uart0";
name[0] = 'U';
```

这可能导致程序错误。

## 8.2 字符数组和字符指针的区别

字符数组：

```c
char name[] = "uart0";
```

这里 `name` 是一个数组，数组中保存了字符串内容。

字符指针：

```c
const char *name = "uart0";
```

这里 `name` 是一个指针，指向字符串常量。

简单理解：

1. 字符数组自己拥有一份数据。
2. 字符指针只是指向一段已有字符串。

## 9 嵌入式代码中的典型用法

## 9.1 地址变量

寄存器基地址经常用无符号整数或指针表示：

```c
#define UART_BASE 0x28001000UL
```

在裸机代码中可能会看到：

```c
volatile unsigned int *reg = (volatile unsigned int *)UART_BASE;
```

含义是把一个地址转换成寄存器指针。

## 9.2 volatile 指针

访问硬件寄存器时，常使用 `volatile`：

```c
volatile unsigned int *reg;
```

`volatile` 告诉编译器：这个地址上的值可能被硬件改变，不要随意优化读写。

示例：

```c
*reg = 0x1;
```

表示向寄存器写入值。

## 9.3 缓冲区

数组常用于表示缓冲区：

```c
char buf[128];
```

函数处理缓冲区时，通常传入指针和长度：

```c
int send_data(const char *buf, unsigned int len)
{
    if (buf == NULL) {
        return -1;
    }

    /* send len bytes */

    return 0;
}
```

这种写法比只传一个指针更安全，因为函数知道缓冲区长度。

## 10 常见错误

## 10.1 使用未初始化变量

错误：

```c
int ret;

if (ret < 0) {
    return ret;
}
```

`ret` 没有初始化，值是不确定的。

正确：

```c
int ret = 0;
```

## 10.2 使用未初始化指针

错误：

```c
int *p;

*p = 10;
```

`p` 没有指向有效地址。

正确：

```c
int value = 0;
int *p = &value;

*p = 10;
```

## 10.3 数组越界

错误：

```c
int values[4];

values[4] = 10;
```

正确下标范围是 `0` 到 `3`。

## 10.4 忘记传数组长度

不推荐：

```c
void clear_buffer(char *buf)
{
    /* 不知道 buf 有多大 */
}
```

推荐：

```c
void clear_buffer(char *buf, unsigned int len)
{
    if (buf == NULL) {
        return;
    }

    /* clear len bytes */
}
```

处理数组或缓冲区时，尽量同时传入长度。

## 10.5 试图修改字符串常量

错误：

```c
char *name = "uart0";

name[0] = 'U';
```

推荐：

```c
const char *name = "uart0";
```

如果需要修改字符串，应使用字符数组：

```c
char name[] = "uart0";

name[0] = 'U';
```

## 11 建议掌握的习惯

## 11.1 变量习惯

建议：

1. 定义变量时尽量初始化。
2. 变量名表达含义。
3. 地址、长度、bit mask 优先使用无符号类型。
4. 寄存器字段优先使用固定宽度类型。

## 11.2 函数习惯

建议：

1. 函数只做一件相对明确的事情。
2. 函数名表达动作。
3. 返回 `0` 表示成功，返回负数表示失败。
4. 指针参数使用前检查 `NULL`。
5. 只读指针参数加 `const`。

## 11.3 数组习惯

建议：

1. 注意数组下标从 0 开始。
2. 不要越界访问。
3. 数组传参时同时传长度。
4. 字符串注意结尾的 `\0`。

## 11.4 指针习惯

建议：

1. 指针定义时初始化。
2. 暂时不用的指针设为 `NULL`。
3. 不确定指针是否有效时，先检查。
4. 不要返回局部变量地址。
5. 不要随意进行复杂指针运算。

## 12 简单练习

## 12.1 变量练习

定义以下变量：

1. 一个整数 `count`，初始值为 0。
2. 一个无符号整数 `flags`，初始值为 0。
3. 一个字符数组 `name`，内容为 `"uart0"`。
4. 一个指针 `p`，指向 `count`。

参考：

```c
int count = 0;
unsigned int flags = 0;
char name[] = "uart0";
int *p = &count;
```

## 12.2 函数练习

写一个函数，把传入的整数清零：

```c
int clear_value(int *value)
{
    if (value == NULL) {
        return -1;
    }

    *value = 0;
    return 0;
}
```

调用：

```c
int count = 10;

clear_value(&count);
```

## 12.3 数组练习

写一个函数，计算数组元素之和：

```c
int sum_array(const int *values, unsigned int count)
{
    unsigned int i;
    int sum = 0;

    if (values == NULL) {
        return 0;
    }

    for (i = 0; i < count; i++) {
        sum += values[i];
    }

    return sum;
}
```

调用：

```c
int values[] = {1, 2, 3, 4};
int count = sizeof(values) / sizeof(values[0]);
int sum;

sum = sum_array(values, count);
```

## 12.4 指针练习

阅读下面代码，判断 `a` 最后的值：

```c
int a = 10;
int *p = &a;

*p = 20;
```

结果：`a` 的值变成 20。

## 13 小结

变量、函数、数组、指针是 C 语言最基础的内容。

需要重点记住：

1. 变量用于保存数据。
2. 函数用于组织代码逻辑。
3. C 语言函数参数默认是值传递。
4. 如果函数要修改外部变量，需要传入指针。
5. 数组用于保存一组相同类型的数据。
6. 数组下标从 0 开始。
7. 数组传参时通常需要同时传长度。
8. 指针保存的是地址。
9. `&` 用于取地址。
10. `*` 用于通过地址访问内容。
11. 使用指针前要确认它有效。
12. 嵌入式和驱动代码中，指针常用于访问缓冲区、结构体和寄存器。

这些基础概念掌握之后，再学习结构体、宏定义、寄存器操作和 Linux 驱动代码会顺畅很多。
