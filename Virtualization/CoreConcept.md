# QEMU 和 KVM 概述

## 1.1 虚拟化简介

### 1.1.1 虚拟化思想

计算机发展中虚拟化思想也在不断地发展。一些例子：

CPU执行的机器指令到汇编语言、C语言再到高级语言。
磁盘的磁道柱面概念到文件与目录。
TCP/IP协议栈使网卡透明，我们只需要发送数据和地址就能进行处理。
每一个进程都是对计算机的抽象，是一个状态机。
操作系统会提供API给应用程序，进行服务。

底层的资源通过空间的分割或者时间的分割，转换成一种简单易用的资源给上层使用，这就是虚拟化。

### 1.1.2 虚拟机简介

虚拟机是一个环境，可以完成用户指定的操作的机器。

最简单的虚拟机是进程。

模拟器是另一种形式的虚拟机，可以实现ISA的翻译。QEMU就可以实现用户态程序的模拟。

高级语言的虚拟机在模拟器上更进一步，没有具体的运行在物理上的ISA，而是定义虚拟的字节码。
典型的高级语言虚拟机比如JVM虚拟机、Python虚拟机。

进程、模拟器、高级语言的虚拟机都是提供指令的执行环境，而系统虚拟机是提供一个完整的系统环境。
系统级虚拟化中，管理全局物理资源的软件是虚拟机监控器 VMM。VMM 与 VM 的关系就像操作系统与进程一样。
系统虚拟化解决方案有：VMware Workstation、QEMU、Virtual Box

### 1.1.3 系统虚拟化历史

大型机时代的多用户使用需求。

普通PC也支持多个操作系统，多个用户。

云计算给租户提供虚拟化的资源。

Intel AMD 都在CPU硬件层面增加了虚拟化支持。随着性能要求的提高，内存、外设也在硬件层面提供了虚拟化支持。

KVM是一个非常成功的VMM。

## 1.2 QEMU 和 KVM 架构介绍

### 1.2.1 QEMU 和 KVM 历史

QEMU 和 KVM 经常放在一起讨论，其实二者是可以解耦合的。

QEMU可以完成用户程序的模拟和系统虚拟化模拟。
用户程序的模拟是一种平台编译的二进制可以在另一种平台上运行。TCG引擎翻译指令集。
系统虚拟化模拟是让虚拟机有自己的vCPU、芯片组、虚拟内存和外部设备，虚拟机中运行的操作系统看到的硬件视图和物理机是一样的。

KVM 本身是一个内核模块，导出了一系列接口到用户空间，用户空间可以使用这些接口创建虚拟机。
KVM 完成核心的CPU虚拟化和内存虚拟化，使用QEMU作为其用户态组件。

QEMU-KVM系统虚拟化软件已经成为了云计算的基础。
