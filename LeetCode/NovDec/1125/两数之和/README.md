# LeetCode 第1题：两数之和

### 问题描述

你提到想学习LeetCode第1题，这个题目要求我们在给定的数组中找到两个数，使它们的和等于目标值，并返回它们的索引。

### 代码实现

以下是“两数之和”的 Python 解法：

```python
def two_sum(nums, target):
    # 创建一个哈希表来存储数值和它们的索引
    num_dict = {}
    
    # 遍历数组中的每个元素
    for i, num in enumerate(nums):
        # 计算差值，看看是否已经存在于哈希表中
        complement = target - num
        if complement in num_dict:
            # 如果找到匹配的元素，返回它们的索引
            return [num_dict[complement], i]
        # 否则将当前元素存入哈希表
        num_dict[num] = i

# 示例用法
nums = [2, 7, 11, 15]
target = 9
print(two_sum(nums, target))  # 输出: [0, 1]
```

### 问题和讲解

**1. `enumerate(nums)` 是什么意思？**

`enumerate(nums)` 是 Python 中的一个内置函数，它在遍历列表 `nums` 时同时提供每个元素的索引和值。

在代码 `for i, num in enumerate(nums):` 中，`i` 是当前元素的索引，`num` 是当前元素的值。例如，对于 `nums = [2, 7, 11, 15]`，`enumerate(nums)` 会产生 `(0, 2), (1, 7), (2, 11), (3, 15)`，使得我们可以同时访问索引和元素值。

**2. `num_dict[complement]` 的作用是什么？**

`num_dict[complement]` 表示在字典 `num_dict` 中查找键为 `complement` 的值。在代码中，`complement` 是目标值减去当前元素的结果。

例如，当我们遍历到元素 `7` 时，目标值为 `9`，则 `complement = 9 - 7 = 2`。如果字典中已经存在键 `2`，则说明我们找到了与 `7` 之和等于 `9` 的另一个数，即为 `2`，并返回它们的索引。

**3. 为什么使用哈希表（字典）？**

使用哈希表（字典）是为了加快查找的速度。哈希表的查找时间复杂度为 O(1)，使得在遍历数组时可以快速找到满足条件的数对。

在遍历数组的过程中，我们将每个数存储在哈希表中，键是数值，值是对应的索引。当我们需要找到目标值时，只需通过查找哈希表中是否存在匹配的差值即可，从而大大提高了效率。

**4. 算法的时间复杂度是什么？**

该算法的时间复杂度是 O(n)，其中 `n` 是数组的长度。原因是我们只需要遍历数组一次，对于每个元素的查找和插入操作都是 O(1) 的。因此，总体复杂度为 O(n)，适合用于大多数场景。
