## 正文（Hint）

启动 LFS 系统已经不再像以前那样简单地执行 `grub-install /dev/sda` 了。现在有了更多的选择和考虑因素。

在开始 LFS 项目时，书中假设你已经安装了一个“主流”的 Linux 发行版。然而，这个安装可能会使得创建用于构建和启动 LFS 的分区变得困难。

主要涉及两个问题：**分区表格式** 和 **磁盘上的分区布局**。

有些发行版的默认安装仍然使用已经过时的 **MSDOS 分区表**，而不是更现代的 **GPT（GUID 分区表）**
。此外，多数发行版的默认安装会使用整个硬盘，不留出空闲空间。

解决办法有几个：

* 重新安装宿主操作系统，手动分区；
* 或者使用第二块硬盘；
* 如果是在虚拟环境（如 QEMU 或 VMware）中安装 LFS，也需要像单硬盘系统一样安装宿主系统。

另一个问题是与你所用计算机的硬件有关。旧系统使用的是传统 BIOS 固件，而新系统则使用 **UEFI（统一可扩展固件接口）**。

使用 UEFI 会带来一些额外的假设和复杂性：

1. **Secure Boot（安全启动）**：LFS 使用的是自定义构建的内核，建议禁用 Secure Boot，否则会带来干扰；
2. **兼容性模式（Compatibility Mode）**：有些 UEFI 系统支持，有些不支持。不同厂商的 BIOS 设置中叫法不同，可能是：

    * CSM（兼容性支持模块）
    * Legacy Mode（传统模式）
    * 或者一个“启用/禁用 EFI 模式”的选项
      如果你的系统支持，建议启用兼容模式，构建 LFS 更简单。

## 分区建议

建议使用启动盘进入安装系统，使用 `fdisk` 或 `gdisk` 创建 GPT 分区表，并创建如下分区布局：

### 如果是 BIOS 启动：

```
编号    起始扇区     结束扇区     大小         类型代码 名称
1       2048         4095         1 MiB        EF02     BIOS boot partition
2       4096         413695       200 MiB      8300     /boot
3       413696       4607999      2 GiB        8200     swap
4       4608000      46551039     20 GiB       8300     debian
```

### 如果是 UEFI 启动：

```
编号    起始扇区     结束扇区     大小         类型代码 名称
1       2048         411647       200 MiB      EF00     EFI System
2       411648       821247       200 MiB      8300     /boot
3       821248       5015551      2 GiB        8200     swap
4       5015552      46958591     20 GiB       8300     debian
```

---

如果你是在已有 Linux 系统下使用第二块硬盘安装 LFS，你只需要前两个分区。无论哪种情况，你都将在 LFS 手册的第 2.4 节中创建最终用于构建
LFS 的主分区。

> 使用单独的 `/boot` 分区的最大好处是可以让 GRUB 从统一位置读取多个系统的引导文件，简化 GRUB 配置。

---

我个人建议 `/boot` 分区使用 ext2 文件系统，并执行以下命令格式化：

```bash
mkfs.ext2 -m0 /dev/sda2
```

或如果是 NVMe 硬盘：

```bash
mkfs.ext2 -m0 /dev/nvme0n1p2
```

解释：

* 使用 ext2 是因为 `/boot` 中的内容几乎不变；
* `-m0` 表示不保留任何超级用户块（因为不需要写入）；
* 也没必要使用带日志的文件系统。

最后一点，分区名称（例如 "debian"）可以随时用 `gdisk` 或 `fdisk` 的专家模式添加。这在后续配置 GRUB 时很有帮助。

注意：

* 上面的 “debian” 仅为示例标签，可以改成任何你喜欢的名称；
* 它代表一个标准 Linux 系统用于作为构建 LFS 的起点。
