LeetCode第39题是《组合总和》（Combination Sum），题目要求给定一个**无重复元素**的候选数字集合 `candidates` 和一个目标数字 `target`，返回所有可以组合成 `target` 的 **组合**，每个数字在每个组合中可以使用 **无限次**。

### 题目描述

给定一个无重复元素的整数数组 `candidates` 和一个目标值 `target`，返回 **所有** 可以使得数字和为 `target` 的组合。**你可以重复使用数组中的数字**。

### 示例

**示例 1：**

```plaintext
输入: candidates = [2,3,6,7], target = 7
输出:
[[7], [2, 2, 3]]
```

**示例 2：**

```plaintext
输入: candidates = [2,3,5], target = 8
输出:
[[2, 2, 2, 2], [2, 3, 3], [3, 5]]
```

**示例 3：**

```plaintext
输入: candidates = [2], target = 1
输出: []
```

### 解题思路

这个问题可以通过 **回溯算法** 来解决。回溯算法非常适合处理组合问题，尤其是当我们需要从一个列表中选择元素并允许重复使用时。

#### 回溯算法思路：

1. **递归**：从 `candidates` 数组中选择一个数字，加入当前的组合中，然后递归处理剩余的目标值。
2. **重复使用数字**：与上一层的数字可以重复，因此我们可以在递归时不限制数字的使用次数。
3. **终止条件**：当 `target` 减少到 0 时，表示当前组合是一个有效的解；当 `target` 小于 0 时，表示当前组合无效，返回上一层。
4. **剪枝**：如果当前数字已经大于目标 `target`，可以直接跳过，不再考虑后续的数字。

### 代码实现

```python
def combinationSum(candidates, target):
    result = []

    def backtrack(start, target, path):
        if target == 0:
            result.append(path[:])  # 当前组合有效，加入结果
            return
        if target < 0:
            return  # 当前组合无效，结束递归
        
        # 从当前位置开始，尝试每个候选数字
        for i in range(start, len(candidates)):
            path.append(candidates[i])  # 选择当前数字
            backtrack(i, target - candidates[i], path)  # 递归调用，注意不改变起始位置 i
            path.pop()  # 回溯，撤销选择
    
    backtrack(0, target, [])
    return result
```

### 代码解析

1. **`backtrack(start, target, path)`**：
    - `start`: 表示当前递归应该从 `candidates[start]` 开始，保证每个数字可以重复使用。
    - `target`: 当前剩余需要组合的目标数值。
    - `path`: 当前的组合路径，用来存储已选的数字。

2. **递归终止条件**：
    - `if target == 0`: 如果目标为 0，表示当前的组合有效，将其加入结果。
    - `if target < 0`: 如果目标小于 0，表示当前路径无效，返回上一层。

3. **回溯过程**：
    - 从 `start` 索引开始，遍历候选数字 `candidates`。
    - 对于每个数字，选择它并将其加入 `path`。
    - 然后递归调用 `backtrack(i, target - candidates[i], path)`，注意 `i` 不变，允许重复使用当前数字。
    - 回溯时，通过 `path.pop()` 移除上一个选择的数字，尝试其他的组合。

4. **返回结果**：最后返回 `result`，即所有有效的组合。

### 时间复杂度

- **时间复杂度**：`O(2^n)`，因为回溯算法会遍历所有可能的组合。每次递归有两个选择：选择当前数字或不选择当前数字，因此复杂度是指数级的。
- **空间复杂度**：`O(target / min(candidates))`，因为递归的深度最多为 `target / min(candidates)`，即每次递归选择最小的数字，最终可以得到的递归深度。

### 例子解析

以 `candidates = [2, 3, 6, 7]` 和 `target = 7` 为例：

1. 初始调用：`backtrack(0, 7, [])`
2. 选择 `2`，剩余目标为 `7 - 2 = 5`，递归调用：`backtrack(0, 5, [2])`
3. 选择 `2`，剩余目标为 `5 - 2 = 3`，递归调用：`backtrack(0, 3, [2, 2])`
4. 选择 `2`，剩余目标为 `3 - 2 = 1`，递归调用：`backtrack(0, 1, [2, 2, 2])`
5. 选择 `2`，剩余目标为 `1 - 2 = -1`，终止递归（无效）。
6. 回溯，选择 `3`，剩余目标为 `1 - 3 = -2`，终止递归（无效）。
7. 回溯，选择 `6`，剩余目标为 `1 - 6 = -5`，终止递归（无效）。
8. 回溯，选择 `7`，剩余目标为 `1 - 7 = -6`，终止递归（无效）。
9. 回到 `backtrack(0, 5, [2])`，选择 `3`，递归调用：`backtrack(1, 2, [2, 3])`
10. 继续尝试，最终得到有效组合 `[2, 2, 3]` 和 `[7]`。

最终返回的结果是：

```plaintext
[[7], [2, 2, 3]]
```

### 总结

这道题的解法是回溯算法的经典应用，特别适合处理组合问题，其中允许重复使用元素。通过递归构建组合，并利用回溯撤销选择，从而生成所有可能的有效组合。
