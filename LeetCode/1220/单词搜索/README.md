LeetCode第79题是《单词搜索》（Word Search），题目要求给定一个二维字符网格 `board` 和一个字符串 `word`，判断 `word` 是否存在于网格中。每次可以从网格中选择一个字符，并沿着水平方向或竖直方向进行连接。每个字符只能使用一次。

### 题目描述

给定一个二维字符网格 `board` 和一个字符串 `word`，在网格中查找字符串 `word`。如果存在返回 `true`，否则返回 `false`。

**注意**：
- 单词必须按字母顺序连续的方向（水平或垂直）在网格中形成。
- 字符必须沿着网格的上下或左右相邻的方向连接。
- 每个格子内的字母可以在构建单词时重复使用。

### 示例

**示例 1：**

```plaintext
输入: board = [
  ['A','B','C','E'],
  ['S','F','C','S'],
  ['A','D','E','E']
], word = "ABCF"
输出: true
```

**示例 2：**

```plaintext
输入: board = [
  ['A','B','C','E'],
  ['S','F','C','S'],
  ['A','D','E','E']
], word = "SEE"
输出: true
```

**示例 3：**

```plaintext
输入: board = [
  ['A','B','C','E'],
  ['S','F','C','S'],
  ['A','D','E','E']
], word = "ABCB"
输出: false
```

### 解题思路

该题目可以使用 **深度优先搜索（DFS）** 来解决。具体来说，可以从网格中的每个字符开始，尝试寻找是否能形成 `word`。DFS 是一种通过递归不断深入，直到找到目标或走不通时返回的策略。

#### 步骤

1. **遍历每个字符**：首先遍历网格中的每个字符，从每个字符开始，尝试在四个方向上进行 DFS 搜索。

2. **递归搜索**：从当前字符开始，依次尝试字符的相邻位置，并与 `word` 中的字符逐一进行匹配。如果匹配成功，就继续搜索下一个字符。

3. **回溯**：当一个字符不符合要求时，回到上一层，撤销当前位置的选择。

4. **边界条件**：检查搜索是否超出网格边界，或者当前字符是否与 `word` 中的目标字符不匹配。

5. **终止条件**：当遍历完成整个 `word` 时，表示找到了一个匹配的路径。

### 代码实现

```python
def exist(board, word):
    # 定义四个方向：上、下、左、右
    directions = [(-1, 0), (1, 0), (0, -1), (0, 1)]
    
    def dfs(i, j, word_index):
        # 如果当前索引等于单词长度，表示已经找到完整的单词
        if word_index == len(word):
            return True
        
        # 边界检查
        if i < 0 or i >= len(board) or j < 0 or j >= len(board[0]):
            return False
        
        # 如果当前字符不匹配，返回 False
        if board[i][j] != word[word_index]:
            return False
        
        # 记录当前字符，并将其标记为已访问
        temp = board[i][j]
        board[i][j] = '#'
        
        # 递归搜索四个方向
        for di, dj in directions:
            ni, nj = i + di, j + dj
            if dfs(ni, nj, word_index + 1):
                return True
        
        # 恢复当前字符，回溯
        board[i][j] = temp
        return False
    
    # 遍历每个字符作为起始点
    for i in range(len(board)):
        for j in range(len(board[0])):
            if dfs(i, j, 0):  # 从当前位置开始搜索
                return True
    
    return False
```

### 代码解析

1. **`directions`**：表示四个方向的移动：上（`(-1, 0)`）、下（`(1, 0)`）、左（`(0, -1)`）、右（`(0, 1)`）。

2. **`dfs(i, j, word_index)`**：
    - `i, j`：当前搜索的位置（行和列）。
    - `word_index`：当前匹配 `word` 的字符索引。
    - **边界条件**：如果 `i` 和 `j` 超出了网格的边界，返回 `False`。
    - **匹配检查**：如果当前字符不匹配 `word` 中的字符，返回 `False`。
    - **回溯**：将当前字符标记为 `#`（已访问），避免重复访问；然后递归检查四个方向。
    - **恢复原字符**：如果当前搜索路径失败，恢复当前字符，并回溯到上一层。

3. **遍历网格**：从每个字符开始调用 `dfs`，判断是否能找到匹配的 `word`。

4. **返回值**：如果找到匹配的单词路径，返回 `True`，否则遍历完整个网格后返回 `False`。

### 时间复杂度

- **时间复杂度**：`O(m * n * 4^L)`，其中 `m` 和 `n` 分别是网格的行和列数，`L` 是单词 `word` 的长度。因为每个位置都可能递归探索四个方向，而最大递归深度为 `L`（单词的长度）。最坏情况下，可能需要访问整个网格的每个位置，因此复杂度为 `O(m * n * 4^L)`。

- **空间复杂度**：`O(L)`，主要是递归栈的空间，最大递归深度为 `L`。

### 示例解析

以 `board = [['A','B','C','E'], ['S','F','C','S'], ['A','D','E','E']]` 和 `word = "ABCF"` 为例：

1. **开始搜索**：从第一个字符 `'A'` 开始，`word_index = 0`。
    - 字符 `'A'` 匹配成功，继续搜索下一个字符 `'B'`。
2. **继续递归**：从 `'B'` 开始，找到字符 `'B'`，继续搜索下一个字符 `'C'`。
3. **继续递归**：从 `'C'` 开始，找到字符 `'C'`，继续搜索下一个字符 `'F'`。
4. **完成搜索**：最终找到字符 `'F'`，成功匹配。

最终返回 `True`。

### 总结

- **深度优先搜索（DFS）** 是解决此类问题的常见方法，尤其适合寻找路径和匹配问题。
- **回溯** 通过标记和恢复位置，避免重复计算和无效路径的搜索。
- 该问题的关键在于处理边界条件和回溯过程，以确保算法高效并且能覆盖所有可能的路径。
