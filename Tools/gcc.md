GCC（GNU Compiler Collection）是 Linux 平台上最常用的 C/C++ 编译器，提供了丰富的编译选项和特性。以下是 GCC 的常用编译选项和特性分类：

---

## **1. 基本编译选项**

| 选项           | 说明                                  |
|--------------|-------------------------------------|
| `-o <file>`  | 指定输出文件名称                            |
| `-c`         | 只编译，不链接，生成 `.o` 目标文件                |
| `-Wall`      | 启用大多数常见的警告                          |
| `-Wextra`    | 启用额外的警告                             |
| `-pedantic`  | 严格遵循 C/C++ 标准                       |
| `-std=c11`   | 使用 C11 标准（可选 `c89`、`c99`、`gnu11` 等） |
| `-std=c++20` | 使用 C++20 标准（可选 `c++11`、`c++17` 等）   |

---

## **2. 代码优化选项**

| 选项              | 说明                                   |
|-----------------|--------------------------------------|
| `-O0`           | 不优化，调试模式                             |
| `-O1`           | 基础优化                                 |
| `-O2`           | 常用优化（推荐用于生产环境）                       |
| `-O3`           | 开启最高级别优化，包括循环展开、自动向量化等               |
| `-Ofast`        | `-O3` 的基础上启用不严格符合标准的优化               |
| `-Os`           | 优化代码大小，适用于嵌入式系统                      |
| `-march=native` | 针对当前 CPU 架构优化代码                      |
| `-mtune=native` | 生成兼容代码，但优化目标是本机 CPU                  |
| `-flto`         | 启用链接时优化（Link Time Optimization, LTO） |

---

## **3. 调试相关选项**

| 选项                        | 说明                |
|---------------------------|-------------------|
| `-g`                      | 生成调试信息（适用于 `gdb`） |
| `-ggdb`                   | 为 `gdb` 生成额外的调试信息 |
| `-fsanitize=address`      | 启用地址（内存）泄漏检测      |
| `-fsanitize=undefined`    | 启用未定义行为检测         |
| `-fsanitize=thread`       | 启用线程数据竞争检测        |
| `-fstack-protector`       | 启用栈保护以防止栈溢出攻击     |
| `-fno-omit-frame-pointer` | 不优化栈帧指针（便于调试）     |

---

## **4. 预处理相关选项**

| 选项           | 说明                    |
|--------------|-----------------------|
| `-E`         | 仅进行预处理，输出预处理后的代码      |
| `-I <dir>`   | 指定头文件搜索路径             |
| `-D <macro>` | 定义预处理宏                |
| `-U <macro>` | 取消定义预处理宏              |
| `-M`         | 生成依赖文件（Makefile 依赖管理） |

---

## **5. 语言特性相关选项**

| 选项                    | 说明                                           |
|-----------------------|----------------------------------------------|
| `-fno-rtti`           | 关闭 C++ 的运行时类型识别（RTTI）                        |
| `-fno-exceptions`     | 关闭 C++ 异常处理                                  |
| `-fPIC`               | 生成位置无关代码（Position Independent Code，适用于共享库）   |
| `-fPIE`               | 生成位置无关可执行文件（Position Independent Executable） |
| `-fvisibility=hidden` | 隐藏符号，减少动态库符号表大小                              |

---

## **6. 代码分析和警告**

| 选项                  | 说明                          |
|---------------------|-----------------------------|
| `-Werror`           | 将所有警告视为错误                   |
| `-Wshadow`          | 警告变量名隐藏问题                   |
| `-Wformat`          | 检查 `printf` 和 `scanf` 格式字符串 |
| `-Wconversion`      | 检测可能的隐式转换问题                 |
| `-Wsign-compare`    | 检测有符号和无符号整数比较问题             |
| `-Wstrict-overflow` | 可能导致溢出的情况                   |
| `-fanalyzer`        | 静态代码分析                      |

---

## **7. 链接相关选项**

| 选项                  | 说明                                      |
|---------------------|-----------------------------------------|
| `-l<lib>`           | 指定要链接的库，如 `-lm`（数学库）                    |
| `-L<dir>`           | 指定库文件搜索路径                               |
| `-static`           | 生成静态可执行文件                               |
| `-shared`           | 生成动态链接库（`.so`）                          |
| `-rdynamic`         | 在可执行文件中保留符号表（便于 `dlopen` 和 `backtrace`） |
| `-Wl,-rpath,<path>` | 指定运行时动态库搜索路径                            |
| `-Wl,--gc-sections` | 删除未使用的代码段，优化二进制大小                       |

---

## **8. 目标架构相关**

| 选项                 | 说明                       |
|--------------------|--------------------------|
| `-m32`             | 生成 32 位代码                |
| `-m64`             | 生成 64 位代码                |
| `-march=armv8-a`   | 指定目标 CPU 架构（适用于 ARM 平台）  |
| `-mcpu=cortex-a72` | 指定目标 CPU 型号              |
| `-mfpu=neon`       | 启用 NEON SIMD 指令集（ARM 平台） |

---

## **9. 其他**

| 选项            | 说明                           |
|---------------|------------------------------|
| `-v`          | 显示详细编译过程                     |
| `--version`   | 显示 GCC 版本                    |
| `-time`       | 统计编译耗时                       |
| `-save-temps` | 保存编译过程中的中间文件（`.i`、`.s`、`.o`） |

---

### **示例**

#### **1. 生成可执行文件**

```bash
gcc -o myprog myprog.c
```

#### **2. 生成带调试信息的可执行文件**

```bash
gcc -g -Wall -o myprog myprog.c
```

#### **3. 生成优化后的可执行文件**

```bash
gcc -O2 -o myprog myprog.c
```

#### **4. 生成静态可执行文件**

```bash
gcc -static -o myprog myprog.c
```

#### **5. 仅编译不链接**

```bash
gcc -c myprog.c
```

#### **6. 生成动态库**

```bash
gcc -shared -fPIC -o libmylib.so mylib.c
```

#### **7. 启用 `AddressSanitizer` 进行内存错误检测**

```bash
gcc -fsanitize=address -g -o myprog myprog.c
```

---

GCC 提供了丰富的编译选项，可以针对不同需求进行优化和调整。如果你有特定的需求，比如 **内核开发、嵌入式开发或性能优化**
，可以针对特定场景深入调整编译参数。
