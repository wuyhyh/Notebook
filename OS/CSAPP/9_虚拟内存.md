# 第9章 虚拟内存

现代系统提供了一种对主存的抽象概念，叫做虚拟内存。

虚拟内存提供了3个重要的能力：

- 主存作为磁盘的高速缓存，主存中只保存活动区域，高效利用了主存；
- 为每个进程提供了一致的地址空间，简化了内存管理；
- 保护每个进程的地址空间不被其他进程破坏

本章从2个角度看待虚拟内存：
前一部分描述虚拟内存如何工作；后一部分描述应用程序如何使用和管理虚拟内存。

程序如何使用和管理虚拟内存：通过显式的内存映射和像malloc这样的动态内存分配器。

## 9.1 物理寻址和虚拟寻址

计算机的主存被组织成一个由 M 个连续的字节大小的单元组成的数组。
每个字节都有唯一的物理地址。

## 9.2 地址空间

地址空间是一个非负整数地址的有序集合。

主存中的每字节都有一个选自虚拟地址空间的虚拟地址和一个选自物理地址空间的物理地址。

## 9.3 虚拟内存作为缓存的工具

虚拟内存被组织为磁盘上由多个连续单元组成的数组。

磁盘上的数据被分割成块，这些块作为磁盘和主存之间的传输单元。

虚拟内存被分割为虚拟页。
物理内存被分割为物理页，也称为页帧。

在任意时刻，虚拟页面的集合都分为3个不相交的集合：

- 未分配的，没有任何数据和他们关联，不占用磁盘空间。
- 缓存的，已经缓存在物理内存中的已分配页。
- 未缓存的，未缓存在物理内存中的已分配页。

### 9.3.1 DRAM缓存的组织结构

DRAM缓存是全相联的，任何虚拟页都可以放置在任何物理页中。

DRAM缓存总是使用write back

### 9.3.2 页表

页表是一个PTE的数组。

虚拟地址空间中的每个页都对应一个PTE

### 9.3.3 页命中

访问已经缓存到主存的页。

### 9.3.4 缺页

DRAM 缓存不命中称为缺页。

MMU 从内存中读取了 PTE3，从有效位推断出 VP3 未被缓存，并且触发一个缺页异常。缺页异常会调用内核中的缺页异常处理程序，该程序会选择一个牺牲页（存放在PP3中的VP4）；
接下来，内核会修改 PTE4，如果 VP4 已经被修改了，内核将会把他复制回磁盘。
最后内核从磁盘复制 VP3 到内存中的 PP3，并更新 PTE3。

当异常处理程序返回的时，它会重新启动导致缺页的指令。

在虚拟内存的习惯说法中，块被称为页。

在磁盘和内存之间传送页的活动叫做交换`swapping`或者页面调度`paging`。

一直等待，直到发生不命中时才换入页面的策略称为按需页面调度 `demand paging`。

### 9.3.5 分配页面

VP5 的分配过程是在磁盘上创建空间并更新 PTE5，使它指向磁盘上这个新创建的页面。

### 9.3.6 又是局部性救了我们

虚拟内存工作得很高效，主要是局部性。

在程序的整个运行过程中引用的页面总数可能超过物理内存的大小。
但是局部性保证在任意时刻，程序将在一个较小的活动页面集合上工作，这个集合叫做工作集。

如果工作集的大小超过了物理内存的大小，页面不断换入换出，产生抖动，程序性能会大大下降。

## 9.4 虚拟内存作为内存管理工具

虚拟内存大大简化了内存管理，并且提供了一种自然的保护内存的办法。

操作系统为每个进程提供了一个独立的页表。

多个虚拟页面是可以映射到同一个共享物理页面的。

- 简化链接
- 简化加载
- 简化共享
- 简化内存分配

## 9.5 虚拟内存作为内存保护的工具

计算机系统必须为操作系统提供手段来控制对内存系统的访问。

地址翻译机制可以自然的扩展来实现访问控制。
每次 CPU 生成一个 VA 时，地址翻译硬件都会读一个 PTE，在 PTE 上添加一些额外的位就可以控制对虚拟页面的访问。

SUP位 READ位 WRITE位

如果一条指令违反了访问许可条件，CPU 就会触发保护故障，将控制传递给内核的异常处理程序。

## 9.6 地址翻译

我们需要了解硬件在虚拟内存中的角色。

从功能上说，CR3 控制寄存器和 PTBR（Page Table Base Register）是等价的概念，它们都表示当前正在使用的页表的起始物理地址。不过它们存在一些命名和架构上的差异。

物理页面偏移 PPO 和虚拟页面偏移 VPO 是相同的。

页命中完全由硬件来处理，处理缺页则需要硬件与操作系统协同。

如果 PTE 中的有效位为0，MMU 会触发一次异常，传递控制到内核中的缺页异常处理程序。

### 9.6.1 结合高速缓存和虚拟内存

有一个问题，选择用物理地址还是虚拟地址来访问 SRAM 高速缓存？

现在的系统采用的都是物理寻址，所以地址翻译出现在高速缓存查找之前。

### 9.6.2 利用 TLB 加速地址翻译

TLB 是一个小的、虚拟寻址的缓存，其中每一行都保存着一个由单个 PTE 组成的块。

MMU 将 VPN 发送给 TLB，从中取回 PTE; 如果 TLB 不命中，那么 MMU 就从L1 cache 中取。

### 9.6.3 多级页表

使用层次结构的页表可以压缩页表的空间开销。

从两个方面减少了内存要求：

- 如果第一级页表中PTE是空的，相应的第二级页表就不存在；
- 只有第一级页表才需要总是在内存中。

页表的核心任务是实现虚拟地址 → 物理地址的映射。因此，页表的每一级（如 PGD、PUD、PMD、PTE）都保存的是下一层页表所在的物理地址（再加上一些标志位，比如
Present、RW、User 等）。

## 9.7 案例研究：Intel Core i7 / Linux 内存子系统

“多少路组相连”描述的是：每个缓存组中有几个缓存块可以存放数据。

### 9.7.1 Core i7 地址翻译

已分配了的页关联的页表都驻留在内存中，CR3控制寄存器指向第一级页表的起始位置。

MMU 翻译一个虚拟地址时，还会更新两个内核缺页处理程序会用到的位。

- 引用位，内核用来实现页替换算法；
- 修改位，告诉内核在复制替换页之前是否必须写回牺牲页。

### 9.7.2 Linux 虚拟内存子系统

一个虚拟内存系统需要硬件和内核软件之间的紧密协作。

我们需要了解一个实际的操作系统是如何组织虚拟内存，以及如何处理缺页的。

内核虚拟内存包含内核中的代码和数据结构。
内核虚拟内存的某些区域被映射到所有进程共享的物理页面，还有一些区域包含每个进程都不一样的数据。

#### 1. Linux 虚拟内存区域

Linux 将虚拟内存组织为一些区域(段)的集合。

每个存在的虚拟页面都保存在某个区域中，不属于某个区域的页是不存在的，并且不能被进程引用。
区域的概念允许虚拟地址空间有间隙。

task_struct -> mm_struct -> pdg mmap mm_rb -> vm_area_struct

#### 2. Linux 缺页异常处理

缺页异常处理程序需要完成：

- 虚拟地址 A 是合法的吗？A 是否在某个 vma 内？段错误就产生于此；
- 试图进行的内存访问是否合法？这是一个权限问题，进程可能在此被终止；
- 如果是正常合法的缺页，那么就选择牺牲页面，必要时先将其换出，然后换入需要的页面，并更新页表。

## 9.8 内存映射

Linux将一个虚拟内存区域与一个磁盘上的对象关联起来，以初始化这个虚拟内存区域的内容，这个过程就是内存映射（memory mapping）。

虚拟内存区域可以映射到2种对象上：

- 文件系统中的普通文件，一个区域可以映射到文件中连续的一部分；如果区域比文件大，余下部分用0填充。
- 匿名文件，匿名文件是内核创建的，里面全是0。demand-zero page

mapping的大小由用户指定，但是内核时按页处理的，会发生向上对齐。

内存映射提供了一种简单高效的将数据加载到内存中的方法。

一个对象被映射到虚拟内存区域中，要么是共享的，要么是私有的。
共享的对象一般是只读的。？？、
私有对象使用一种写时复制的技术映射到虚拟内存区域中。

## 9.9 动态内存分配

当程序运行时需要更多内存时，就需要使用动态分配器。

分配器将堆视作一组不同大小的块的集合来维护，每个块都是一个chunk，要么是已分配的，要么是空闲的。

分配器必须对齐块，使得他们可以保存任意类型的对象。
分配器只能操作或改变空闲块。

内部碎片是已分配的块比有效载荷大的时候产生的。

外部碎片是空闲内存总计起来足够一个新的分配请求，但是没有一个单独的空闲块足够满足请求。
