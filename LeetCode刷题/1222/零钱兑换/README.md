LeetCode第322题是《零钱兑换》（Coin Change），题目要求给定不同面额的硬币和一个总金额 `amount`，找出最少需要的硬币数量来组成这个金额。如果没有任何硬币组合能组成该金额，返回 `-1`。

### 题目描述

给定一个整数数组 `coins` 表示不同面额的硬币，和一个整数 `amount` 表示总金额。你需要找出组成 `amount` 所需的最少硬币个数。如果没有任何组合可以组成 `amount`，返回 `-1`。

**示例 1：**

```plaintext
输入: coins = [1, 2, 5], amount = 11
输出: 3
解释: 11 = 5 + 5 + 1
```

**示例 2：**

```plaintext
输入: coins = [2], amount = 3
输出: -1
```

**示例 3：**

```plaintext
输入: coins = [1], amount = 0
输出: 0
```

**示例 4：**

```plaintext
输入: coins = [1], amount = 2
输出: 2
```

### 解题思路

这是一个典型的 **动态规划** 问题，类似于背包问题。我们的目标是通过最少的硬币组成 `amount`，因此我们可以从小的子问题开始解决，逐步求解更大的问题。

#### 动态规划状态定义：
- 定义一个数组 `dp`，其中 `dp[i]` 表示构成金额 `i` 的最少硬币数。
- 初始化 `dp[0] = 0`，表示零元不需要任何硬币。
- 对于其他金额 `i`，初始值设为一个较大的数（如 `float('inf')`），表示还没有找到解决方案。

#### 状态转移：
- 对于每个金额 `i`，我们尝试用每个硬币 `coin` 更新 `dp[i]`：
  \[
  dp[i] = \min(dp[i], dp[i - coin] + 1)
  \]
  这意味着，如果使用硬币 `coin`，我们可以通过已经计算出的 `dp[i - coin]` 得到 `i` 的最小硬币数，并加 1 表示使用了当前硬币。

#### 最终结果：
- `dp[amount]` 即为所需的最少硬币数。如果 `dp[amount]` 还是初始值 `float('inf')`，说明没有办法用给定的硬币组成该金额，返回 `-1`。

### 代码实现

```python
def coinChange(coins, amount):
    # 初始化 dp 数组，dp[i] 表示金额 i 的最少硬币数
    dp = [float('inf')] * (amount + 1)
    dp[0] = 0  # 0 元不需要任何硬币
    
    # 遍历每个硬币
    for coin in coins:
        # 更新每个金额的最小硬币数
        for i in range(coin, amount + 1):
            dp[i] = min(dp[i], dp[i - coin] + 1)
    
    # 如果 dp[amount] 仍为 inf，说明无法组成该金额
    return dp[amount] if dp[amount] != float('inf') else -1
```

### 代码解析

1. **初始化 `dp` 数组**：
    - `dp[0] = 0`：表示构成金额 `0` 不需要硬币。
    - 对于其他金额 `i`，初始化为 `float('inf')`，表示还没有找到解决方案。

2. **遍历硬币**：
    - 对于每个硬币 `coin`，从 `coin` 到 `amount` 遍历，更新 `dp[i]`。
    - 对于每个金额 `i`，更新 `dp[i]` 为 `dp[i]` 和 `dp[i - coin] + 1` 的较小值，表示使用硬币 `coin` 更新 `dp[i]` 的最小硬币数。

3. **最终结果**：
    - `dp[amount]` 即为最小硬币数。如果 `dp[amount]` 仍为 `float('inf')`，说明不能用给定的硬币组成该金额，返回 `-1`。

### 时间复杂度

- **时间复杂度**：`O(amount * n)`，其中 `amount` 是目标金额，`n` 是硬币的数量。我们对于每个硬币遍历金额，从 `coin` 到 `amount`，因此时间复杂度是 `O(amount * n)`。
- **空间复杂度**：`O(amount)`，我们需要一个大小为 `amount + 1` 的数组 `dp` 来存储每个金额的最少硬币数。

### 示例解析

#### 示例 1：`coins = [1, 2, 5], amount = 11`

1. **初始化**：
    - `dp = [0, inf, inf, inf, inf, inf, inf, inf, inf, inf, inf, inf]`

2. **更新 `dp` 数组**：
    - 使用硬币 `1` 更新：
        - `dp[1] = 1`, `dp[2] = 2`, `dp[3] = 3`, ..., `dp[11] = 11`
    - 使用硬币 `2` 更新：
        - `dp[2] = 1`, `dp[3] = 2`, `dp[4] = 2`, ..., `dp[11] = 6`
    - 使用硬币 `5` 更新：
        - `dp[5] = 1`, `dp[6] = 2`, ..., `dp[11] = 3`

3. **最终结果**：
    - `dp[11] = 3`，即 11 元需要最少 3 个硬币（5 + 5 + 1）。

#### 示例 2：`coins = [2], amount = 3`

1. **初始化**：
    - `dp = [0, inf, inf, inf]`

2. **更新 `dp` 数组**：
    - 使用硬币 `2` 更新：
        - `dp[2] = 1`

3. **最终结果**：
    - `dp[3] = inf`，说明不能用硬币 `2` 组成金额 3，返回 `-1`。

#### 示例 3：`coins = [1], amount = 0`

1. **初始化**：
    - `dp = [0]`

2. **最终结果**：
    - `dp[0] = 0`，即 0 元不需要任何硬币，返回 `0`。

#### 示例 4：`coins = [1], amount = 2`

1. **初始化**：
    - `dp = [0, 1, inf]`

2. **最终结果**：
    - `dp[2] = 2`，即 2 元需要 2 个硬币（1 + 1）。

### 总结

- 通过**动态规划**，我们可以逐步求解每个金额的最小硬币数。
- **状态转移方程**是 `dp[i] = min(dp[i], dp[i - coin] + 1)`，表示对于每个硬币 `coin`，我们更新构成金额 `i` 的最少硬币数。
- 时间复杂度为 `O(amount * n)`，空间复杂度为 `O(amount)`，适用于较大的金额和硬币数组。
