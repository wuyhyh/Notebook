LeetCode第153题是《寻找旋转排序数组中的最小值》（Find Minimum in Rotated Sorted Array）。题目要求在一个已经旋转过的升序排序数组中找到并返回最小的元素。

---

### 题目描述

给定一个**无重复元素**的旋转排序数组 `nums`，其中原数组是按照升序排列的。数组中的某个未知的点被旋转过一次，旋转后数组可能包含两个有序的子数组。你的任务是找到并返回数组中的最小元素。

**注意**：数组中的元素是互不相同的。

#### 示例

```plaintext
输入: nums = [3,4,5,1,2]
输出: 1
解释: 原始数组为 [1,2,3,4,5]，旋转后得到 [3,4,5,1,2]。最小的元素是 1。
```

```plaintext
输入: nums = [4,5,6,7,0,1,2]
输出: 0
解释: 原始数组为 [0,1,2,4,5,6,7]，旋转后得到 [4,5,6,7,0,1,2]。最小的元素是 0。
```

```plaintext
输入: nums = [11,13,15,17]
输出: 11
解释: 数组没有旋转，最小的元素是 11。
```

---

### 解题思路

由于数组是由一个升序数组旋转而来，并且没有重复元素，我们可以利用**二分查找**来高效地找到最小元素。二分查找的时间复杂度为 `O(log n)`，远优于线性搜索的 `O(n)`。

#### 关键观察

1. **有序性**：虽然数组被旋转了，但数组中至少有一部分仍然是有序的。
2. **最小元素的特点**：最小元素是数组中唯一一个比其前一个元素小的元素。如果数组没有旋转，最小元素就是第一个元素。
3. **二分查找策略**：
    - 比较中间元素与右边界元素。
    - 如果中间元素大于右边界元素，说明最小元素在右半部分。
    - 否则，最小元素在左半部分（包括中间元素）。

### 详细步骤

1. **初始化指针**：
    - `left = 0`
    - `right = len(nums) - 1`

2. **二分查找循环**：
    - 当 `left < right` 时，执行以下步骤：
        - 计算中间位置 `mid = left + (right - left) // 2`
        - 比较 `nums[mid]` 与 `nums[right]`：
            - 如果 `nums[mid] > nums[right]`，说明最小元素在 `mid + 1` 到 `right` 之间，更新 `left = mid + 1`
            - 否则，说明最小元素在 `left` 到 `mid` 之间，更新 `right = mid`

3. **结束条件**：
    - 当 `left == right` 时，循环结束，此时 `nums[left]` 就是最小元素。

### 代码实现（Python）

```python
def findMin(nums):
    left, right = 0, len(nums) - 1
    
    while left < right:
        mid = left + (right - left) // 2
        if nums[mid] > nums[right]:
            # 最小元素在 mid + 1 到 right 之间
            left = mid + 1
        else:
            # 最小元素在 left 到 mid 之间
            right = mid
    return nums[left]
```

### 复杂度分析

- **时间复杂度**：`O(log n)`。每次迭代都将搜索范围缩小一半。
- **空间复杂度**：`O(1)`。只使用了常数级别的额外空间。

### 例子解析

以 `nums = [4,5,6,7,0,1,2]` 为例：

1. **第一次迭代**：
    - `left = 0`, `right = 6`
    - `mid = 3`, `nums[mid] = 7`, `nums[right] = 2`
    - 由于 `7 > 2`，最小元素在 `mid + 1 = 4` 到 `right = 6` 之间，更新 `left = 4`

2. **第二次迭代**：
    - `left = 4`, `right = 6`
    - `mid = 5`, `nums[mid] = 1`, `nums[right] = 2`
    - 由于 `1 <= 2`，最小元素在 `left = 4` 到 `mid = 5` 之间，更新 `right = 5`

3. **第三次迭代**：
    - `left = 4`, `right = 5`
    - `mid = 4`, `nums[mid] = 0`, `nums[right] = 1`
    - 由于 `0 <= 1`，最小元素在 `left = 4` 到 `mid = 4` 之间，更新 `right = 4`

4. **循环结束**：
    - `left = 4`, `right = 4`
    - 返回 `nums[4] = 0`

最终，最小元素是 `0`。

---

### 注意事项

- **未旋转的数组**：如果数组没有被旋转，最小元素就是第一个元素。上述算法同样适用。
- **数组长度为1**：如果数组只有一个元素，直接返回该元素。
- **无重复元素**：题目保证数组中没有重复元素，这简化了比较逻辑。如果存在重复元素，算法需要做适当调整。

通过上述方法，我们可以高效地在旋转排序数组中找到最小的元素，满足题目的要求。
