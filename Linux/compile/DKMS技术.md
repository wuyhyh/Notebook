你截图中提到的 **DKMS** 是一个在 Linux 下非常重要的工具，特别是涉及到**第三方驱动模块自动编译和安装**的时候。下面是详细解释：

---

## 🧩 什么是 DKMS？

**DKMS（Dynamic Kernel Module Support）** 全称为：**动态内核模块支持系统**。

它的作用是：

> **当你的内核升级或重新编译时，自动重新编译并安装你安装的第三方内核模块。**

---

## 📦 举例场景

假设你使用 Fedora 或 Ubuntu：

* 你安装了第三方驱动，比如显卡驱动、无线网卡驱动，这些驱动作为内核模块存在；
* 你后来升级了内核（例如从 `6.5.x` 升级到 `6.6.x`）；
* 原来的模块无法直接使用，因为它们是针对旧内核编译的；
* 如果你没有 DKMS，你需要**手动**为新内核重新编译这些模块；
* 有了 DKMS，**它会自动识别内核变动，自动重新构建这些模块**。

---

## 🔧 DKMS 的工作流程图

1. 安装模块（如 `rtl8821ce` 驱动）：

   ```bash
   sudo dkms add ./rtl8821ce
   sudo dkms build rtl8821ce/1.0
   sudo dkms install rtl8821ce/1.0
   ```

2. 当你升级内核并重启：

    * DKMS 自动检测到新的内核版本；
    * 自动重新构建并安装 `rtl8821ce` 驱动；
    * 模块可用，无需用户干预。

---

## 📂 DKMS 使用时依赖什么？

* 需要有：

    * `/usr/src/kernels/$(uname -r)/`（也就是 `kernel-devel` 提供的内容）
    * 编译工具链（如 `gcc`, `make` 等）

---

## ✅ 常见支持 DKMS 的驱动：

| 驱动名              | 功能                 |
|------------------|--------------------|
| `nvidia-dkms`    | NVIDIA 显卡驱动        |
| `rtl8821ce-dkms` | Realtek 无线网卡       |
| `vboxdrv`        | VirtualBox 内核模块    |
| 自定义驱动            | 自己编写的模块也可用 DKMS 管理 |

---

## 📌 总结

| 项  | 说明                                  |
|----|-------------------------------------|
| 名称 | DKMS（Dynamic Kernel Module Support） |
| 作用 | 内核变动时自动重编译并安装第三方模块                  |
| 依赖 | `kernel-devel`、编译工具链                |
| 特点 | 自动化、高可维护性、适合桌面系统或快速开发               |
