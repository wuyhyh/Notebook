TFTP 和 NFS 都是用于在网络中传输文件的协议，在嵌入式系统和 U-Boot 中非常常见，尤其是在启动阶段加载内核、设备树、根文件系统时经常用到。下面分别解释：

---

## 一、TFTP（Trivial File Transfer Protocol）

### ✅ 简介：

TFTP 是 **简单文件传输协议**，基于 UDP 协议工作，适合在受控的局域网中传输文件。

### ✅ 特点：

* **轻量级**：没有复杂的身份验证或目录浏览功能。
* **无连接**：使用 UDP，适合简单快速的传输。
* **常用于嵌入式启动阶段**：从服务器下载内核镜像（如 `uImage`、`zImage`）、设备树（`*.dtb`）或 U-Boot 本身。

### ✅ 使用例子（在 U-Boot 中）：

```bash
setenv serverip 192.168.1.100    # TFTP 服务器 IP
setenv ipaddr 192.168.1.10       # 本设备 IP
tftpboot 0x82000000 uImage       # 从服务器下载 uImage 到内存地址 0x82000000
```

---

## 二、NFS（Network File System）

### ✅ 简介：

NFS 是 **网络文件系统协议**，允许一台机器通过网络挂载远程服务器上的文件系统，就像访问本地磁盘一样。

### ✅ 特点：

* 基于 TCP/UDP。
* 支持完整的文件系统操作：读、写、目录遍历等。
* 通常用来挂载 **根文件系统（rootfs）**，便于调试和开发。

### ✅ 使用例子（Linux 内核启动参数中）：

当 U-Boot 启动 Linux 内核时，可以通过参数告诉内核使用 NFS 挂载根文件系统：

```bash
setenv bootargs root=/dev/nfs nfsroot=192.168.1.100:/nfs/rootfs ip=dhcp
```

表示：内核启动后用 `192.168.1.100:/nfs/rootfs` 作为根文件系统，并通过 DHCP 获取 IP。

---

## 三、总结对比：

| 协议   | 用途          | 优点          | 缺点          |
|------|-------------|-------------|-------------|
| TFTP | 简单下载文件（如内核） | 轻量，易配置      | 无安全性，功能简单   |
| NFS  | 挂载远程整个文件系统  | 支持读写操作，开发方便 | 配置略复杂，安全性有限 |
