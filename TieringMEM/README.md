## 🎯 为什么这是一个困难点？

### 1. **页面状态多样、不可直接操作**

- **不是所有页面都允许迁移**，如：
    - 内核锁页（mlocked）
    - 非LRU页（PageLRU=0）
    - 不可回收页（PageUnevictable=1）
- **误操作易造成系统崩溃**，如访问 page cache 过程中 unmap 页表时死锁或 soft lockup。

### 2. **页面可能仍在使用中**

- 用户进程正在访问该页面；
- 映射未解除，会产生并发修改页表的风险；
- 必须确保**unmap 成功**、页表和TLB已同步更新后才能迁移。

### 3. **大页（如2M）存在分裂和碎片化问题**

- 无法整体迁移时，需 fallback 拆分为 4K 页；
- 拆分后容易引发后续页面重组失败，导致性能下降。

---

## 如何解决（工程实现层面）

### 1. 引入 **页面过滤策略**

```c++
if (PageLRU(page) && page_mapped(page) && !PageUnevictable(page)) {
    list_add(&page->lru, &src_list);
}
```

- `PageLRU()`：确保页面在回收链中，可由内核回收或迁移；
- `page_mapped()`：至少有一个映射（即页表项有效）；
- `!PageUnevictable()`：可被换出、可迁移页面。

---

### 2. 使用 `try_to_unmap()` + `rmap_walk()` 准确解除映射

- 调用链路：

```
migrate_pages()
 └── unmap_and_move()
      ├── try_to_unmap()
      └── move_to_new_page()
```

- **try_to_unmap**：遍历所有映射该物理页的 `vma`，执行页表项解除；

---

### ✅ 3. 对大页做粒度感知处理

- 热页可保留为 2M，冷页分裂为 4K 后迁移；
- 使用 `split_huge_page()` 保证安全拆分；
- 可加锁 `page->compound_lock` 或使用 `trylock_page()` 避免并发拆分冲突。

## ✅ 改进方向：**从固定阈值识别热页 → 动态自适应热度识别**

---

### 当前实现中的固定阈值机制

```c++
if (pages[i].access_count < migrate_threshold) {
    // 认为是冷页，加入迁移队列
}
```

- `migrate_threshold` 是用户指定的常数，例如 `100`；
- 优点：简单、直观、容易实现；
- 缺点：**不适应负载变化**，**阈值敏感**，在实际部署中容易出现误判：

---

### 现实问题：固定阈值的典型误区

| 场景           | 结果                | 原因     |
|--------------|-------------------|--------|
| 某负载整体访问频率较低  | 所有页都被误判为热页        | 绝对阈值太高 |
| 某负载突发式访问局部页面 | 刚访问过但很快冷却的页被误当作热点 | 无时间感知  |

---

### 改进思路：**动态、自适应的热度判定机制**

#### 1. 相对阈值（基于排序后的热度分布）

```c++
// 排序后，保留前10%的页面为热页
qsort(pages, num_tracked_pages, cmp_by_access);
int cutoff_index = num_tracked_pages * 0.9;
```

- 动态适配不同 workload 的访问强度；
- 具备更强鲁棒性，不需要人为调参。

---

#### 2. 直方图 + 自动分布分段（histogram binning）

- 将所有访问频率归入 N 个区间；
- 选择频率分布突然下降的点作为“热-冷分界线”；
- 类似直方图的“边缘检测”策略。

---

### 🔄 对迁移逻辑的修改示意

```c++
// 设置一个动态阈值而非固定值
unsigned long dynamic_threshold = compute_dynamic_threshold(pages, num_tracked_pages);

for (int i = 0; i < num_tracked_pages; i++) {
    if (pages[i].access_count < dynamic_threshold) {
        page = pfn_to_page(pages[i].pfn);
        if (PageLRU(page) && !PageUnevictable(page)) {
            list_add(&page->lru, &src_list);
        }
    }
}
```

---

### 📌 总结

> 固定阈值虽然简单，但在实际系统中会受 workload 差异与动态行为影响较大。我们通过引入**相对阈值判断**与**直方图分析机制**
> ，实现了页面冷热识别策略的自适应化，有效降低了误迁移率，并增强了系统的稳定性与泛化能力。
