# 第4章 字符设备驱动程序

Linux 基本概念，一切皆文件。

字符设备驱动通过`/dev`目录下的特殊文件公开设备的属性和功能，通过这些文件可以在设备和用户应用程序之间交换数据，也可以通过它来控制实际的物理设备。

字符设备在内核中表示为`struct cdev`实例，定义在`include/linux/cdev.h`中；

```c++
struct cdev{
    struct kobject kobj;
    struct module *owner;
    const struct file_operations *ops;
    struct list_head list;
    dev_t dev;
    unsigned int count;
}
```

- 如何创建、识别和向系统注册设备
- 使用`struct file_operations`定义设备文件方法，内核通过这些方法向用户空间公开设备的功能，这些功能可以通过文件相关的系统调用
  read write open 进行访问。

## 4.1 主设备和次设备的概念

内核使用dev_t类型变量来维持设备号。

分配设备号: `alloc_chrdev_region()`

## 4.2 设备文件操作

可以在文件上执行的操作取决于管理文件的设备驱动程序。这样的操作在内核中定义为`struct file_operations`实例。

其中的回调函数都不是必须的，如果定义了，就可以从用户态调用，如果没有就返回错误。

### 内核中文件的表示

内核把文件描述为`struct inode`的实例。`struct file`代表打开的文件。

同一个文件可以打开多次，有不同的文件描述符，但是他们都指向同一个inode。

## 4.3 分配和注册字符设备

字符设备在内核中表示为`struct cdev`实例。

编写字符设备驱动程序的最终目标是创建并注册与`struct file_operations`关联的`cdev`结构实例，为用户空间提供一组可以在该设备上执行操作的函数。

需要执行4个操作：

- 分配设备号`alloc_chrdev_region()`获取主设备号和次设备号，供内核识别设备。
- 创建设备类 在`/sys/class`中可见，为后续使用 `udev` 创建设备节点提供设备类支持。
- 初始化并注册设备`cdev_init()` 和 `cdev_add()`，将 `file_operations` 绑定到 `struct cdev`
- 在/dev下创建出设备节点 在 `/dev` 下创建对应的设备文件，供用户空间访问。

到此，用户空间就可以通过打开 `/dev/xxx` 来访问驱动程序提供的接口了。

### 对照表：内核接口与目的

| 步骤     | 函数                      | 作用                               |
|--------|-------------------------|----------------------------------|
| 分配设备号  | `alloc_chrdev_region()` | 申请主次设备号                          |
| 初始化设备  | `cdev_init()`           | 初始化 `cdev` 并绑定 `file_operations` |
| 注册设备   | `cdev_add()`            | 将设备注册到内核                         |
| 创建设备类  | `class_create()`        | 创建设备类，提供给 `udev`                 |
| 创建设备节点 | `device_create()`       | 创建 `/dev/xxx` 设备文件               |

## 4.4 写文件操作

### 4.4.1 内核空间和用户空间之间交换数据

将缓冲区的数据从内核空间复制到用户空间，或者从用户空间复制到内核空间

```c++
copy_from_user()
copy_to_user()
```

单值复制

```c++
put_user()
get_user()
```

### 4.4.2 open 方法

open()方法在每次打开设备时调用。通常用这个方法来执行设备和数据结构的初始化。

### 4.4.3 release方法

release()方法在关闭设备时调用，之后必须撤销在open()任务中已经执行的所有操作。

### 4.4.4 write方法

write()方法用于向数据发送数据。

### 4.4.5 read方法

read()方法返回值是读取的数据量。

### 4.4.6 llseek方法

在文件中移动光标位置时，会调用llseek()函数。

### 4.4.7 poll方法

poll() 函数是字符设备驱动里实现异步/非阻塞等待机制的关键一环。

当需要实现被动等待，则必须实现poll()函数。

“被动等待”指的是：用户空间程序不想一直“主动阻塞”等待某个操作完成，而是希望你通知它“设备准备好了”。

典型的用户空间接口是 poll()、select()、epoll()，它们可以等待文件描述符就绪（可读、可写等）。

如果你想让用户空间使用 poll() 这类机制来等待设备事件，你的字符设备驱动就必须实现 file_operations 中的 poll() 方法。

---

select()、poll()、epoll()，它们的核心思想是一致的：
统一等待多个文件描述符上的事件（比如可读、可写、异常等），实现“非阻塞 I/O”或“异步 I/O”模型。

换句话说：程序不用“一个个地 read() 去试”，而是可以“阻塞地等待一组 fd”，一旦有事件就被唤醒。
实现的是事件驱动模型，而不是“每次轮询检查”。

三者使用的数据结构不一样，性能也有差距。

### 4.4.8 ioctl方法

有时设备需要实现系统调用未提供的特定命令。

解决方案是使用输入输出控制`ioctl`，这种方法可以扩展与设备相关的系统调用列表(实际上的命令)。

为了安全和有效，ioctl命令需要由系统唯一的号码来标识。
应该在专用的头文件中生成自己的ioctl编号(命令)，这个头文件在用户空间也要可以使用。
