# 01-vim基础操作与配置文件修改

## 1 文档目标

本文用于帮助实习生掌握 `vim` 的基本使用方法，能够在 Linux 终端中完成简单的文件查看、编辑、保存和配置文件修改。

学习完本文后，应能完成以下任务：

1. 使用 `vim` 打开文件
2. 在文件中移动光标
3. 进入编辑模式并修改内容
4. 保存文件或退出文件
5. 查找文本
6. 修改常见 Linux 配置文件

## 2 vim 是什么

## 2.1 基本概念

`vim` 是 Linux 系统中常用的文本编辑器，常用于修改配置文件、脚本文件、代码文件等。

例如：

```bash
vim /etc/hosts
vim test.c
vim config.txt
```

在服务器、开发板、嵌入式 Linux 环境中，很多时候没有图形界面，因此需要熟练使用 `vim` 进行文件修改。

## 2.2 vim 的常见模式

`vim` 最重要的是理解它的模式。

常用模式有三种：

| 模式 | 作用 |
|---|---|
| 普通模式 | 移动光标、复制、删除、查找 |
| 插入模式 | 输入和修改文本 |
| 命令模式 | 保存、退出、搜索、替换 |

打开文件后，默认进入的是普通模式。

## 3 打开和退出文件

## 3.1 打开文件

```bash
vim 文件名
```

示例：

```bash
vim hello.txt
```

如果文件存在，会打开该文件。

如果文件不存在，保存后会创建新文件。

## 3.2 退出 vim

在普通模式下，输入 `:` 进入命令模式。

常用命令如下：

| 命令 | 作用 |
|---|---|
| `:q` | 退出 |
| `:q!` | 强制退出，不保存 |
| `:w` | 保存 |
| `:wq` | 保存并退出 |
| `:x` | 保存并退出 |
| `:w 文件名` | 另存为指定文件 |

示例：

```text
:wq
```

表示保存并退出。

## 4 普通模式基本操作

## 4.1 光标移动

在普通模式下，可以使用方向键移动光标，也可以使用以下按键：

| 按键 | 作用 |
|---|---|
| `h` | 向左 |
| `j` | 向下 |
| `k` | 向上 |
| `l` | 向右 |

也可以使用更快速的跳转：

| 按键 | 作用 |
|---|---|
| `gg` | 跳到文件开头 |
| `G` | 跳到文件末尾 |
| `数字 + G` | 跳到指定行 |
| `0` | 跳到行首 |
| `$` | 跳到行尾 |

示例：

```text
20G
```

表示跳到第 20 行。

## 4.2 删除内容

| 命令 | 作用 |
|---|---|
| `x` | 删除当前字符 |
| `dd` | 删除当前行 |
| `数字 + dd` | 删除多行 |

示例：

```text
3dd
```

表示删除当前行开始的 3 行。

## 4.3 复制和粘贴

| 命令 | 作用 |
|---|---|
| `yy` | 复制当前行 |
| `数字 + yy` | 复制多行 |
| `p` | 粘贴到当前行下面 |
| `P` | 粘贴到当前行上面 |

示例：

```text
5yy
p
```

表示复制 5 行，并粘贴到当前行下面。

## 4.4 撤销和恢复

| 命令 | 作用 |
|---|---|
| `u` | 撤销 |
| `Ctrl + r` | 恢复撤销 |

## 5 插入模式基本操作

## 5.1 进入插入模式

在普通模式下，可以通过以下按键进入插入模式：

| 按键 | 作用 |
|---|---|
| `i` | 在光标前插入 |
| `a` | 在光标后插入 |
| `o` | 在当前行下面新建一行 |
| `O` | 在当前行上面新建一行 |

最常用的是：

```text
i
```

## 5.2 退出插入模式

编辑完成后，按：

```text
Esc
```

即可回到普通模式。

这是使用 `vim` 时最重要的习惯之一：

**编辑完成后先按 `Esc`，再进行保存、退出、查找等操作。**

## 6 查找文本

## 6.1 向下查找

在普通模式下输入：

```text
/关键字
```

然后按回车。

示例：

```text
/root
```

表示查找 `root`。

## 6.2 查找下一个或上一个

| 按键 | 作用 |
|---|---|
| `n` | 查找下一个 |
| `N` | 查找上一个 |

## 7 显示行号

## 7.1 临时显示行号

在普通模式下输入：

```text
:set number
```

或者简写：

```text
:set nu
```

## 7.2 取消显示行号

```text
:set nonumber
```

或者：

```text
:set nonu
```

## 8 修改配置文件的基本流程

## 8.1 推荐流程

修改 Linux 配置文件时，建议按照以下流程操作：

1. 查看文件内容
2. 备份原文件
3. 使用 `vim` 修改
4. 保存退出
5. 检查修改结果
6. 重启相关服务或重新加载配置

## 8.2 修改前先备份

修改重要配置文件前，先备份：

```bash
cp /etc/hosts /etc/hosts.bak
```

如果修改出错，可以恢复：

```bash
cp /etc/hosts.bak /etc/hosts
```

## 8.3 使用 vim 修改配置文件

示例：

```bash
sudo vim /etc/hosts
```

进入文件后：

1. 按 `/` 查找需要修改的位置
2. 按 `i` 进入插入模式
3. 修改内容
4. 按 `Esc`
5. 输入 `:wq` 保存退出

## 9 常见配置文件修改示例

## 9.1 修改 /etc/hosts

打开文件：

```bash
sudo vim /etc/hosts
```

添加一行：

```text
192.168.1.100 board.local
```

保存退出：

```text
:wq
```

之后可以测试：

```bash
ping board.local
```

## 9.2 修改脚本文件

打开脚本：

```bash
vim test.sh
```

写入内容：

```bash
#!/bin/bash
echo "hello linux"
```

保存后赋予执行权限：

```bash
chmod +x test.sh
```

运行：

```bash
./test.sh
```

## 10 常见问题

## 10.1 不知道现在在哪个模式

直接按：

```text
Esc
```

一般可以回到普通模式。

## 10.2 文件无法保存

如果提示没有权限，可能是因为当前用户没有写权限。

可以退出后使用：

```bash
sudo vim 文件名
```

如果已经修改了很多内容，可以在 vim 中尝试：

```text
:w !sudo tee %
```

但初学者建议退出后重新用 `sudo vim` 打开。

## 10.3 不小心改错了

在普通模式下按：

```text
u
```

可以撤销上一步操作。

如果不想保存任何修改，使用：

```text
:q!
```

强制退出。

## 12 练习任务

## 12.1 创建并编辑文件

创建文件：

```bash
vim vim-test.txt
```

写入以下内容：

```text
hello linux
hello vim
hello embedded
```

保存退出。

## 12.2 查找文本

重新打开文件：

```bash
vim vim-test.txt
```

查找：

```text
/vim
```

## 12.3 修改内容

将：

```text
hello vim
```

修改为：

```text
hello vim editor
```

保存退出。

## 12.4 备份并修改配置文件

选择一个普通测试文件进行备份：

```bash
cp vim-test.txt vim-test.txt.bak
```

然后再次修改 `vim-test.txt`，确认备份文件仍然保留原内容。

## 13 小结

`vim` 的学习重点不是记住所有命令，而是先掌握最常用的操作流程：

```text
打开文件 → 查找位置 → i 编辑 → Esc → :wq 保存退出
```

对于初学者，只要熟练掌握打开、编辑、保存、退出、查找、复制、删除、撤销这些基本操作，就已经可以完成大部分 Linux 配置文件修改任务。
