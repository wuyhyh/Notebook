LeetCode第198题是《打家劫舍》（House Robber），题目要求你在不偷相邻房子的情况下，最大化能偷到的金额。每个房子有一定的金额，你不能连续偷两个相邻的房子。

### 题目描述

你是一个专业的小偷，计划偷窃沿街的房屋，每个房子有一个非负的整数值代表该房子的现金。如果你偷窃相邻的两间房屋，警报会触发。那么，如何在不触发警报的情况下偷窃，最大化你能偷窃的金额呢？

**示例 1：**

```plaintext
输入: nums = [2, 7, 9, 3, 1]
输出: 12
解释: 你应该偷窃第1、3、5间房子的金钱。总金额 = 2 + 9 + 1 = 12。
```

**示例 2：**

```plaintext
输入: nums = [1, 2, 3, 1]
输出: 4
解释: 你应该偷窃第1和第3间房子的金钱。总金额 = 1 + 3 = 4。
```

### 解题思路

这个问题可以使用 **动态规划** 来解决。我们用一个动态规划数组 `dp[i]` 来表示偷窃前 `i` 个房子时能够获得的最大金额。

#### 关键点：
- 如果我们偷窃第 `i` 个房子，那么我们不能偷窃第 `i-1` 个房子，因此最大金额是 `dp[i-2] + nums[i]`。
- 如果我们不偷窃第 `i` 个房子，那么最大金额就是 `dp[i-1]`。
- 所以，状态转移方程是：
  \[
  dp[i] = \max(dp[i-1], dp[i-2] + nums[i])
  \]

#### 边界条件：
- `dp[0]` 表示只有一个房子的情况下，偷窃这个房子的最大金额是 `nums[0]`。
- `dp[1]` 表示前两个房子中的最大偷窃金额，可以选择偷窃较大金额的房子 `max(nums[0], nums[1])`。

#### 最终解：
最终的答案是 `dp[n-1]`，即偷窃前 `n` 个房子时能够获得的最大金额。

### 代码实现

```python
def rob(nums):
    if not nums:
        return 0
    elif len(nums) == 1:
        return nums[0]
    
    # 初始化 dp 数组
    dp = [0] * len(nums)
    dp[0] = nums[0]
    dp[1] = max(nums[0], nums[1])
    
    # 填充 dp 数组
    for i in range(2, len(nums)):
        dp[i] = max(dp[i - 1], dp[i - 2] + nums[i])
    
    return dp[-1]
```

### 代码解析

1. **处理边界情况**：
    - 如果 `nums` 为空，返回 0。
    - 如果 `nums` 只有一个房子，则返回该房子的金额。

2. **动态规划数组 `dp`**：
    - `dp[0]` 表示偷窃第一个房子的最大金额，即 `nums[0]`。
    - `dp[1]` 表示偷窃前两个房子中的最大金额，即 `max(nums[0], nums[1])`。

3. **填充动态规划数组**：
    - 对于每个房子，从第 2 个房子开始，我们使用状态转移方程 `dp[i] = max(dp[i-1], dp[i-2] + nums[i])` 进行计算。

4. **返回结果**：返回 `dp[-1]`，即偷窃所有房子时能够获得的最大金额。

### 时间复杂度

- **时间复杂度**：`O(n)`，其中 `n` 是房子的数量。我们只需要遍历一次 `nums` 数组，进行状态转移计算。
- **空间复杂度**：`O(n)`，需要一个 `dp` 数组来保存每个房子偷窃时的最大金额。

### 优化空间

由于 `dp[i]` 只与 `dp[i-1]` 和 `dp[i-2]` 有关，因此可以只用两个变量来代替 `dp` 数组，优化空间复杂度。

### 优化后的代码

```python
def rob(nums):
    if not nums:
        return 0
    elif len(nums) == 1:
        return nums[0]
    
    prev2 = nums[0]  # dp[i-2]
    prev1 = max(nums[0], nums[1])  # dp[i-1]
    
    for i in range(2, len(nums)):
        current = max(prev1, prev2 + nums[i])
        prev2 = prev1
        prev1 = current
    
    return prev1
```

### 优化后的代码解析

1. 使用两个变量 `prev2` 和 `prev1` 分别代替 `dp[i-2]` 和 `dp[i-1]`。
2. 每次迭代时，计算当前房子的最大偷窃金额 `current`，然后更新 `prev2` 和 `prev1`。
3. 最终返回 `prev1`，即偷窃所有房子时的最大金额。

### 时间复杂度

- **时间复杂度**：`O(n)`，与之前的解法相同。
- **空间复杂度**：`O(1)`，由于我们只使用了两个变量来存储状态，因此空间复杂度优化为常数级别。

### 示例解析

#### 示例 1：`nums = [2, 7, 9, 3, 1]`

1. **初始状态**：
    - `prev2 = 2`，`prev1 = 7`（第一个房子和第二个房子的最大偷窃金额）。

2. **迭代过程**：
    - 第 3 个房子（金额 9），`current = max(7, 2 + 9) = 11`，更新 `prev2 = 7`，`prev1 = 11`。
    - 第 4 个房子（金额 3），`current = max(11, 7 + 3) = 11`，更新 `prev2 = 11`，`prev1 = 11`。
    - 第 5 个房子（金额 1），`current = max(11, 11 + 1) = 12`，更新 `prev2 = 11`，`prev1 = 12`。

3. **最终结果**：返回 `prev1 = 12`。

#### 示例 2：`nums = [1, 2, 3, 1]`

1. **初始状态**：
    - `prev2 = 1`，`prev1 = 2`（第一个房子和第二个房子的最大偷窃金额）。

2. **迭代过程**：
    - 第 3 个房子（金额 3），`current = max(2, 1 + 3) = 4`，更新 `prev2 = 2`，`prev1 = 4`。
    - 第 4 个房子（金额 1），`current = max(4, 2 + 1) = 4`，更新 `prev2 = 4`，`prev1 = 4`。

3. **最终结果**：返回 `prev1 = 4`。

### 总结

- **动态规划** 是解决此类“选择问题”时常用的技术，能够有效避免重复计算并在合理时间内解决问题。
- 通过**空间优化**，可以将空间复杂度从 `O(n)` 优化为 `O(1)`。
