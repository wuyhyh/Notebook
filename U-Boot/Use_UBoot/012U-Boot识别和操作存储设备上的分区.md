这个页面是 U-Boot 官方文档中的一节，标题为 **Partitions**（分区支持），主要内容是讲解 U-Boot 如何识别和操作存储设备上的分区（如
eMMC、SD 卡、硬盘、USB 等），包括分区的识别、访问以及支持的分区类型。

---

## ✅ 页面主要内容总结：

### 一、U-Boot 中的分区支持是什么？

U-Boot 可以识别并访问磁盘上的分区，使你能从某个具体的分区中读取或写入文件，常用于：

* 加载内核、设备树、ramdisk 镜像等启动文件；
* 写入新的固件；
* 操作文件系统（如 FAT/ext4）内的文件。

---

### 二、支持的分区表类型：

U-Boot 支持以下两种常见分区格式：

1. **MBR（Master Boot Record）**

    * 旧式的分区表格式，最多支持 4 个主分区（或 3 个主分区 + 1 个扩展分区）。
    * 常用于旧式 BIOS 系统，嵌入式设备也广泛采用。

2. **GPT（GUID Partition Table）**

    * 较新的分区表格式，支持多个分区、长名称和更大的磁盘。
    * 通常用于 UEFI 系统，U-Boot 也有 GPT 支持。

---

### 三、如何查看分区信息？

可以使用如下命令查看分区：

```bash
part list mmc 0
```

表示查看 **第 0 个 mmc 设备** 的分区列表。

示例输出：

```
Partition Map for MMC device 0  --   Partition Type: DOS

Partition Start LBA End LBA Name
 1          8192   532480  boot
 2        532481  6250000  rootfs
```

---

### 四、如何选择特定分区进行操作？

#### 指定分区的方式（几种形式）：

1. **通过分区号**（如 mmc 0:1）：

   ```bash
   fatls mmc 0:1  # 列出 mmc0 的第 1 分区上的 FAT 文件系统内容
   ```

2. **通过分区名称（如果启用了名字识别功能）**：

   ```bash
   fatload mmc 0:boot uImage 0x82000000
   ```

3. **通过 UUID（GPT 分区唯一标识符）**：
   需要提前设置或读取 GPT UUID 来定位分区。

---

### 五、自动设置环境变量（自动识别 rootfs 分区）：

U-Boot 可以自动识别分区并设置一些环境变量，例如：

* `bootpartition`
* `rootfsuuid`

这些变量可用于后续启动脚本中动态指定启动分区。

---

### 六、使用 GPT 分区的建议：

* 如果你使用的是大容量存储设备（如 >2TB 的硬盘或高端 eMMC），推荐使用 GPT。
* 可通过 `gpt write` 命令在 U-Boot 中写入 GPT 分区表（需要提前定义 GPT layout）。
