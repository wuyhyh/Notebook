# 01-磁盘分区挂载与文件系统基础

## 1 文档目标

本文用于帮助实习生理解 Linux 中磁盘、分区、文件系统和挂载的基本概念，能够完成常见的磁盘查看、分区识别、文件系统查看、手动挂载和卸载操作。

学习完本文后，应能完成以下任务：

1. 查看系统中的磁盘和分区
2. 理解磁盘、分区、文件系统、挂载点之间的关系
3. 查看磁盘空间使用情况
4. 手动挂载和卸载分区
5. 理解格式化和分区操作的风险
6. 初步理解 `/etc/fstab` 的作用

## 2 基础概念

## 2.1 磁盘是什么

磁盘是用于存储数据的块设备。

常见磁盘类型：

| 类型 | 常见设备名 |
|---|---|
| SATA/SCSI 磁盘 | `/dev/sda`、`/dev/sdb` |
| NVMe SSD | `/dev/nvme0n1` |
| USB 存储设备 | `/dev/sda`、`/dev/sdb` |
| eMMC | `/dev/mmcblk0` |

注意：设备名不是固定不变的，不同机器、不同启动顺序下可能变化。

## 2.2 分区是什么

一个磁盘可以划分为多个分区。

例如 NVMe 磁盘：

```text
/dev/nvme0n1      整块磁盘
/dev/nvme0n1p1    第 1 个分区
/dev/nvme0n1p2    第 2 个分区
/dev/nvme0n1p3    第 3 个分区
```

SATA 磁盘示例：

```text
/dev/sda      整块磁盘
/dev/sda1     第 1 个分区
/dev/sda2     第 2 个分区
```

## 2.3 文件系统是什么

分区创建后，还需要格式化为文件系统，才能正常存放文件。

常见文件系统：

| 文件系统 | 常见用途 |
|---|---|
| `ext4` | Linux 常用文件系统 |
| `xfs` | 服务器常用文件系统 |
| `vfat` | U 盘、EFI 分区常见 |
| `swap` | 交换分区 |

简单理解：

```text
磁盘 → 分区 → 文件系统 → 挂载到目录 → 访问文件
```

## 2.4 挂载是什么

Linux 中不能直接通过盘符访问分区，而是需要把分区挂载到某个目录。

例如：

```bash
sudo mount /dev/nvme0n1p2 /mnt
```

表示把 `/dev/nvme0n1p2` 这个分区挂载到 `/mnt` 目录。

挂载后，访问 `/mnt` 就是在访问该分区里的文件。

## 3 查看磁盘和分区

## 3.1 使用 lsblk 查看块设备

```bash
lsblk
```

常见输出：

```text
NAME        MAJ:MIN RM  SIZE RO TYPE MOUNTPOINT
nvme0n1     259:0    0 238G  0 disk
├─nvme0n1p1 259:1    0   1G  0 part /boot
├─nvme0n1p2 259:2    0 100G  0 part /
└─nvme0n1p3 259:3    0 100G  0 part /data
```

常见字段：

| 字段 | 含义 |
|---|---|
| `NAME` | 设备名称 |
| `SIZE` | 设备或分区大小 |
| `TYPE` | 类型，disk 表示磁盘，part 表示分区 |
| `MOUNTPOINT` | 当前挂载点 |

## 3.2 查看文件系统类型

```bash
lsblk -f
```

该命令可以查看文件系统类型、UUID 和挂载点。

常见输出中可能看到：

```text
NAME        FSTYPE LABEL UUID                                 MOUNTPOINT
nvme0n1p1   ext4         xxxx-xxxx                            /boot
nvme0n1p2   ext4         yyyy-yyyy                            /
```

## 3.3 使用 fdisk 查看分区表

```bash
sudo fdisk -l
```

该命令会显示磁盘大小、分区表类型和分区信息。

注意：本文只要求会查看，不要求初学者直接用 `fdisk` 修改分区。

## 4 查看磁盘空间

## 4.1 查看文件系统空间使用情况

```bash
df -h
```

常见输出：

```text
Filesystem      Size  Used Avail Use% Mounted on
/dev/nvme0n1p2  100G   30G   70G  30% /
```

常见字段：

| 字段 | 含义 |
|---|---|
| `Filesystem` | 文件系统来源 |
| `Size` | 总容量 |
| `Used` | 已使用 |
| `Avail` | 可用容量 |
| `Use%` | 使用率 |
| `Mounted on` | 挂载点 |

## 4.2 查看目录占用空间

```bash
du -sh 目录名
```

示例：

```bash
du -sh /var/log
```

查看当前目录下各项目大小：

```bash
du -h --max-depth=1
```

如果目录较大，该命令可能执行较慢。

## 5 手动挂载和卸载

## 5.1 创建挂载点

挂载点本质上是一个普通目录。

```bash
sudo mkdir -p /mnt/test
```

## 5.2 挂载分区

```bash
sudo mount /dev/nvme0n1p3 /mnt/test
```

挂载后查看：

```bash
df -h
ls /mnt/test
```

## 5.3 卸载分区

```bash
sudo umount /mnt/test
```

也可以按设备卸载：

```bash
sudo umount /dev/nvme0n1p3
```

注意：命令是 `umount`，不是 `unmount`。

## 5.4 卸载失败怎么办

如果提示设备忙：

```text
target is busy
```

说明当前有进程正在使用该挂载点。

可以先离开该目录：

```bash
cd /
```

再尝试卸载：

```bash
sudo umount /mnt/test
```

也可以查看谁在使用：

```bash
sudo lsof +f -- /mnt/test
```

或者：

```bash
sudo fuser -vm /mnt/test
```

## 6 格式化文件系统

## 6.1 格式化是什么

格式化会在分区上创建新的文件系统。

危险：格式化会清空原分区中的数据。

执行格式化前必须确认：

1. 目标设备是否正确
2. 该分区是否已经备份
3. 是否误选了系统盘
4. 是否正在被挂载使用

## 6.2 格式化为 ext4

```bash
sudo mkfs.ext4 /dev/nvme0n1p3
```

再次强调：该命令会清空 `/dev/nvme0n1p3` 上原有数据。

## 6.3 查看格式化结果

```bash
lsblk -f
```

确认目标分区的 `FSTYPE` 是否变为 `ext4`。

## 7 /etc/fstab 基础

## 7.1 fstab 是什么

`/etc/fstab` 用于配置系统启动时自动挂载哪些文件系统。

查看文件：

```bash
cat /etc/fstab
```

常见格式：

```text
UUID=xxxx-xxxx  /data  ext4  defaults  0  2
```

字段含义：

| 字段 | 含义 |
|---|---|
| `UUID=xxxx` | 文件系统 UUID |
| `/data` | 挂载点 |
| `ext4` | 文件系统类型 |
| `defaults` | 默认挂载参数 |
| `0` | dump 备份参数，通常为 0 |
| `2` | fsck 检查顺序 |

## 7.2 查看 UUID

```bash
lsblk -f
```

或者：

```bash
blkid
```

推荐在 `/etc/fstab` 中使用 UUID，而不是 `/dev/sda1` 这类设备名。

因为设备名可能变化，UUID 通常更稳定。

## 7.3 测试 fstab 配置

修改 `/etc/fstab` 后，不要直接重启。

应该先测试：

```bash
sudo mount -a
```

如果没有报错，再使用：

```bash
df -h
```

确认是否挂载成功。

## 8 常见排查流程

## 8.1 看不到磁盘

先查看块设备：

```bash
lsblk
```

再查看内核日志：

```bash
dmesg | grep -i -E "nvme|sd|mmc|usb"
```

如果是 NVMe 设备，可以查看：

```bash
dmesg | grep -i nvme
```

## 8.2 分区没有挂载

查看分区和挂载点：

```bash
lsblk
```

查看文件系统：

```bash
lsblk -f
```

手动挂载测试：

```bash
sudo mount /dev/分区名 /mnt/test
```

## 8.3 磁盘空间满了

查看文件系统空间：

```bash
df -h
```

查看目录占用：

```bash
du -h --max-depth=1 /
```

如果是日志目录过大，可以查看：

```bash
du -sh /var/log
```

## 8.4 fstab 写错导致挂载失败

先查看 `/etc/fstab`：

```bash
cat /etc/fstab
```

测试挂载：

```bash
sudo mount -a
```

如果报错，根据提示检查 UUID、挂载点、文件系统类型是否正确。

## 9 常见问题

## 9.1 /dev/sda 和 /dev/sda1 有什么区别

| 名称 | 含义 |
|---|---|
| `/dev/sda` | 整块磁盘 |
| `/dev/sda1` | 该磁盘上的第 1 个分区 |

一般挂载的是分区，例如 `/dev/sda1`，不是整块磁盘 `/dev/sda`。

## 9.2 mount 后原目录内容不见了

挂载后，挂载点目录会显示被挂载分区中的内容。

原来目录里的内容并没有删除，只是被临时遮住了。

卸载后会重新显示原目录内容：

```bash
sudo umount /mnt/test
```

## 9.3 为什么不建议随便 mkfs

`mkfs` 会重新创建文件系统，原有数据通常会丢失。

初学者在执行 `mkfs` 前必须让负责人确认设备名。

## 9.4 为什么推荐使用 UUID

`/dev/sda1` 这类设备名可能随着硬盘顺序变化而变化。

UUID 是文件系统的唯一标识，更适合写入 `/etc/fstab`。

## 10 必须掌握的命令总结

| 命令 | 作用 |
|---|---|
| `lsblk` | 查看磁盘和分区 |
| `lsblk -f` | 查看文件系统类型、UUID 和挂载点 |
| `sudo fdisk -l` | 查看磁盘分区表 |
| `df -h` | 查看文件系统空间使用情况 |
| `du -sh 目录名` | 查看目录占用空间 |
| `sudo mkdir -p /mnt/test` | 创建挂载点 |
| `sudo mount /dev/分区名 /mnt/test` | 手动挂载分区 |
| `sudo umount /mnt/test` | 卸载分区 |
| `sudo mkfs.ext4 /dev/分区名` | 格式化为 ext4，危险操作 |
| `blkid` | 查看块设备 UUID |
| `cat /etc/fstab` | 查看自动挂载配置 |
| `sudo mount -a` | 测试 fstab 自动挂载配置 |

## 12 小结

磁盘使用的基本链路是：

```text
磁盘 → 分区 → 文件系统 → 挂载点 → 文件访问
```

日常排查时，优先使用：

```text
lsblk 看设备
lsblk -f 看文件系统和 UUID
df -h 看空间
mount/umount 做临时挂载测试
journalctl/dmesg 看异常日志
```

实习生需要特别记住：

1. `mkfs` 是危险操作，会清空数据
2. `fdisk`、`parted` 修改分区前必须确认目标磁盘
3. 挂载的是分区，不是随便挂整块磁盘
4. 修改 `/etc/fstab` 后先执行 `sudo mount -a` 测试，不要直接重启
