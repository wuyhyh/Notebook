## 🧭 回溯算法的核心思想

我们可以把回溯算法类比为：

> **“走迷宫 + 探索所有可能的路径”**。

每次你做了一个选择，就相当于“往某个方向走了一步”。  
但是你并不知道这是不是唯一的正确路，所以你要：

1. **记住你走了哪一步**（加入 path）
2. **走下去看看会发生什么**（递归）
3. **如果走完了，回头撤销这个选择，尝试别的方向**（pop）

---

## 🧪 举个简单的例子：`s = "aab"`

让我们用一张图说明你为什么需要回退（pop）：

```
初始：[]

第一步选了 "a"：
→ ["a"]

第二步再选 "a"：
→ ["a", "a"]

第三步选 "b"：
→ ["a", "a", "b"] ✅ 是一个结果，加入

此时我们要：
↩️ 回退，撤销 "b" → ["a", "a"]
↩️ 再撤销 "a" → ["a"]
然后尝试：
→ ["a", "ab"] ❌ 不行，再撤销 "a" → []

然后尝试：
→ ["aa"]
→ ["aa", "b"] ✅ 是另一个结果
```

👉 **如果你不撤销选择：**

假如你不执行 `path.pop()`，路径会变成：

```
["a", "a", "b"] → ["a", "a", "b", "b"] → ["a", "a", "b", "b", "b"] ...
```

这样就永远走不回来，也无法探索其他路径了！**你只会一直沿着一条路走到底，而不是探索所有可能的路径。**

---

## ✅ 所以总结来说：

### ❓为什么要回退（`path.pop()`）？

| 原因         | 解释                         |
|------------|----------------------------|
| 1. 探索所有可能  | 回溯算法要尝试所有路径，不撤销就不能尝试别的     |
| 2. 回到上一个状态 | 就像走迷宫要原路返回，才能试另一条路         |
| 3. 保持路径正确  | 如果不 pop，`path` 会带着旧数据干扰新尝试 |
| 4. 控制递归分支  | 每次递归只负责当前的一个选择，回退后再试别的     |

---

## 🔍 简化理解：

```python
path.append(choice)  # 做一个选择
backtrack(...)  # 往下走
path.pop()  # 撤销这个选择，回到上一步
```

这个模板就是回溯算法的基础，非常常见！
