LeetCode 第 189 题：**旋转数组（Rotate Array）**，是一个关于数组元素位置变动的问题，要求你将一个数组中的元素向右旋转 `k` 个位置。

### 问题描述

给定一个数组 `nums`，和一个整数 `k`，你需要将该数组中的元素向右旋转 `k` 个位置。

### 示例

**输入:**
```plaintext
nums = [1, 2, 3, 4, 5, 6, 7], k = 3
```

**输出:**
```plaintext
[5, 6, 7, 1, 2, 3, 4]
```

**解释:**
- 向右旋转 3 次，数组变为 `[5, 6, 7, 1, 2, 3, 4]`。

### 思路分析

要解决这个问题，我们可以利用几种不同的方法。我们将讨论几种常见的解法。

#### 解法 1：暴力法（逐个移动元素）

最直观的方法是将每个元素移动到它的新位置。具体步骤是：
1. 将数组的最后 `k` 个元素移动到数组的前面。
2. 然后将剩下的元素向右移动。

**时间复杂度**：`O(n * k)`，其中 `n` 是数组的大小，`k` 是旋转的次数。

**空间复杂度**：`O(1)`，这是一个原地操作。

#### 解法 2：使用额外的数组（存储结果）

1. 创建一个新数组 `new_nums`，它的大小和原数组相同。
2. 遍历原数组，将每个元素按新的索引位置复制到新数组中。
3. 将新数组赋值回原数组。

这种方法的时间复杂度是 `O(n)`，但是需要额外的空间 `O(n)` 来存储新数组。

#### 解法 3：反转法（最优解法）

我们可以通过 **反转** 数组来进行旋转，反转法是最优的解法。

**步骤**：
1. **反转整个数组**：先将整个数组反转，这样数组的顺序就变成了我们需要的“倒序”形式。
2. **反转前 `k` 个元素**：将数组前 `k` 个元素反转回正确的顺序。
3. **反转剩余的 `n-k` 个元素**：最后，将剩余的部分反转回正确的顺序。

这样，最终我们就得到了旋转 `k` 次的数组。

**时间复杂度**：`O(n)`，其中 `n` 是数组的长度。
**空间复杂度**：`O(1)`，这是一个原地操作。

### 代码实现（使用反转法）

```python
def rotate(nums, k):
    # 处理 k 超过数组长度的情况
    n = len(nums)
    k = k % n  # k 可能会大于 n，因此取 k % n

    # 1. 反转整个数组
    nums.reverse()

    # 2. 反转前 k 个元素
    nums[:k] = reversed(nums[:k])

    # 3. 反转后 n-k 个元素
    nums[k:] = reversed(nums[k:])
```

### 代码解释

1. **处理 `k` 超过数组长度的情况**：
   ```python
   k = k % n
   ```
    - 如果 `k` 大于数组长度 `n`，我们可以利用 `k % n` 来确保旋转次数不超过数组长度。例如，旋转 7 次数组 `[1, 2, 3, 4, 5]`，其实和旋转 2 次效果是一样的。

2. **反转整个数组**：
   ```python
   nums.reverse()
   ```
    - 我们首先将整个数组反转。这样，数组的顺序就变成了逆序。

3. **反转前 `k` 个元素**：
   ```python
   nums[:k] = reversed(nums[:k])
   ```
    - 接着，我们将前 `k` 个元素反转，使得它们回到正确的顺序。

4. **反转剩下的部分**：
   ```python
   nums[k:] = reversed(nums[k:])
   ```
    - 最后，我们将剩下的 `n-k` 个元素反转，使得它们也回到正确的顺序。

---

### 例子分析

假设 `nums = [1, 2, 3, 4, 5, 6, 7]`，`k = 3`：

1. **反转整个数组**：
    - 原始数组：`[1, 2, 3, 4, 5, 6, 7]`
    - 反转后：`[7, 6, 5, 4, 3, 2, 1]`

2. **反转前 `k=3` 个元素**：
    - 反转前 3 个元素：`[7, 6, 5]` 变为 `[5, 6, 7]`
    - 数组变为：`[5, 6, 7, 4, 3, 2, 1]`

3. **反转后 `n-k=4` 个元素**：
    - 反转后 4 个元素：`[4, 3, 2, 1]` 变为 `[1, 2, 3, 4]`
    - 数组最终变为：`[5, 6, 7, 1, 2, 3, 4]`

这就是我们最终的旋转结果。

---

### 时间复杂度与空间复杂度

- **时间复杂度**：
    - 反转数组的时间复杂度是 `O(n)`，其中 `n` 是数组的长度。
    - 由于我们只进行三次反转操作，因此总时间复杂度是 `O(n)`。

- **空间复杂度**：
    - 反转操作是原地操作，不需要额外的空间，因此空间复杂度是 `O(1)`。

---

### 总结

- 使用 **反转法** 是解决旋转数组问题的最优解法。
- 通过三次反转，我们可以高效地将数组旋转 `k` 次，并且实现了原地操作，节省了空间。
- 这种方法的时间复杂度是 `O(n)`，空间复杂度是 `O(1)`，非常适合处理大规模数据。
