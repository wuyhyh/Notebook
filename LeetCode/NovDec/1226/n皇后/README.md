LeetCode第51题是《N 皇后》（N-Queens），这是一个经典的回溯问题，要求在一个 `N x N` 的棋盘上放置 `N` 个皇后，使得它们彼此之间不能攻击对方。即：每个皇后只能放在一个格子里，并且任何两个皇后不能处于同一行、同一列、或同一对角线上。

### 题目描述

给定一个整数 `n`，返回所有不同的 `n` 皇后问题的解决方案。

每个解决方案包含一个 `n x n` 的棋盘，其中 `'.'` 表示一个空位，`'Q'` 表示一个皇后。

#### 示例 1：

```plaintext
输入：n = 4
输出：[[".Q..","...Q","Q...","..Q."],["..Q.","Q...","...Q",".Q.."]]
解释：如上所示，返回 [ ["...Q", "Q...", "....", "...Q"] ] 是其中之一。
```

#### 示例 2：

```plaintext
输入：n = 1
输出：[["Q"]]
```

### 解题思路

这个问题是一个典型的回溯问题，我们要尝试在棋盘上放置每个皇后，并且逐行进行尝试。每次放置皇后时，我们要检查是否存在攻击关系，即是否在同一列、同一对角线上。

#### 步骤

1. **递归思路**：从第一行开始，尝试将皇后放置在不同的列中，并递归到下一行。

2. **剪枝优化**：
    - **列冲突**：我们可以用一个数组 `cols` 来记录每一列是否已经被占用。
    - **主对角线冲突**：主对角线的标记可以用 `diag1` 数组来表示，`diag1[i - j]` 标记 `i` 行 `j` 列的主对角线是否被占用。
    - **副对角线冲突**：副对角线的标记可以用 `diag2` 数组来表示，`diag2[i + j]` 标记 `i` 行 `j` 列的副对角线是否被占用。

3. **回溯**：在每一行，我们尝试将皇后放置在每一个位置，如果放置后没有冲突，就递归到下一行。如果到达最后一行，说明成功放置了所有皇后，记录这个解。

4. **解的存储**：每次递归到最后一行并成功放置皇后时，我们将当前的棋盘配置添加到结果中。

### 代码实现

```python
def solveNQueens(n):
    def backtrack(row, cols, diag1, diag2, board):
        # 如果已经放置完所有的皇后
        if row == n:
            result.append(["".join(row) for row in board])
            return

        for col in range(n):
            # 检查列、主对角线、副对角线是否有冲突
            if col in cols or (row - col) in diag1 or (row + col) in diag2:
                continue

            # 做选择：将当前列、主对角线、副对角线标记为已占用
            cols.add(col)
            diag1.add(row - col)
            diag2.add(row + col)

            # 放置皇后并继续处理下一行
            board[row][col] = 'Q'
            backtrack(row + 1, cols, diag1, diag2, board)

            # 撤销选择
            board[row][col] = '.'
            cols.remove(col)
            diag1.remove(row - col)
            diag2.remove(row + col)

    result = []
    board = [['.' for _ in range(n)] for _ in range(n)]  # 初始化棋盘
    cols = set()  # 记录已占用的列
    diag1 = set()  # 记录已占用的主对角线
    diag2 = set()  # 记录已占用的副对角线
    backtrack(0, cols, diag1, diag2, board)
    return result
```

### 代码解析

1. **`backtrack` 函数**：递归函数负责回溯，尝试在当前行放置皇后，并递归到下一行。每当找到一个有效的解，就将其加入 `result` 中。

2. **列、主对角线和副对角线的标记**：
    - `cols` 用来标记每一列是否已经有皇后。
    - `diag1` 用来标记每条主对角线（从左上到右下）是否已经有皇后。
    - `diag2` 用来标记每条副对角线（从右上到左下）是否已经有皇后。

3. **递归过程**：
    - 在每一行中，我们尝试将皇后放置在每一列。
    - 如果某个位置符合条件（即没有与其他皇后冲突），我们放置皇后并递归到下一行。
    - 如果找到一个有效解（当 `row == n` 时），我们将当前棋盘的配置加入到 `result` 中。
    - 每次递归结束后，撤销当前选择，并继续尝试下一个列位置。

4. **`board`**：棋盘是一个二维列表，用来存储当前的皇后放置情况。每一行使用 `'.'` 表示空位，`'Q'` 表示皇后。

### 时间复杂度

- **时间复杂度**：`O(N!)`。因为每一行最多有 `n` 种选择，而每次选择时，递归会进入下一行。在最坏情况下，每行的选择数为 `n`，总的递归调用数为 `n!`。

- **空间复杂度**：`O(N)`。递归深度为 `N`，同时还需要额外的空间来存储列和对角线的标记。

### 示例解析

#### 示例 1：

```python
print(solveNQueens(4))
```

输出：

```plaintext
[['.Q..', '...Q', 'Q...', '..Q.'], ['..Q.', 'Q...', '...Q', '.Q..']]
```

1. 在第一行放置皇后后，递归地尝试在剩下的行中放置皇后，直到所有皇后都成功放置。
2. 结果是两个不同的解：

    1. 皇后分别放置在 `(0, 1)`、`(1, 3)`、`(2, 0)` 和 `(3, 2)`。
    2. 皇后分别放置在 `(0, 2)`、`(1, 0)`、`(2, 3)` 和 `(3, 1)`。

#### 示例 2：

```python
print(solveNQueens(1))
```

输出：

```plaintext
[['Q']]
```

1. 当 `n=1` 时，唯一的解是直接在第一行第一列放置一个皇后。

### 总结

| 方法           | 时间复杂度  | 空间复杂度  | 适用场景 |
|----------------|-------------|-------------|----------|
| 回溯法         | `O(N!)`     | `O(N)`      | 适用于需要遍历所有可能解的组合问题，如排列、组合、棋盘问题等。 |

通过回溯法解决 N 皇后问题可以有效地找到所有解，同时借助剪枝优化大大减少了不必要的计算。
