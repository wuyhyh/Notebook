## 🧭 二、它属于哪一层？和 bootloader 是什么关系？

### 运行层级如下：

```
[硬件电源上电]
     ↓
[UEFI 固件（主板提供）]
     ↓
[你的 UEFI 应用程序（BOOTX64.EFI）]
     ↓
[可选：加载 OS bootloader（如 GRUB） → 加载内核]
```

| 项目                 | 所属层级         | 是否你能控制/开发         |
|--------------------|--------------|-------------------|
| UEFI 固件            | 主板 SPI Flash | ❌ 固化、不可改          |
| UEFI 应用程序          | 软件层          | ✅ 你写的 BOOTX64.EFI |
| bootloader（如 GRUB） | UEFI 应用      | ✅ 更复杂的 UEFI App   |
| Linux 内核           | OS 层         | ✅ 系统软件            |

✅ 所以你写的程序和 bootloader 在同一“软件层级”上：**UEFI 应用层**，只是你目前实现的是一个**最小化版本，不承担加载内核的任务。
**

## 💾 三、这个程序运行在哪里？需要放到哪里？

### ✅ 放置位置：

必须放在 **UEFI 固件能识别的 EFI System Partition（ESP）中的标准路径**：

```
/EFI/BOOT/BOOTX64.EFI
```

| 条件      | 内容                                   |
|---------|--------------------------------------|
| 分区格式    | GPT（或 MBR 兼容），FAT32                  |
| 分区类型    | ESP 分区（标记为 `EF00`）                   |
| 路径要求    | 固定路径 `/EFI/BOOT/BOOTX64.EFI` 是默认引导入口 |
| U盘引导时作用 | 固件插入 U盘后会搜索这个路径并执行你的 `.efi` 文件       |

## 🧠 四、你的程序是固件之后第一个**可变软件**吗？

### ✅ 是的！

* 固件（UEFI）是写死在主板上的，只能升级但不能轻易修改。
* UEFI 固件负责扫描所有可引导设备上的 ESP 分区，查找 `.efi` 文件。
* 一旦找到，就会加载并执行这个文件（比如你的 `BOOTX64.EFI`）。

### ✅ 所以你现在写的 `.efi` 程序，就是：

> **固件执行完毕后，第一个用户自定义的可变程序。**
