v2 的实现方式还不是完整的 ADT，`stack.h`暴露了`Stack`类型的实现方式，无法组织用户直接使用Stack结构，甚至访问top和contents。

我们需要一种方式真正组织用户知道Stack类型的具体实现方式。

C语言提供的封装工具是`不完整类型`.不完整类型是描述了对象，但缺少定义对象大小所需要的信息。

因为编译器不知道不完整对象的大小，所以不能用不完整对象来声明变量。
办法是，我们可以定义一个指针类型引用不完整类型。
```c++
struct t;
typedef struct t *T;
```
还有一个问题是，我们不能对T类型使用`->`运算符，因为编译器对t的成员信息一无所知。
