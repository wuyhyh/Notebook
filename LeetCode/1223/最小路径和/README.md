LeetCode第64题是《最小路径和》（Minimum Path Sum），题目要求给定一个 `m x n` 的网格 `grid`，每个格子中都有一个数字，从左上角（`grid[0][0]`）开始，要求找到一条路径到达右下角（`grid[m-1][n-1]`），路径只能向右或向下移动。路径的总和就是路径上所有数字之和，找出所有路径中总和最小的路径。

### 题目描述

给定一个包含非负整数的 `m x n` 网格 `grid`，从左上角到右下角的最小路径和是多少？

- **说明**：每次只能向下或者向右移动一步。

### 示例 1：

```plaintext
输入：
grid = [
  [1, 3, 1],
  [1, 5, 1],
  [4, 2, 1]
]
输出：7
解释：因为路径 1 → 3 → 1 → 1 → 1 的总和最小。
```

### 示例 2：

```plaintext
输入：
grid = [
  [1, 2, 3],
  [4, 5, 6]
]
输出：12
解释：因为路径 1 → 2 → 3 → 6 的总和最小。
```

### 解题思路

这个问题可以使用 **动态规划** 来解决。我们定义一个 `dp` 数组，`dp[i][j]` 表示从 `grid[0][0]` 到达 `grid[i][j]` 的最小路径和。为了更新 `dp[i][j]`，我们可以选择从上方或者左方过来的路径。

#### 动态规划的步骤：

1. **状态定义**：
    - 定义一个二维数组 `dp`，其中 `dp[i][j]` 表示从起点到达位置 `(i, j)` 的最小路径和。

2. **状态转移**：
    - 对于每个位置 `(i, j)`，最小路径和为从上面或者左面来的最小路径和加上当前位置的值：
      \[
      dp[i][j] = grid[i][j] + \min(dp[i-1][j], dp[i][j-1])
      \]
    - 需要注意边界情况：
        - 如果 `i == 0`，只能从左边过来：`dp[0][j] = grid[0][j] + dp[0][j-1]`
        - 如果 `j == 0`，只能从上面过来：`dp[i][0] = grid[i][0] + dp[i-1][0]`

3. **初始化**：
    - 初始化 `dp[0][0] = grid[0][0]`，因为起点的路径和就是其自身的值。

4. **最终结果**：
    - `dp[m-1][n-1]` 即为从起点到终点的最小路径和。

### 代码实现

```python
def minPathSum(grid):
    if not grid:
        return 0
    
    m, n = len(grid), len(grid[0])
    
    # 初始化 dp 数组，dp[i][j] 代表从 (0,0) 到 (i,j) 的最小路径和
    dp = [[0] * n for _ in range(m)]
    
    # 初始化左上角
    dp[0][0] = grid[0][0]
    
    # 初始化第一列
    for i in range(1, m):
        dp[i][0] = dp[i-1][0] + grid[i][0]
    
    # 初始化第一行
    for j in range(1, n):
        dp[0][j] = dp[0][j-1] + grid[0][j]
    
    # 填充 dp 数组
    for i in range(1, m):
        for j in range(1, n):
            dp[i][j] = grid[i][j] + min(dp[i-1][j], dp[i][j-1])
    
    # 返回右下角的最小路径和
    return dp[m-1][n-1]
```

### 代码解析：

1. **初始化**：
    - 使用 `dp = [[0] * n for _ in range(m)]` 初始化一个大小为 `m x n` 的二维数组 `dp`。
    - `dp[0][0]` 初始化为 `grid[0][0]`，表示起点的路径和。

2. **填充第一列和第一行**：
    - 第一列的每个位置只可能从上方过来，`dp[i][0] = dp[i-1][0] + grid[i][0]`。
    - 第一行的每个位置只可能从左边过来，`dp[0][j] = dp[0][j-1] + grid[0][j]`。

3. **动态规划填充其余位置**：
    - 对于其余的每个位置 `(i, j)`，我们根据状态转移方程 `dp[i][j] = grid[i][j] + min(dp[i-1][j], dp[i][j-1])` 来更新 `dp[i][j]`。

4. **返回结果**：
    - 最终，`dp[m-1][n-1]` 即为从左上角到右下角的最小路径和。

### 时间复杂度

- **时间复杂度**：`O(m * n)`，其中 `m` 是网格的行数，`n` 是网格的列数。我们需要遍历整个网格，更新 `dp` 数组。
- **空间复杂度**：`O(m * n)`，用于存储 `dp` 数组。

### 优化：空间优化

如果不需要保存整个 `dp` 数组，我们可以优化空间，只使用一维数组来记录当前行的最小路径和。

#### 空间优化解法

```python
def minPathSum(grid):
    if not grid:
        return 0
    
    m, n = len(grid), len(grid[0])
    
    # 初始化一个一维 dp 数组
    dp = [0] * n
    
    # 初始化第一列
    dp[0] = grid[0][0]
    for j in range(1, n):
        dp[j] = dp[j-1] + grid[0][j]
    
    # 动态规划更新 dp 数组
    for i in range(1, m):
        # 更新第一列
        dp[0] += grid[i][0]
        for j in range(1, n):
            dp[j] = grid[i][j] + min(dp[j], dp[j-1])
    
    # 最终结果是 dp[n-1]
    return dp[n-1]
```

### 优化解法解析：

1. **使用一维数组 `dp`**：
    - 我们只使用一维数组 `dp` 来存储当前行的最小路径和，避免使用整个二维数组。

2. **更新第一列**：
    - 由于每一行的第一个元素只能从上面过来，我们需要逐行累加 `dp[0]`。

3. **更新其他位置**：
    - 对于每一行的其他位置，更新 `dp[j]` 为当前元素 `grid[i][j]` 和 `dp[j-1]`、`dp[j]` 的最小值之和。

### 时间复杂度

- **时间复杂度**：`O(m * n)`，和之前一样，依然需要遍历整个网格。
- **空间复杂度**：`O(n)`，我们只需要一个大小为 `n` 的一维数组。

### 示例解析

#### 示例 1：`grid = [[1, 3, 1], [1, 5, 1], [4, 2, 1]]`

1. **初始化**：
    - `dp[0][0] = 1`
    - 第一行：`dp[0][1] = 1 + 3 = 4`, `dp[0][2] = 4 + 1 = 5`
    - 第一列：`dp[1][0] = 1 + 1 = 2`, `dp[2][0] = 2 + 4 = 6`

2. **填充剩余的 `dp`**：
    - `dp[1][1] = 5 + min(2, 4) = 7`, `dp[1][2] = 1 + min(7, 5) = 6`
    - `dp[2][1] = 2 + min(7, 6) = 8`, `dp[2][2] = 1 + min(8, 6) = 7`

3. **结果**：`dp[2][2] = 7`
