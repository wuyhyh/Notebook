LeetCode第22题是《括号生成》（Generate Parentheses），题目要求生成所有可能的有效括号组合。

### 题目描述

给定 `n` 对括号，生成所有可能的有效括号组合。

**有效括号组合**的定义是：

- 左括号 `(` 必须总是出现在右括号 `)` 之前。
- 括号必须是成对出现的。

### 示例

**示例 1：**

```plaintext
输入: n = 3
输出: ["((()))", "(()())", "(())()", "()(())", "()()()"]
```

**示例 2：**

```plaintext
输入: n = 1
输出: ["()"]
```

### 解题思路

这道题目可以通过 **回溯算法** 来解决。回溯算法非常适合生成所有可能的组合，特别是对于这种成对匹配的括号问题。

#### 思路：

1. **递归构建括号**：我们从一个空的字符串开始，递归地添加左括号 `(` 或右括号 `)`，直到字符串的长度等于 `2 * n`。

2. **有效的括号组合**：
    - 左括号 `(` 必须先于右括号 `)`。
    - 需要确保在任何时候，已经添加的右括号数不能大于左括号数。

3. **递归条件**：
    - 可以添加左括号的条件是：当前左括号数目小于 `n`。
    - 可以添加右括号的条件是：当前右括号数目小于左括号数目。

4. **回溯终止条件**：当当前生成的括号串的长度达到 `2 * n` 时，表示一组合法的括号组合，加入到结果列表中。

### 代码实现

```python
def generateParenthesis(n):
    result = []

    def backtrack(current_string, open_count, close_count):
        # 如果当前字符串的长度达到 2 * n，说明是一个有效组合
        if len(current_string) == 2 * n:
            result.append(current_string)
            return
        
        # 如果左括号的数量小于 n，可以添加一个左括号
        if open_count < n:
            backtrack(current_string + '(', open_count + 1, close_count)
        
        # 如果右括号的数量小于左括号的数量，可以添加一个右括号
        if close_count < open_count:
            backtrack(current_string + ')', open_count, close_count + 1)
    
    # 初始化回溯，当前字符串为空，左括号和右括号数量都为 0
    backtrack("", 0, 0)
    
    return result
```

### 代码解析

1. **`backtrack(current_string, open_count, close_count)`**：
    - `current_string`：当前生成的括号字符串。
    - `open_count`：当前已经添加的左括号 `(` 的数量。
    - `close_count`：当前已经添加的右括号 `)` 的数量。

2. **递归逻辑**：
    - **添加左括号**：如果 `open_count < n`，我们可以添加一个左括号 `(`。
    - **添加右括号**：如果 `close_count < open_count`，我们可以添加一个右括号 `)`，以保证括号配对。

3. **终止条件**：
    - 当 `current_string` 的长度等于 `2 * n` 时，表示当前字符串是一个有效的括号组合，加入 `result`。

4. **初始化回溯**：开始时调用 `backtrack("", 0, 0)`，即从空字符串开始，左右括号的数量都为 0。

### 时间复杂度

- **时间复杂度**：`O(4^n / sqrt(n))`，这是由卡特兰数（Catalan number）给出的复杂度，表示生成所有有效括号组合的数量。对于每个组合，构建和检查的时间复杂度为 `O(n)`，因此总的时间复杂度是 `O(4^n / sqrt(n))`。

- **空间复杂度**：`O(n)`，递归栈的最大深度为 `2 * n`，因为每次递归调用时会添加一个字符。

### 示例解析

以 `n = 3` 为例，来看如何生成所有的有效括号组合：

1. 初始调用：`backtrack("", 0, 0)`，即从空字符串开始，左右括号数都为 0。

2. **第一次递归**：
    - 可以添加左括号，因为 `open_count < 3`，调用 `backtrack("(", 1, 0)`。

3. **第二次递归**：
    - 可以继续添加左括号，调用 `backtrack("((", 2, 0)`。

4. **第三次递归**：
    - 可以继续添加左括号，调用 `backtrack("(((", 3, 0)`。

5. **第四次递归**：
    - 现在 `open_count` 达到 3，无法再添加左括号。
    - 可以添加右括号，因为 `close_count < open_count`，调用 `backtrack("((()", 3, 1)`。

6. 继续递归，直到生成完整的组合 `((()))`，然后回溯到上一层，生成其他组合。

最终，返回所有合法的组合：

```plaintext
["((()))", "(()())", "(())()", "()(())", "()()()"]
```

### 总结

- **回溯算法**：这道题通过回溯算法递归地生成所有可能的括号组合，并利用有效括号的约束（左括号总是先于右括号）来限制递归路径。
- **递归终止条件**：当字符串长度达到 `2 * n` 时，我们得到一个有效的组合。
- **优化条件**：确保添加右括号的数量不能超过左括号的数量。
