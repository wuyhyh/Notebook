### **LeetCode 第62题 - 不同路径 (Unique Paths)**

#### **题目描述**
一个机器人位于一个 `m x n` 的网格的左上角（起点 `(0, 0)`），机器人只能向下或向右移动。机器人试图到达网格的右下角 `(m - 1, n - 1)`。问有多少条不同的路径？

**要求**：
- 给定 `m` 和 `n`，返回从左上角到右下角的不同路径总数。

---

### **解题思路**

本题可以通过 **动态规划**（Dynamic Programming）来求解。我们从起点 `(0, 0)` 出发，到达右下角 `(m-1, n-1)`，每一步只能向右或向下移动。因此，我们可以通过动态规划来计算每个位置到达的路径数。

#### **状态转移方程**：
设 `dp[i][j]` 表示从 `(0, 0)` 到达位置 `(i, j)` 的路径数。可以推导出以下关系：
- **边界条件**：
    - 第一行的每个位置 `dp[0][j]`，只能从左侧的格子到达，因此每个位置的路径数为 1。
    - 第一列的每个位置 `dp[i][0]`，只能从上方的格子到达，因此每个位置的路径数为 1。

- **状态转移**：
  对于其他位置 `dp[i][j]`，可以从上方 `dp[i-1][j]` 或左方 `dp[i][j-1]` 位置到达。因此，状态转移方程为：
  \[
  dp[i][j] = dp[i-1][j] + dp[i][j-1]
  \]

最终答案就是 `dp[m-1][n-1]`，即右下角的位置的路径数。

---

### **代码实现**

```python
class Solution:
    def uniquePaths(self, m: int, n: int) -> int:
        # 创建一个 m x n 的二维 DP 数组
        dp = [[1] * n for _ in range(m)]
        
        # 填充 DP 数组，注意边界条件已经设置为 1
        for i in range(1, m):
            for j in range(1, n):
                dp[i][j] = dp[i-1][j] + dp[i][j-1]
        
        # 返回右下角的路径数
        return dp[m-1][n-1]
```

### **代码解析**
1. **初始化二维 DP 数组**：我们创建一个 `m x n` 的二维数组 `dp`，并将所有的元素初始化为 1。因为第一行和第一列的路径数均为 1（它们只能从左边或上方来）。
2. **填充 DP 数组**：我们从 `dp[1][1]` 开始填充，根据状态转移方程 `dp[i][j] = dp[i-1][j] + dp[i][j-1]` 更新每个位置的路径数。
3. **返回结果**：最终结果为 `dp[m-1][n-1]`，即右下角的位置的路径数。

---

### **时间复杂度**
- **时间复杂度**：`O(m * n)`，需要遍历整个 `m x n` 的二维数组，更新每个位置的路径数。
- **空间复杂度**：`O(m * n)`，需要一个 `m x n` 的二维数组来存储路径数。

---

### **优化空间复杂度**

如果不需要保存整个二维数组，我们可以优化空间复杂度到 `O(n)`。因为每一行只需要依赖上一行的信息，因此可以使用一维数组来存储当前行的路径数。

#### **优化后的代码**：

```python
class Solution:
    def uniquePaths(self, m: int, n: int) -> int:
        # 创建一个大小为 n 的一维数组
        dp = [1] * n
        
        # 逐行更新 dp 数组
        for i in range(1, m):
            for j in range(1, n):
                dp[j] += dp[j-1]
        
        # 返回最后一个元素，即 dp[n-1]
        return dp[n-1]
```

### **优化后的时间与空间复杂度**
- **时间复杂度**：`O(m * n)`，仍然是遍历整个网格。
- **空间复杂度**：`O(n)`，只需要一个大小为 `n` 的一维数组来存储当前行的信息。

---

### **示例测试**

#### 示例 1：
```python
m = 3
n = 7
solution = Solution()
print(solution.uniquePaths(m, n))  # 输出 28
```

#### 示例 2：
```python
m = 3
n = 2
solution = Solution()
print(solution.uniquePaths(m, n))  # 输出 3
```

---

### **总结**
- 本题通过动态规划的方法，利用状态转移方程计算从起点到终点的不同路径数。
- 初步实现的时间和空间复杂度为 `O(m * n)`，空间复杂度为 `O(m * n)`，通过优化可以将空间复杂度降到 `O(n)`，但时间复杂度保持不变。
