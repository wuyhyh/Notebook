# 文件查看搜索与文本处理

## 文档目标

本文档用于帮助实习生掌握 Linux 下常用的文件查看、内容搜索和简单文本处理方法。

完成本文档学习后，应能够：

1. 使用 `cat`、`less`、`head`、`tail` 查看文本文件；
2. 使用 `grep` 在文件、目录、命令输出中搜索关键字；
3. 使用 `find` 查找文件；
4. 使用 `wc` 统计文件行数、单词数和字节数；
5. 使用 `sort`、`uniq`、`cut`、`awk`、`sed` 做基础文本处理；
6. 使用管道 `|` 组合多个命令；
7. 使用重定向 `>`、`>>` 保存命令输出；
8. 能够从日志、配置文件、内核源码中快速定位关键信息；
9. 能够按要求保存搜索结果和分析记录。

本文档重点不是学习复杂脚本，而是训练一种基本能力：

**面对大量文本输出、日志文件、源码文件时，能够快速找到有用信息。**

## 学习要求

学习本文档时，必须实际执行命令。

建议准备一个练习目录：

```bash
mkdir -p ~/text-command-test
cd ~/text-command-test
```

后续实验可以在该目录下完成。

注意事项：

1. 不要在系统关键目录中随意修改文件；
2. 搜索源码或日志时，应优先使用只读命令；
3. 使用 `>` 重定向前要确认目标文件是否可以覆盖；
4. 对系统日志、启动日志、构建日志要尽量保存原始文本，不要只截图；
5. 不理解命令作用时，先在测试目录中验证。

## 文本文件是什么

Linux 中大量信息都以文本形式存在，例如：

```text
配置文件
日志文件
源码文件
Makefile
Kconfig
DTS
系统状态文件
命令输出结果
```

常见文本文件示例：

```text
/etc/os-release
/etc/fstab
/proc/cmdline
/proc/interrupts
/var/log/messages
Makefile
Kconfig
*.c
*.h
*.dts
*.dtsi
*.config
```

在 Linux 开发、内核调试、驱动排查中，很多工作本质上都是：

```text
查看文本 → 搜索关键字 → 提取信息 → 保存结果 → 分析现象
```

## 使用 cat 查看小文件

`cat` 用于直接输出文件内容。

查看文件：

```bash
cat file.txt
```

查看系统版本：

```bash
cat /etc/os-release
```

查看当前内核启动参数：

```bash
cat /proc/cmdline
```

查看内存信息：

```bash
cat /proc/meminfo
```

`cat` 适合查看较短文件。

不适合直接查看很大的日志文件，例如：

```bash
cat dmesg.txt
cat build.log
```

大文件应优先使用 `less`、`head`、`tail` 或 `grep`。

## 使用 less 分页查看文件

`less` 用于分页查看文本文件，适合查看大日志、大源码文件。

打开文件：

```bash
less file.txt
```

常用按键：

```text
方向键       上下移动
空格         向下翻页
b            向上翻页
g            跳到文件开头
G            跳到文件结尾
/关键字      向下搜索关键字
?关键字      向上搜索关键字
n            下一个搜索结果
N            上一个搜索结果
q            退出
```

示例：

```bash
less dmesg.txt
```

在 `less` 中搜索 `error`：

```text
/error
```

搜索 `stmmac`：

```text
/stmmac
```

搜索 `phy`：

```text
/phy
```

`less` 是查看日志和源码时非常重要的工具。

## 使用 head 查看文件开头

`head` 用于查看文件开头内容。

查看前 10 行：

```bash
head file.txt
```

查看前 50 行：

```bash
head -n 50 file.txt
```

常用示例：

```bash
head -n 30 dmesg.txt
head -n 50 build.log
head -n 20 /etc/os-release
```

查看启动日志开头时可以使用：

```bash
dmesg | head -n 50
```

## 使用 tail 查看文件结尾

`tail` 用于查看文件结尾内容。

查看最后 10 行：

```bash
tail file.txt
```

查看最后 50 行：

```bash
tail -n 50 file.txt
```

常用示例：

```bash
tail -n 100 dmesg.txt
tail -n 100 build.log
```

查看命令输出的最后部分：

```bash
dmesg | tail -n 100
journalctl -b | tail -n 100
```

构建失败时，错误通常出现在日志末尾附近，因此常用：

```bash
tail -n 200 build.log
```

## 使用 tail -f 实时查看文件变化

`tail -f` 可以实时查看文件新增内容。

例如：

```bash
tail -f log.txt
```

当另一个进程继续向 `log.txt` 写入内容时，终端会持续显示新增行。

常用场景：

```bash
tail -f /var/log/messages
```

或者：

```bash
journalctl -f
```

如果系统使用 systemd，实时查看服务日志通常使用：

```bash
journalctl -f
```

查看某个服务日志：

```bash
journalctl -u NetworkManager -f
```

## 使用 grep 搜索关键字

`grep` 用于在文本中搜索关键字。

基本格式：

```bash
grep "关键字" 文件名
```

示例：

```bash
grep "error" dmesg.txt
```

搜索时忽略大小写：

```bash
grep -i "error" dmesg.txt
```

显示匹配行号：

```bash
grep -n "error" dmesg.txt
```

同时忽略大小写并显示行号：

```bash
grep -ni "error" dmesg.txt
```

常用示例：

```bash
grep -i "error" dmesg.txt
grep -i "fail" dmesg.txt
grep -i "warning" build.log
grep -i "stmmac" dmesg.txt
grep -i "phy" dmesg.txt
grep -i "pci" dmesg.txt
```

## grep 搜索命令输出

`grep` 不仅可以搜索文件，也可以搜索命令输出。

例如：

```bash
dmesg | grep -i error
```

含义是：

```text
先执行 dmesg 输出内核日志
再把输出交给 grep 搜索 error
```

常用示例：

```bash
dmesg | grep -i eth
dmesg | grep -i phy
dmesg | grep -i stmmac
dmesg | grep -i pci
dmesg | grep -i firmware
journalctl -b | grep -i error
ip addr | grep inet
ps aux | grep ssh
cat /proc/interrupts | grep eth
```

在开发板调试中，这类用法非常常见。

## grep 递归搜索目录

在源码目录中搜索关键字时，经常需要递归搜索。

基本格式：

```bash
grep -rn "关键字" 目录
```

示例：

```bash
grep -rn "CONFIG_DWMAC" .
```

含义：

```text
-r      递归搜索目录
-n      显示匹配行号
```

常用示例：

```bash
grep -rn "compatible" arch/arm64/boot/dts/
grep -rn "phytium" drivers/
grep -rn "stmmac" drivers/net/
grep -rn "of_match_table" drivers/
grep -rn "platform_driver" drivers/
```

忽略大小写：

```bash
grep -rni "stmmac" drivers/net/
```

只显示文件名：

```bash
grep -rl "CONFIG_DWMAC" .
```

只统计匹配数量：

```bash
grep -rc "CONFIG_DWMAC" .
```

## grep 搜索多个关键字

搜索 `error` 或 `fail`：

```bash
grep -Ei "error|fail" dmesg.txt
```

搜索 `eth`、`phy` 或 `stmmac`：

```bash
grep -Ei "eth|phy|stmmac" dmesg.txt
```

说明：

```text
-E      使用扩展正则表达式
-i      忽略大小写
|       表示或者
```

常用示例：

```bash
dmesg | grep -Ei "error|fail|warning"
dmesg | grep -Ei "eth|phy|stmmac|mdio"
journalctl -b | grep -Ei "failed|timeout|error"
```

## grep 显示匹配上下文

有时只看匹配行不够，需要查看前后几行。

显示匹配行前 3 行：

```bash
grep -B 3 "error" dmesg.txt
```

显示匹配行后 3 行：

```bash
grep -A 3 "error" dmesg.txt
```

显示匹配行前后各 3 行：

```bash
grep -C 3 "error" dmesg.txt
```

常用示例：

```bash
grep -C 5 -i "panic" dmesg.txt
grep -C 5 -i "stmmac" dmesg.txt
grep -C 5 -i "Cannot attach to PHY" dmesg.txt
```

排查内核错误时，`-C` 非常有用。

## grep 排除关键字

排除包含某个关键字的行：

```bash
grep -v "关键字" file.txt
```

示例：

```bash
grep -v "^#" config.txt
```

含义是过滤掉以 `#` 开头的注释行。

排除空行：

```bash
grep -v "^$" file.txt
```

同时排除注释行和空行：

```bash
grep -v "^#" file.txt | grep -v "^$"
```

常用场景：

```bash
grep -v "^#" /etc/fstab
grep -v "^$" file.txt
```

## grep 搜索配置项

查看内核配置项：

```bash
grep "CONFIG_PCI" .config
```

查看多个相关配置项：

```bash
grep -E "CONFIG_PCI|CONFIG_OF|CONFIG_NET" .config
```

搜索 stmmac 相关配置：

```bash
grep -Ei "STMMAC|DWMAC|PHYTIUM" .config
```

搜索 PHY 相关配置：

```bash
grep -Ei "PHY|REALTEK|MOTORCOMM|YT8521" .config
```

常用示例：

```bash
grep -E "CONFIG_STMMAC|CONFIG_DWMAC|CONFIG_PHY" .config
```

注意：

```text
CONFIG_XXX=y          编进内核
CONFIG_XXX=m          编译为模块
# CONFIG_XXX is not set  未启用
```

## 使用 find 查找文件

`find` 用于按条件查找文件。

在当前目录查找文件名：

```bash
find . -name "Makefile"
```

查找所有 C 文件：

```bash
find . -name "*.c"
```

查找所有头文件：

```bash
find . -name "*.h"
```

查找 DTS 文件：

```bash
find . -name "*.dts"
```

查找 DTSI 文件：

```bash
find . -name "*.dtsi"
```

查找 DTB 文件：

```bash
find . -name "*.dtb"
```

常用示例：

```bash
find arch/arm64/boot/dts/ -name "*pd2008*"
find drivers/net/ -name "*stmmac*"
find . -name "Kconfig"
find . -name "*.ko"
```

## find 按时间和大小查找

查找最近 1 天修改过的文件：

```bash
find . -mtime -1
```

查找 7 天前修改过的文件：

```bash
find . -mtime +7
```

查找大于 100MB 的文件：

```bash
find . -size +100M
```

查找空文件：

```bash
find . -empty
```

常用场景：

```bash
find . -mtime -1
find . -size +500M
find . -name "*.log" -size +100M
```

当磁盘空间不足时，可以先查找大文件：

```bash
find . -size +100M
```

## find 与 grep 组合使用

先找文件，再搜索内容。

例如只在 `.c` 文件中搜索 `platform_driver`：

```bash
find . -name "*.c" -exec grep -n "platform_driver" {} \\;
```

搜索 DTS 中的 compatible：

```bash
find arch/arm64/boot/dts/ -name "*.dts" -exec grep -n "compatible" {} \\;
```

更简单的方式通常是：

```bash
grep -rn "platform_driver" --include="*.c" .
```

只搜索 `.c` 文件：

```bash
grep -rn "platform_driver" --include="*.c" .
```

只搜索 `.dts` 文件：

```bash
grep -rn "compatible" --include="*.dts" arch/arm64/boot/dts/
```

排除某些目录：

```bash
grep -rn "stmmac" . --exclude-dir=.git
```

排除构建目录：

```bash
grep -rn "stmmac" . --exclude-dir=.git --exclude-dir=build
```

## 使用 wc 统计行数

`wc` 用于统计行数、单词数、字节数。

统计文件行数：

```bash
wc -l file.txt
```

统计多个文件行数：

```bash
wc -l *.c
```

统计命令输出行数：

```bash
dmesg | wc -l
```

统计搜索结果数量：

```bash
grep -rn "stmmac" drivers/net/ | wc -l
```

常用示例：

```bash
wc -l dmesg.txt
wc -l build.log
find . -name "*.c" | wc -l
grep -rn "CONFIG_" .config | wc -l
```

## 使用 sort 排序

`sort` 用于排序文本行。

基本用法：

```bash
sort file.txt
```

按数字排序：

```bash
sort -n numbers.txt
```

倒序排序：

```bash
sort -r file.txt
```

按可读大小排序：

```bash
sort -h size.txt
```

常用示例：

```bash
du -sh * | sort -h
```

含义是查看当前目录各项大小，并按大小排序。

查看进程并按 CPU 或内存排序通常用 `top`，但简单文本排序可用 `sort`。

## 使用 uniq 去重

`uniq` 用于去除相邻重复行。

通常和 `sort` 组合使用：

```bash
sort file.txt | uniq
```

统计重复次数：

```bash
sort file.txt | uniq -c
```

按重复次数排序：

```bash
sort file.txt | uniq -c | sort -n
```

常用场景：

```bash
cat list.txt | sort | uniq
```

统计日志中某类字段出现次数时，可以结合 `awk`、`sort`、`uniq` 使用。

## 使用 cut 提取列

`cut` 用于按分隔符提取字段。

查看 `/etc/passwd` 中的用户名：

```bash
cut -d ':' -f 1 /etc/passwd
```

含义：

```text
-d ':'      使用冒号作为分隔符
-f 1        提取第 1 列
```

提取第 1 列和第 3 列：

```bash
cut -d ':' -f 1,3 /etc/passwd
```

从空格分隔内容中提取字段时，`cut` 不如 `awk` 灵活。

## 使用 awk 处理文本列

`awk` 适合按列处理文本。

打印第 1 列：

```bash
awk '{print $1}' file.txt
```

打印第 1 列和第 2 列：

```bash
awk '{print $1, $2}' file.txt
```

查看 `df -h` 中的文件系统和使用率：

```bash
df -h | awk '{print $1, $5, $6}'
```

查看网络接口名：

```bash
ip link | awk -F ': ' '/^[0-9]+:/ {print $2}'
```

查看 `/proc/interrupts` 前几列：

```bash
cat /proc/interrupts | awk '{print $1, $2, $3, $NF}'
```

注意：`awk` 功能很强，初学阶段只需要掌握简单列提取。

## awk 按条件过滤

打印包含 `eth` 的行：

```bash
awk '/eth/ {print}' file.txt
```

打印第 5 列大于 80 的行：

```bash
awk '$5 > 80 {print}' file.txt
```

示例：

```bash
df -h | awk '$5 ~ /%/ {print}'
```

初学阶段不要求复杂掌握，只要知道：

```text
awk 可以按列提取，也可以按条件过滤。
```

## 使用 sed 替换文本

`sed` 常用于替换文本。

把文件中的 `old` 替换为 `new`，只输出结果，不修改原文件：

```bash
sed 's/old/new/g' file.txt
```

保存到新文件：

```bash
sed 's/old/new/g' file.txt > new-file.txt
```

直接修改原文件：

```bash
sed -i 's/old/new/g' file.txt
```

注意：`sed -i` 会直接修改文件，使用前必须谨慎。

建议修改前先备份：

```bash
cp file.txt file.txt.bak
sed -i 's/old/new/g' file.txt
```

常用示例：

```bash
sed 's/foo/bar/g' test.txt
```

删除空行：

```bash
sed '/^$/d' file.txt
```

删除注释行：

```bash
sed '/^#/d' file.txt
```

## 使用 diff 比较文件差异

`diff` 用于比较两个文本文件差异。

基本用法：

```bash
diff old.txt new.txt
```

使用 unified 格式：

```bash
diff -u old.txt new.txt
```

常用示例：

```bash
diff -u old.dts new.dts
diff -u config.old config.new
diff -u old.log new.log
```

保存差异到文件：

```bash
diff -u old.txt new.txt > change.patch
```

在项目中，比较 DTS、内核配置、脚本修改时经常使用：

```bash
diff -u before.dts after.dts
diff -u config-before config-after
```

## 使用 comm 比较排序后的文件

`comm` 用于比较两个已经排序的文件。

先排序：

```bash
sort a.txt > a.sorted
sort b.txt > b.sorted
```

比较：

```bash
comm a.sorted b.sorted
```

只看两个文件共有内容：

```bash
comm -12 a.sorted b.sorted
```

只看 a 中有、b 中没有的内容：

```bash
comm -23 a.sorted b.sorted
```

只看 b 中有、a 中没有的内容：

```bash
comm -13 a.sorted b.sorted
```

这个命令在比较文件列表时很有用。

例如比较两个目录下的文件名：

```bash
find dir-a -type f | sort > a.txt
find dir-b -type f | sort > b.txt
comm -23 a.txt b.txt
```

## 使用 tee 同时显示和保存输出

`tee` 可以把命令输出同时显示在屏幕上并保存到文件。

示例：

```bash
dmesg | tee dmesg.txt
```

追加保存：

```bash
dmesg | tee -a dmesg.txt
```

构建时保存日志：

```bash
make 2>&1 | tee build.log
```

这样既能在终端看到输出，也能保存完整日志。

## 管道的基本用法

管道符号 `|` 用于把前一个命令的输出交给后一个命令。

示例：

```bash
dmesg | grep -i error
```

多个命令可以连续组合：

```bash
dmesg | grep -i error | tail -n 20
```

含义：

```text
先输出 dmesg
再搜索 error
最后只显示最后 20 行
```

常用示例：

```bash
journalctl -b | grep -Ei "error|fail|timeout" | less
dmesg | grep -Ei "eth|phy|stmmac" | tee net-related.log
find . -name "*.c" | wc -l
du -sh * | sort -h
```

管道是 Linux 文本处理的核心能力。

## 输出重定向

把输出保存到文件：

```bash
command > file.txt
```

示例：

```bash
dmesg > dmesg.txt
```

追加到文件末尾：

```bash
command >> file.txt
```

示例：

```bash
date >> report.txt
uname -a >> report.txt
```

保存标准输出和错误输出：

```bash
command > log.txt 2>&1
```

构建时常用：

```bash
make > build.log 2>&1
```

或者：

```bash
make 2>&1 | tee build.log
```

注意：

```text
>       覆盖文件
>>      追加文件
2>&1    把错误输出合并到标准输出
```

## 常见日志搜索关键字

排查问题时，常用关键字包括：

```text
error
fail
failed
warning
timeout
panic
oops
BUG
call trace
segfault
denied
not found
invalid
probe
firmware
```

内核日志搜索：

```bash
dmesg | grep -Ei "error|fail|warning|timeout|panic|oops|BUG"
```

本次启动日志搜索：

```bash
journalctl -b | grep -Ei "error|fail|warning|timeout"
```

构建日志搜索：

```bash
grep -Ei "error:|warning:|undefined reference|No such file" build.log
```

注意：不是所有 `warning` 都一定导致失败，但所有 `error` 都应该优先关注。

## 网络问题常用搜索

网络问题常用关键字：

```text
eth
net
phy
mdio
link
stmmac
dwmac
NetworkManager
DHCP
timeout
carrier
```

常用命令：

```bash
dmesg | grep -Ei "eth|net|phy|mdio|link|stmmac|dwmac"
journalctl -b -u NetworkManager | grep -Ei "error|fail|timeout|dhcp|link"
ip addr | tee ip-addr.txt
ip route | tee ip-route.txt
```

网卡驱动相关日志：

```bash
dmesg | grep -Ei "stmmac|dwmac|phy|mdio"
```

如果出现：

```text
Cannot attach to PHY
```

需要重点检查：

1. DTS 中 PHY/MDIO 描述；
2. PHY 驱动是否启用；
3. MAC 与 PHY 连接关系；
4. dmesg 中更早的相关日志。

## PCIe 问题常用搜索

PCIe 问题常用关键字：

```text
pci
pcie
aer
link
BAR
bridge
enumeration
resource
MSI
```

常用命令：

```bash
dmesg | grep -Ei "pci|pcie|aer|BAR|MSI|resource"

lspci -xxxx > lspci-xxxx.txt

```

如果没有 `lspci`，可以先查看：

```bash
ls /sys/bus/pci/devices
```

## 查看文件编码和特殊字符

有时脚本从 Windows 拷贝到 Linux 后会出现换行符问题。

查看文件类型：

```bash

```

如果看到：

```text
with CRLF line terminators
```

说明文件可能使用 Windows 换行符。

可以使用 `dos2unix` 转换：

```bash
dos2unix script.sh
```

如果系统没有 `dos2unix`，可以使用：

```bash
sed -i 's/\\r$//' script.sh
```

注意：使用 `sed -i` 前建议备份。

## 文本处理时的常见错误

直接用 `cat` 查看巨大日志，导致终端输出大量内容：

```bash
cat build.log
```

更好的方式：

```bash
less build.log
tail -n 200 build.log
grep -Ei "error|fail" build.log
```

使用 `>` 覆盖重要文件：

```bash
dmesg > important-report.md
```

如果目标文件已经存在，会被覆盖。

不确定时先使用新文件名：

```bash
dmesg > dmesg-$(date +%Y%m%d-%H%M%S).txt
```

使用 `sed -i` 直接修改文件但没有备份：

```bash
sed -i 's/old/new/g' config.txt
```

更安全的方式：

```bash
cp config.txt config.txt.bak
sed -i 's/old/new/g' config.txt
diff -u config.txt.bak config.txt
```

搜索关键字时没有忽略大小写，导致漏掉结果：

```bash
grep "error" dmesg.txt
```

更好的方式：

```bash
grep -i "error" dmesg.txt
```

源码搜索时没有排除 `.git` 或构建目录，导致结果过多：

```bash
grep -rn "stmmac" .
```

可以改为：

```bash
grep -rn "stmmac" . --exclude-dir=.git --exclude-dir=build
```

## 实验任务

请在 Linux 开发机或开发板上完成以下实验。

第一部分：创建实验文件。

```bash
mkdir -p ~/text-command-test

[    0.100000] Linux version 5.10.0-test
[    1.000000] pci 0000:00:00.0: host bridge
[    2.000000] stmmaceth 2820c000.eth: initializing
[    3.000000] phy 0:01: attached PHY driver
[    4.000000] warning: test warning message
[    5.000000] error: test error message
[    6.000000] failed to load firmware test.bin
[    7.000000] network link up
EOF
```

第二部分：查看文件。

```bash
cat sample.log
head -n 3 sample.log
tail -n 3 sample.log
less sample.log
```

第三部分：搜索关键字。

```bash
grep -i "error" sample.log
grep -i "warning" sample.log
grep -Ei "error|failed|warning" sample.log
grep -Ei "pci|stmmac|phy|network" sample.log
grep -C 2 -i "error" sample.log
```

第四部分：保存搜索结果。

```bash
mkdir -p report

grep -Ei "error|failed|warning" sample.log > report/errors.txt
grep -Ei "pci|stmmac|phy|network" sample.log > report/device-related.txt
wc -l sample.log > report/line-count.txt
```

第五部分：查找文件。

```bash
find . -name "*.log"
find . -name "*.txt"
find . -type f
```

第六部分：排序和统计。

```bash
cat > names.txt << 'EOF'
eth
pci
phy
eth
stmmac
pci
eth
EOF

sort names.txt
sort names.txt | uniq
sort names.txt | uniq -c
sort names.txt | uniq -c | sort -n
```

第七部分：文本列处理。

```bash
df -h | awk '{print $1, $5, $6}'
cut -d ':' -f 1 /etc/passwd | head
```

第八部分：保存系统日志搜索结果。

```bash
dmesg > report/dmesg.txt
dmesg | grep -Ei "error|fail|warning|timeout" > report/dmesg-errors.txt
dmesg | grep -Ei "pci|pcie|eth|phy|stmmac|firmware" > report/dmesg-device-related.txt
journalctl -b | tail -n 200 > report/journal-tail-200.txt
```

```bash
tar -czvf text-command-test-report.tar.gz report sample.log names.txt
```

## 验收标准

验收时重点检查以下内容：

1. 是否能正确使用 `cat`、`less`、`head`、`tail` 查看文件；
2. 是否能使用 `grep` 搜索普通文件和命令输出；
3. 是否能使用 `grep -r` 搜索目录；
4. 是否能使用 `grep -Ei` 搜索多个关键字；
5. 是否能使用 `grep -C` 查看匹配行上下文；
6. 是否能使用 `find` 查找文件；
7. 是否能使用 `wc`、`sort`、`uniq` 做简单统计；
8. 是否能使用 `awk` 或 `cut` 提取简单字段；
9. 是否能使用 `>`、`>>`、`tee` 保存命令输出；
10. 是否能从 `dmesg`、`journalctl` 中提取错误和设备相关日志；
11. 是否能提交原始文本日志，而不是只提交截图；
12. `summary.md` 是否能用自己的话说明命令用途和区别。

通过本文档学习后，实习生应具备基础的 Linux 文本查看、搜索和处理能力，能够在后续日志分析、源码搜索、配置检查、驱动调试中完成基础信息定位工作。
