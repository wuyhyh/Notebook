# U-Boot 版本规则（C 方案：Tag + Commit Hash）

本文档规定本仓库 U-Boot 的版本号生成与发布流程，用于保证：

- **对外可读（有正式版本名）**
- **对内可追溯（能定位到唯一 commit）**
- **长期一致（后续版本继续遵循同一规则）**

---

## 1. 版本字符串格式

U-Boot 启动 Banner 中版本字符串遵循：

```

U-Boot <UpstreamBase>-<ProjectTag>+g<CommitShort>

```

示例：

```

U-Boot 2019.07-v1.37-hpc-baseline-20260210+g0901837e

````

字段说明：

- `2019.07`：U-Boot 上游基线版本（vendor 基于该版本派生）
- `v1.37-hpc-baseline-20260210`：**Git Tag**（正式版本名，用于发布/基线）
- `g0901837e`：当前构建所用源码的 **Git commit 短 hash**（8 位），确保可追溯

> 该规则可以做到：看见版本字符串即可定位到唯一源码版本（tag 给人看，hash 给机器/排查用）。

---

## 2. 版本生成机制（实现点）

版本字符串由 `scripts/setlocalversion` 生成，规则为：

- 若 `HEAD` **正好**在某个 tag 上：
    - 输出 `-<tag>+g<sha8>`
- 若 `HEAD` 不在 tag 上：
    - 输出 `-g<sha8>`
- 不再输出分支名（如 `master`）
- 不再输出构建时间戳（避免同一源码重复构建版本不一致）

构建产物中最终落地的位置：

- `include/generated/version_autogenerated.h` 中的 `PLAIN_VERSION`

构建后可检查版本：

```bash
cat include/generated/version_autogenerated.h
````

## 3. 打“正式基线版本”Tag 的流程

### 3.1 前置条件

* 工作区必须干净（避免 `dirty`）：

```bash
git status --porcelain
```

输出应为空。

* 确认当前 commit 就是要发布的基线：

```bash
git log -1 --oneline
```

### 3.2 创建 annotated tag（必须带说明）

示例（基线发布）：

```bash
git tag -a v1.37-hpc-baseline-20260210 -m "U-Boot v1.37 HPC baseline (tag+gsha version scheme)"
```

### 3.3 推送 tag 到 GitLab（必须）

```bash
git push origin v1.37-hpc-baseline-20260210
```

### 3.4 验证

```bash
git show --no-patch v1.37-hpc-baseline-20260210
git describe --tags --always --dirty
```

---

## 4. Tag 同步规则（别人 clone 是否能拿到）

* 只要 tag 已推送到远端（GitLab），新 clone 的仓库**默认会包含这些 tag**。
* 已经 clone 过的仓库需要更新 tag：

```bash
git fetch --tags
```

---

## 5. 推荐的 Tag 命名规范

建议统一使用：

```
v<vendorVer>-hpc-(baseline|rc|release)-<YYYYMMDD>
```

例：

* `v1.37-hpc-baseline-20260210`
* `v1.37-hpc-rc1-20260301`
* `v1.37-hpc-release-20260315`

要求：

* 版本名应“可读、可长期沿用”
* 不要把分支名（master/dev）当版本名

---

## 6. 常见问题

### Q1: 为什么版本里不再包含分支名（master）？

分支名不是版本概念，容易误导；tag 才是正式“发布标识”。分支信息可通过 git 追溯，无需进入版本串。

### Q2: 为什么不再包含时间戳？

时间戳会导致同一 commit 在不同时间编译出的版本字符串不同，不利于“基线版本”的一致性与验收追溯。

### Q3: 如何判断某个二进制对应哪个源码？

看启动 Banner 或 `include/generated/version_autogenerated.h`：

* tag 确认发布版本
* `+g<sha8>` 定位到唯一 commit
