### 1. 进程、线程和任务的区别

- **进程（Process）**  
  进程是操作系统中资源分配的基本单位，每个进程都有独立的地址空间（虚拟内存）、代码段、数据段、堆、栈等。进程之间是相互独立的，不能直接访问彼此的内存，必须通过进程间通信（IPC）进行数据交换。

- **线程（Thread）**  
  线程是进程中的执行单元，一个进程可以包含多个线程。线程共享进程的地址空间、全局变量、文件描述符等资源，但拥有独立的寄存器、程序计数器和栈。因此，同一进程内的线程可以并发执行，线程间的切换开销比进程切换小。

- **任务（Task）**  
  任务是一个更广义的概念，通常可以指代进程或线程。例如，在 Linux
  内核中，任务（task）泛指所有可调度的实体，包括进程和线程。在某些上下文（如实时系统）中，任务指的是系统执行的特定工作单元，不一定与进程或线程一一对应。

### 2. 模式切换（Mode Switching）和上下文切换（Context Switching）

- **模式切换（Mode Switching）**  
  模式切换指的是 CPU 在用户模式（User Mode）和内核模式（Kernel Mode）之间的转换。例如，当一个用户进程调用 `read()` 访问磁盘，CPU
  需要从用户模式切换到内核模式执行系统调用，完成后再切回用户模式。这种切换涉及 CPU 特权级变化，但不会更改进程或线程的执行状态。

- **上下文切换（Context Switching）**  
  上下文切换指的是 CPU
  在不同进程或线程之间切换执行时，需要保存当前进程/线程的状态（寄存器、程序计数器、栈指针等），然后恢复下一个进程/线程的状态。上下文切换通常发生在进程调度、线程切换或中断处理时，它的开销比模式切换更大。

  **区别：**
    - 模式切换不涉及进程/线程的转换，而是同一进程内在用户态和内核态之间切换。
    - 上下文切换涉及进程/线程切换，通常伴随模式切换。

### 3. 换页（Paging）和进程交换（Swapping）的区别

- **换页（Paging）**  
  换页是一种内存管理机制，它将进程的虚拟内存划分为固定大小的页（Page），并将页映射到物理内存的页框（Frame）。如果某个页面不在物理内存中，CPU
  访问该页时会触发缺页异常（Page Fault），然后操作系统会从磁盘（如 swap 分区）中加载该页。换页是一种部分交换，通常用于按需加载数据。

- **进程交换（Swapping）**  
  进程交换是指把整个进程的数据从物理内存移到磁盘，以腾出更多内存给其他进程使用。当需要重新执行该进程时，再将其从磁盘调回物理内存。Swapping
  在早期操作系统中较常见，但由于其开销较大，现代操作系统更倾向于使用换页技术。

  **区别：**
    - 换页是按页进行数据调度，而进程交换是整个进程的调度。
    - 换页通常发生在进程运行过程中，而进程交换通常用于内存极度紧张的情况下。

### 4. I/O 密集型和 CPU 密集型工作负载的区别

- **I/O 密集型（I/O-bound）**  
  I/O 密集型任务主要受限于输入/输出操作的速度，例如磁盘读写、网络传输等。CPU 在等待 I/O 完成时通常处于空闲状态，导致 CPU
  利用率较低。例如：
    - 数据库查询
    - 文件读写操作
    - 网络服务器请求处理

- **CPU 密集型（CPU-bound）**  
  CPU 密集型任务主要依赖 CPU 计算能力，I/O 操作较少，CPU 会长时间处于高负载状态。例如：
    - 图像/视频处理
    - 复杂数学计算
    - AI/深度学习模型训练

  **优化方式：**
    - I/O 密集型任务可以通过**异步 I/O（如 `epoll`、`async/await`）**或**多线程**提高性能，以便在等待 I/O 的同时执行其他任务。
    - CPU 密集型任务通常需要**多进程或多线程**来充分利用多核 CPU，同时可以使用**GPU 加速**优化计算。
