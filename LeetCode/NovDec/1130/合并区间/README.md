LeetCode 第 56 题：**合并区间（Merge Intervals）**，是一个经典的区间合并问题。

### 问题描述

给定一个区间的集合 `intervals`，其中每个区间的格式为 `[start, end]`，合并所有重叠的区间，并返回一个不重叠的区间列表。

### 示例

**输入:**
```plaintext
intervals = [[1, 3], [2, 6], [8, 10], [15, 18]]
```

**输出:**
```plaintext
[[1, 6], [8, 10], [15, 18]]
```

**解释:**
- 区间 `[1, 3]` 和 `[2, 6]` 重叠，因此合并为 `[1, 6]`。
- 区间 `[8, 10]` 和 `[15, 18]` 没有重叠，因此不变。

---

## 解题思路

这个问题可以通过 **排序** 和 **贪心算法** 来高效解决。

### 思路分析

1. **排序**：
    - 我们首先按区间的起始位置进行排序。排序后，区间的顺序会使得我们可以从左到右检查相邻区间是否重叠。

2. **合并区间**：
    - 遍历排序后的区间，对于每个区间：
        - 如果当前区间与结果列表中的最后一个区间重叠（即当前区间的起始位置小于或等于结果列表最后一个区间的结束位置），则合并两个区间。
        - 否则，将当前区间直接添加到结果列表中。

3. **时间复杂度**：
    - 排序的时间复杂度是 `O(n log n)`，遍历一次区间的时间复杂度是 `O(n)`，因此总体时间复杂度是 `O(n log n)`。

---

## 代码实现

```python
def merge(intervals):
    # 如果区间为空或只有一个区间，直接返回
    if not intervals:
        return []

    # 按照区间的起始位置进行排序
    intervals.sort(key=lambda x: x[0])

    # 初始化合并后的区间列表
    merged = [intervals[0]]

    # 遍历所有区间，进行合并
    for current in intervals[1:]:
        # 获取合并后的最后一个区间
        last = merged[-1]
        
        # 如果当前区间与最后一个合并区间有重叠，合并它们
        if current[0] <= last[1]:
            last[1] = max(last[1], current[1])
        else:
            # 否则，直接添加当前区间
            merged.append(current)

    return merged
```

### 代码解释：

1. **排序**：
   ```python
   intervals.sort(key=lambda x: x[0])
   ```
    - 我们通过 `lambda x: x[0]` 作为排序的键，按每个区间的起始位置对区间进行排序。

2. **初始化合并后的区间列表**：
   ```python
   merged = [intervals[0]]
   ```
    - 初始时，先把第一个区间加入 `merged` 列表中。

3. **遍历区间并合并**：
   ```python
   for current in intervals[1:]:
       last = merged[-1]
       if current[0] <= last[1]:
           last[1] = max(last[1], current[1])
       else:
           merged.append(current)
   ```
    - 从第二个区间开始遍历。
    - 每次检查当前区间 `current` 是否与 `merged` 列表中的最后一个区间 `last` 重叠。如果重叠，更新最后一个区间的结束位置；如果不重叠，直接将当前区间添加到 `merged` 列表中。

4. **返回结果**：
   ```python
   return merged
   ```

## 示例

以 `intervals = [[1, 3], [2, 6], [8, 10], [15, 18]]` 为例，逐步分析合并过程：

1. **排序后的区间**：
   ```plaintext
   [[1, 3], [2, 6], [8, 10], [15, 18]]
   ```
   排序后区间仍然是相同的，因为已经是按起始位置排序的。

2. **合并过程**：
    - 初始时，`merged = [[1, 3]]`。
    - 遍历到 `[2, 6]`：
        - `current[0] = 2`，`last[1] = 3`，满足重叠条件（`2 <= 3`），合并后变为 `[1, 6]`。
    - 遍历到 `[8, 10]`：
        - `current[0] = 8`，`last[1] = 6`，没有重叠，直接添加到 `merged`，变为 `[[1, 6], [8, 10]]`。
    - 遍历到 `[15, 18]`：
        - `current[0] = 15`，`last[1] = 10`，没有重叠，直接添加到 `merged`，变为 `[[1, 6], [8, 10], [15, 18]]`。

3. **最终返回的合并结果**：
   ```plaintext
   [[1, 6], [8, 10], [15, 18]]
   ```

## 时间复杂度与空间复杂度

- **时间复杂度**：
    - 排序的时间复杂度是 `O(n log n)`，其中 `n` 是区间的数量。
    - 遍历区间的时间复杂度是 `O(n)`。
    - 因此，总时间复杂度是 `O(n log n)`。

- **空间复杂度**：
    - 由于我们需要存储合并后的区间，空间复杂度是 `O(n)`，其中 `n` 是区间的数量。

---

## 总结

1. **排序**：首先将所有区间按起始位置排序，这样就可以确保相邻的区间可能有重叠。
2. **合并**：遍历所有区间，通过检查当前区间是否与结果列表中的最后一个区间重叠来决定是否合并。
3. **时间复杂度**：排序和遍历的组合导致总时间复杂度为 `O(n log n)`，适用于处理较大的输入。
