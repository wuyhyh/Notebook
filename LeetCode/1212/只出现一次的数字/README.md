### **LeetCode 第136题 - 只出现一次的数字**

#### **题目描述**
给定一个非空整数数组 `nums`，其中除了一个元素只出现一次外，其余每个元素都出现两次。请你找出那个只出现一次的元素。

要求：**不能使用额外的空间**，并且**时间复杂度要为 O(n)**。

---

### **解题思路**

本题要求我们在 **O(n)** 时间复杂度和 **O(1)** 空间复杂度下找到只出现一次的元素。我们可以使用 **异或运算**（XOR）来高效地解决这个问题。

#### **异或运算的性质**：
1. **自反性**：`a ^ a = 0`，即任何数与自己进行异或运算的结果是 `0`。
2. **交换性和结合性**：`a ^ b = b ^ a`，且 `a ^ (b ^ c) = (a ^ b) ^ c`，即异或运算的顺序可以任意调整。
3. **与零的异或**：`a ^ 0 = a`，即任何数与 `0` 异或的结果是它本身。

根据这些性质，我们可以推导出以下结论：
- 如果我们对所有的数组元素进行异或运算，成对的元素（即出现两次的元素）会相互抵消（`a ^ a = 0`），而只出现一次的元素会保留下来。

#### **步骤**：
1. 初始化一个变量 `result` 为 `0`。
2. 遍历数组中的每个元素 `num`，将 `result` 和 `num` 进行异或操作：`result ^= num`。
3. 最终，`result` 中存储的就是只出现一次的元素。

---

### **代码实现**

```python
class Solution:
    def singleNumber(self, nums):
        result = 0
        for num in nums:
            result ^= num  # 对每个数进行异或
        return result
```

---

### **代码解析**
1. 初始化 `result = 0`。
2. 遍历数组 `nums` 中的每个元素 `num`，对 `result` 和 `num` 进行异或操作。
3. 最后，`result` 中存储的就是那个只出现一次的数字。

由于异或的特性，成对出现的数字会相互抵消，最后留下的就是只出现一次的数字。

---

### **复杂度分析**
- **时间复杂度**：`O(n)`，其中 `n` 是数组的长度。我们只遍历一次数组。
- **空间复杂度**：`O(1)`，我们只使用了一个额外的变量 `result`。

---

### **示例测试**

#### 示例 1：
```python
nums = [2, 2, 1]
solution = Solution()
print(solution.singleNumber(nums))  # 输出 1
```

#### 示例 2：
```python
nums = [4, 1, 2, 1, 2]
solution = Solution()
print(solution.singleNumber(nums))  # 输出 4
```

#### 示例 3：
```python
nums = [1]
solution = Solution()
print(solution.singleNumber(nums))  # 输出 1
```

---

### **总结**
- 本题的关键在于利用**异或运算**的性质来抵消出现两次的元素，只留下出现一次的元素。
- 异或运算能够在 **O(n)** 时间复杂度和 **O(1)** 空间复杂度下完成任务，非常高效。
